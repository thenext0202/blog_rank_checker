## 목표

`body_text.txt`의 본문을 의미 단위로 `slot_count`개의 슬롯으로 분할하고, 각 슬롯 시작점과 표지 자리(`000`)에 마커를 삽입해 `manuscript_marked.md`를 만든다.

권유 블록(제품/링크 안내 단락)은 마커 없이 본문에 그대로 두고, **그 위치를 별도 메타 파일(`manuscript_marked.meta.json`)에 기록한다.**

## 작업 주체

이 단계는 **Claude Code(LLM)가 직접 본문을 읽고 의미 단위로 분할한다.** Python 스크립트가 자동으로 분할하지 않는다. 자동 분할 스크립트를 새로 만들지 않는다.

**권유 블록 식별도 LLM 책임이다.** 검증기는 의미를 판단하지 않는다.

검증만 별도 스크립트(`src/step3_verifier.py`)로 자동화한다. 검증기는 형식 검사만 하며, 권유 블록 위치는 LLM이 출력한 메타 파일에서 읽는다.

## 핵심 개념 정의

### 단락 블록
빈 줄로 구분되는 연속된 줄 묶음. 한 단락 안의 줄바꿈은 시각용(블로그 가독성)일 뿐 의미 단위 아님. 단락은 의미상 한 덩어리다.

### 소제목
다음 중 하나에 해당하는 짧은 단독 줄:
- 굵은 마크업(`**...**`)으로 둘러싸인 단독 줄 (예: `**밥부터 바꿨다**`)
- 번호가 매겨진 단독 줄 (예: `1. 중성지방과 음식의 연관성`)

**소제목 + 본문 처리:**
- 소제목과 다음 본문 줄 사이에 빈 줄이 있으면 → 별도 단락
- 소제목과 다음 본문 줄 사이에 빈 줄이 없으면 → 같은 단락 (분리 금지)

### 장면
같은 시점·행동·감정을 공유하는 단락 묶음. "한 이미지가 표현할 만한 단위" = 한 장면.

### 권유 블록
제품·링크를 권유하는 단락(들)의 묶음. **마커를 박지 않고**, 본문에는 그대로 보존하고, **위치를 메타 파일에 기록한다.**

## 처리

### 1. 권유 블록 식별 (분할 전 선행 작업)

본문 흐름을 읽고 권유 블록을 **의미상** 식별한다. 작가가 독자에게 "이 제품/링크 한번 보세요"라고 권하는 단락 묶음이 권유 블록이다.

**식별 힌트 (참고용 — 키워드만으로 자동 결정 금지):**

키워드 단서:
- `제품`, `링크`, 특정 제품명/배합명

권유 표현 단서:
- `찾아보세요`, `찾아보시는 것도`
- `참고해보세요`, `참고만 하시면 된다`
- `한번 시도해보세요`, `시도해보는 것도`
- `괜찮을 것 같다`
- `남겨둔다`, `남겨둘 테니`

**권유 블록이 아닌 것 (자주 헷갈리는 케이스):**
- 면책 언급 ("제품 자체가 어떻다는 게 아니고", "효과는 사람마다 다르다")
- 일반 본문의 제품 후기 ("그래서 이 제품을 한 달 먹어봤다")
- 격려·위로 ("힘내세요", "같은 고민 하시는 분")
- 임팩트 마무리 (짧은 굵은 마무리 한두 줄)

**판단 기준은 권유 의도다.** 키워드가 있어도 후기·면책이면 권유 블록 아님. 키워드가 없어도 명확한 권유 표현이 본문에 있으면 권유 블록일 수 있음. 본문 흐름과 작가의 의도로 판단한다.

**확장 규칙:**

식별한 권유 단락에서 위·아래로 인접 단락을 검사하고, 같은 권유 의도가 이어지면 묶는다. 격려·일반 본문·임팩트 마무리가 나오면 거기서 멈춘다.

**단락 합침 케이스 (단락 보존 원칙 우선):**

권유 의도가 있는 줄이 호명·도입과 **빈 줄 없이 한 단락**으로 합쳐져 있으면, 그 합쳐진 단락 전체가 권유 블록에 포함된다. 빈 줄을 새로 끼워 단락을 분리하지 않는다.

이는 **필수 제약 2 — 단락 안은 절대 자르지 마라**가 권유 블록을 깔끔하게 분리하는 것보다 우선하기 때문이다. 작가가 빈 줄 없이 합쳐 썼으면 그게 작가의 호흡이고, 본문 텍스트는 어떤 경우에도 손대지 않는다.

**위치 보존:**

권유 블록은 본문에서 발견된 위치 그대로 둔다. 위치를 옮기지 않는다. 한 원고에 여러 개 있을 수 있다.

### 2. 본문 분할

권유 블록을 제외한 나머지 본문을 의미 단위로 `slot_count`개 슬롯으로 분할한다.

**필수 제약 — 어기면 실패**

1. 정확히 `slot_count`개 슬롯 (권유 블록은 슬롯 카운트에 포함하지 않음)
2. 단락 안(빈 줄로 구분되는 단위)은 절대 자르지 마라
3. 본문 글자를 한 자도 빼거나 추가하지 마라
4. 강조 마크업(`**...**`)을 그대로 보존
5. 본문 줄 순서를 바꾸지 마라
6. 권유 블록 안에는 마커를 박지 마라

**권장 가이드 — 더 좋은 결과를 위해**

1. 한 슬롯 = 한 장면 (같은 시점·행동·감정)
2. 슬롯 첫 단락은 그 장면의 도입이 자연스럽다
3. 화제가 바뀔 때 슬롯 경계로 삼는다
4. 글자수 편차 신경 쓰지 마라 (40자 ~ 370자 다 정상 범위)
5. 마지막 슬롯이 짧은 한 줄 마무리여도 괜찮다
6. 굵은 소제목은 슬롯 시작이 될 수도, 슬롯 끝(다음 슬롯 예고)이 될 수도 있다 — 어느 쪽이 자연스러운지 본문 흐름으로 판단하라

**우선순위 — 트레이드오프 만났을 때**

의미 단위 우선 > 글자수 무관 > 단락 보존 > 슬롯 수 정확

단, "필수 제약"의 6개 항목은 위 우선순위와 무관하게 무조건 지킨다.

### 3. 마커 삽입

각 슬롯 시작점에 마커를 단독 줄로 삽입한다.

**마커 형식:**
- 3자리 0-패딩 숫자: `001`, `002`, ..., `017`
- 단독 줄 (다른 텍스트 없음)
- 앞뒤로 빈 줄 한 개씩

**표지 마커 (`000`) 처리:**
- 파일 맨 위에 `000`을 단독 줄로 박는다
- `000` 다음 빈 줄 → `001` → 빈 줄 → 본문 첫 슬롯
- 표지 자리는 `slot_count`에 포함되지 않는다 (즉 본문 슬롯 13장이면 마커는 `000`, `001`~`013`으로 총 14개)

**권유 블록 처리:**
- 권유 블록 안에는 마커를 박지 않는다
- 권유 블록은 본문 흐름 그대로 두고, 그 앞·뒤 슬롯에 마커가 박힌다

### 4. 빈 줄 처리

**슬롯 내부 빈 줄:**
- 원본의 빈 줄을 그대로 보존한다
- 1개·2개·3개 통일하지 않는다 (작가가 의도한 호흡)

**슬롯 경계 빈 줄 (마커 둘러싸는 빈 줄):**
- 마커 앞뒤로 빈 줄 1개씩 통일
- 원본의 슬롯 경계에 빈 줄이 여러 개 있어도 1개로 정리

**권유 블록 빈 줄:**
- 원본 그대로 보존

### 5. 출력 형식

**`manuscript_marked.md`:**

```
000

001

[슬롯 1의 본문 내용]

002

[슬롯 2의 본문 내용]

...

[권유 블록이 위치한 자리에 그대로 본문 보존, 마커 없음]

...

013

[슬롯 13의 본문 내용]
```

**`manuscript_marked.meta.json` (v1.3 신규):**

```json
{
  "manuscript": "manuscript_marked.md",
  "total_slots": 13,
  "recommendation_blocks": [
    {"start_line": 152, "end_line": 168}
  ]
}
```

- `manuscript`: 분할본 파일명
- `total_slots`: 본문 슬롯 수 (`slot_count`. 표지 `000` 제외)
- `recommendation_blocks`: 권유 블록 줄 범위 배열
  - `start_line`, `end_line`: `manuscript_marked.md` 기준 **1-based 줄 번호**
  - 권유 블록의 첫 비-빈 줄부터 마지막 비-빈 줄까지
  - 권유 블록이 없으면 빈 배열 `[]`
  - 여러 개 있으면 배열에 여러 개

## 검증 체크리스트

`src/step3_verifier.py`가 다음을 자동 검사한다.

**필수 검사 (어기면 실패):**

1. 마커 개수가 `slot_decision.json`의 `slot_count + 1` (000 포함)과 일치
2. 마커가 `000`, `001`, `002`, ..., `슬롯수` 순서대로 누락 없이 등장
3. 모든 마커가 단독 줄에 있고, 앞뒤로 빈 줄 또는 파일 경계가 있음
4. 마커 제외한 본문 글자수가 `body_text.txt`의 본문 글자수와 일치 (`count_body_chars` 사용)
5. 강조 마크업(`**`) 개수가 `body_text.txt`와 동일
6. **메타 파일에 명시된 권유 블록 줄 범위에 마커가 박혀 있지 않음** — 검증기는 메타 파일에서 위치를 읽고, 권유 블록을 자동 식별하지 않는다

**참고 통계 (보고만, 통과/실패 판단 안 함):**

7. 슬롯별 글자수
8. 슬롯별 단락 수
9. 슬롯별 첫 줄 미리보기 (앞 40자)
10. 권유 블록 개수와 위치 (메타 파일 기반, 어느 슬롯과 어느 슬롯 사이에 있는지)

## 작업 후 보고 (필수)

다음 항목을 모두 보고한다.

1. **분할 결과 요약**
   - 슬롯 수: `slot_count`와 일치하는지
   - 본문 글자수 보존: 분할 전 == 분할 후
   - 강조 마크업 보존: 분할 전 개수 == 분할 후 개수
   - 권유 블록 개수와 위치 (`manuscript_marked.md` 줄 번호 기준)
   - 메타 파일 출력 확인

2. **검증 스크립트 결과**
   - 필수 6개 검사 각각 통과/실패
   - 실패 시 사유

3. **슬롯별 통계**
   - 슬롯 번호 / 글자수 / 단락 수 / 첫 줄 미리보기 표 형식
   - 권유 블록은 별도 표시 (예: `[권유블록] | 줄 X~Y | 글자수 | 슬롯 N과 N+1 사이`)

4. **분할 사유 메모 (필수)**
   - 슬롯 경계를 어디에 뒀는지
   - **권유 블록을 어떻게 식별했는지** — 어떤 단락을 권유로 판단했고, 위·아래로 어떻게 확장했고, 어디서 멈췄는지
   - 헷갈리는 케이스(면책·후기처럼 보이는 단락)가 있었으면 어떻게 결정했는지
   - 짧은 슬롯이나 긴 슬롯이 생긴 이유

## 참고: 분할 사고 흐름 가이드

본문을 분할할 때 다음 순서로 사고하면 일관된 결과가 나온다.

**1단계 — 권유 블록 식별**

본문 전체를 읽고 작가가 제품·링크를 권유하는 단락 묶음이 있는지 확인한다. 키워드(`제품`, `링크`)와 권유 표현(`찾아보세요`, `참고하시면` 등)은 단서로 참고하되, **의미상 권유 의도가 명확한 단락만** 권유 블록으로 본다. 면책·후기·격려는 권유 블록 아님.

**2단계 — 큰 흐름 잡기**

권유 블록을 제외한 나머지 본문을 처음부터 끝까지 읽고 큰 화제 흐름을 5~7개 정도로 잡는다.
예: "검진 충격 → 원인 진단 → 식단 변경 → 운동 → 배합 도입 → 변화 → 마무리"

**3단계 — 장면으로 쪼개기**

각 큰 화제 안에서 "한 이미지가 표현할 만한 단위"로 쪼갠다. 한 장면의 기준: 같은 시점, 같은 행동, 같은 감정.

**4단계 — 슬롯 수 맞추기**

3단계 결과의 장면 수가 목표 `slot_count`와 다르면 조정한다.

- 장면이 더 많으면 → 인접하고 같은 화제인 두 장면을 합친다
- 장면이 더 적으면 → 가장 정보가 많은 큰 장면을 두 장면으로 쪼갠다
- 글자수는 신경 쓰지 마라

**5단계 — 마커 박기 + 메타 파일 출력**

각 슬롯 시작점에 단독 줄로 마커를 박는다. 권유 블록은 건너뛰고 본문 그대로 둔다. 본문 한 글자도 손대지 않는다.

**그 다음 `manuscript_marked.meta.json`을 만들어 권유 블록 위치(`manuscript_marked.md` 기준 1-based 줄 번호)를 기록한다.** 권유 블록이 없는 원고면 빈 배열로.

## 권유 블록 식별 예시

### 예시 1: hc — 권유 블록이 본문 끝

원본:
```
완전히 괜찮아진 건 아니지만,
예전처럼 하루에 두세 번
화장실로 뛰어가는 일은 없어졌어요.

장 관리에 관심이 있으시다면,
스토마이신 배합 한번 찾아보세요.

무료 반품도 가능하다고 하니,
이럴 때 시도해보는 것도 좋을 듯합니다.
```

식별:
- "장 관리에 관심이 있으시다면 ... 한번 찾아보세요" — 명확한 권유 의도. 권유 블록.
- "무료 반품도 가능하다고 하니, ... 시도해보는 것도" — 같은 권유 흐름. 권유 블록에 포함.
- 위로 확장: `완전히 괜찮아진 건 아니지만...`은 후기, 권유 의도 없음 → 멈춤.
- 권유 블록 = 마지막 두 단락.

### 예시 2: sc — 권유 블록이 본문 끝, 격려·마무리가 뒤따름

이 예시는 단락 합침 여부에 따라 두 가지 케이스가 있다. 어느 케이스인지는 `body_text.txt`의 빈 줄 구조로 결정된다.

#### 예시 2a — 호명·도입이 별도 단락 (이상적 케이스)

원본:
```
콘드로이친부작용이 걱정되셔서
이 글까지 읽고 계시는 분이라면,

나노카틸 배합
한번 찾아보시는 것도 괜찮을 것 같다.

내가 요즘 챙기고 있는 제품 정보는
아래 남겨둘 테니,
궁금하신 분만 참고하시면 된다.

무릎이 불편하면
마음까지 움츠러들게 된다.
...
```

식별:
- 두 번째·세 번째 단락에 명확한 권유 표현(`찾아보시는 것도`, `남겨둘 테니`, `참고하시면 된다`) → 권유 블록.
- 위로: `콘드로이친부작용이 걱정되셔서...`는 호명·도입, 권유 아님 → 멈춤.
- 아래로: `무릎이 불편하면 마음까지 움츠러들게 된다`는 격려, 권유 아님 → 멈춤.
- 권유 블록 = **두 단락**.

#### 예시 2b — 호명·도입이 권유와 빈 줄 없이 합쳐진 단락 (sc 실제 케이스)

원본:
```
콘드로이친부작용이 걱정되셔서
이 글까지 읽고 계시는 분이라면,
나노카틸 배합
한번 찾아보시는 것도 괜찮을 것 같다.

내가 요즘 챙기고 있는 제품 정보는
아래 남겨둘 테니,
궁금하신 분만 참고하시면 된다.

무릎이 불편하면
마음까지 움츠러들게 된다.
...
```

식별:
- 첫 단락에 호명·도입(`콘드로이친부작용이 걱정되셔서...분이라면`)과 권유(`나노카틸 배합 한번 찾아보시는 것도...`)가 **빈 줄 없이 한 단락**으로 합쳐져 있음.
  - 단락 보존 원칙상 분리 불가. 빈 줄을 새로 끼우지 않는다.
  - 권유 의도가 단락 안에 들어있으므로 **합쳐진 단락 전체를 권유 블록에 포함**.
- 두 번째 단락(`내가 요즘 챙기고 있는 제품 정보는...`)도 권유 표현 → 권유 블록에 포함.
- 아래로: `무릎이 불편하면...`은 격려 → 멈춤.
- 권유 블록 = **두 단락** (첫 단락 = 호명+권유 합침 전체, 두 번째 단락 = 추가 권유).

→ 식별 결과는 2a와 같은 "두 단락"이지만, 첫 단락의 시작점이 다르다 (2a는 "나노카틸..."부터, 2b는 "콘드로이친..."부터). 메타 파일의 `start_line`은 합쳐진 단락의 첫 줄이 된다.

### 예시 3: bc2 — 권유 블록이 본문 끝쪽 중간

원본:
```
중성지방관리는 결국 식단, 운동,
그리고 꾸준히 챙기는 거...
어느 하나만 해서 되는 게 아니라
같이 가야 흐름이 만들어지더라.

요즘 먹고 있는 건
**블러디션 배합**이 들어간 제품인데,
궁금하신 분들은 참고만 하시면 된다.
아래에 정보 남겨둔다.

같은 고민을 하고 있는 분이 있다면,
너무 겁먹지 말고 할 수 있는 것부터
하나씩 바꿔보시라.

**몸은 생각보다 솔직하게 반응하더라.**
**중성지방관리, 지금 시작해도 늦지 않았다.**
```

식별:
- `요즘 먹고 있는 건 **블러디션 배합**이 들어간 제품인데, 궁금하신 분들은 참고만 하시면 된다. 아래에 정보 남겨둔다.` — 권유 의도 명확. 권유 블록.
- 아래로: `같은 고민을 하고 있는 분이 있다면...`은 격려 → 멈춤.
- 격려와 임팩트 마무리는 본문 슬롯에 포함.

## 자주 헷갈리는 케이스

**면책 단락은 권유 블록이 아니다.**

예: "제품 자체가 어떻다는 게 아니고, 사람마다 효과는 다를 수 있다."
→ "제품" 키워드가 있지만 작가가 권하는 게 아니라 면책하는 것. 본문 슬롯에 포함.

**일반 본문의 제품 후기는 권유 블록이 아니다.**

예: "그래서 이 배합 제품을 한 달 정도 먹어봤다. 처음엔 큰 변화 없었는데..."
→ 후기 흐름. 권유 표현 없음. 본문 슬롯에 포함.

**짧은 권유 한 줄도 권유 블록이다.**

예: "관심 있으면 본문 아래 링크 참고."
→ 권유 의도 명확. 한 줄이라도 권유 블록.

**단락 보존이 권유 블록 분리보다 항상 우선한다.**

권유 의도가 있는 줄과 호명·도입이 빈 줄 없이 한 단락으로 합쳐져 있으면, 합쳐진 단락 전체를 권유 블록으로 처리한다. 빈 줄을 새로 끼워서 단락을 자르지 마라. 본문 텍스트는 어떤 경우에도 손대지 않는다 (예시 2b 참고).

## 절대 원칙 재확인

- 의미 단위 분할이 최우선. 글자 수 균등화로 문단 중간 자르기 금지.
- 본문에는 마커 외 어떤 메타정보도 노출 금지. 메타 정보는 별도 `.meta.json`에.
- 본문 텍스트 자체는 손대지 않음 (분할만 함).
- 강조 마크업(`**...**`)은 그대로 보존.
- 권유 블록은 위치를 옮기지 않고, 본문 흐름 그대로 둔다.
- **권유 블록 식별은 LLM 책임. 검증기는 메타 파일을 신뢰한다.**
