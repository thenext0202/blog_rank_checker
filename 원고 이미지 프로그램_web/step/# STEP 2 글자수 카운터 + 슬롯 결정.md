# STEP 2: 글자수 카운터 + 슬롯 결정

> 전제: `원고 이미지 프로그램_web.md` 오리엔테이션을 이미 읽었음.
> STEP 1이 통과되어 `output/body_text.txt`가 만들어진 상태.
> 이 단계만 작업한다. 단계 3 이후의 기능(분할, 마커 삽입 등)은 만들지 않는다.

---

## 목표
`body_text.txt`의 본문 글자수를 측정하고, 분량 규칙에 따라 본문 슬롯 수를 결정한다.

---

## 입력
- `output/body_text.txt` (STEP 1의 산출물)

---

## 처리

### 1. 글자수 측정
- `src/utils.py`의 `count_body_chars()` 함수를 그대로 사용한다.
- **자체 카운팅 로직을 새로 만들지 않는다.** 일관성을 위해 반드시 utils 함수만 쓴다.

### 2. 슬롯 수 매핑

| 본문 글자수 | 본문 슬롯 수 | 경고 |
|---|---|---|
| 1,800 미만 | 13 | "원고 분량이 짧음" 경고 표시 |
| 1,800 ~ 1,999 | 13 | 없음 |
| 2,000 ~ 2,199 | 14 | 없음 |
| 2,200 ~ 2,499 | 15 | 없음 |
| 2,500 이상 | 17 | 없음 (상한) |

표지 슬롯(`000`)은 위 카운트에 **포함되지 않는다.** 표지는 STEP 5에서 별도 배정.

### 3. 경계값 처리

경계는 표 그대로 따른다:
- 1,799자 → 13장 + 경고
- 1,800자 → 13장
- 1,999자 → 13장
- 2,000자 → 14장
- 2,199자 → 14장
- 2,200자 → 15장
- 2,499자 → 15장
- 2,500자 → 17장 (16장 구간은 없음)

### 4. 분량 짧음 경고

글자수가 1,800자 미만일 때:
- `slot_count`는 **13으로 설정** (작업 진행)
- `warning` 필드에 사유와 글자수 명시
- 처리는 그대로 진행 (멈추지 않음)

---

## 출력

`output/slot_decision.json`

### 정상 케이스
```json{
"char_count": 2210,
"slot_count": 15,
"warning": null
}

### 분량 짧음 케이스
```json{
"char_count": 1720,
"slot_count": 13,
"warning": "본문 글자수 1,720자. 1,800자 미만이라 13장 분할이 빡빡할 수 있음. 결과 확인 필요."
}

---

## 검증 체크리스트

1. `count_body_chars()` 함수를 `utils`에서 import해 사용하는가 (자체 카운팅 로직이 새로 만들어지지 않았는가)
2. 글자수가 STEP 1 보고의 "본문 추정 글자수"와 일치하는가
3. 슬롯 수가 표대로 정확히 매핑되는가
4. 경계값(1799/1800/1999/2000/2499/2500)에서 정확한 매핑이 되는가 (단위 테스트로 검증)
5. 1,800자 미만일 때 slot_count가 13이고 warning이 명확한가
6. 1,800자 이상일 때 warning이 null인가
7. JSON이 유효한 형식이고 필드명이 정확(`char_count`, `slot_count`, `warning`)한가

---

## 작업 후 보고 (필수)

1. **글자수**: `count_body_chars()` 결과
2. **슬롯 수**: 매핑된 값
3. **경고 여부**: warning 필드 내용 (없으면 "없음")
4. **단위 테스트 결과**: 경계값 6개에 대한 슬롯 수 출력. 다음 형식으로:1799 → 13 (warning)
1800 → 13
1999 → 13
2000 → 14
2499 → 15
2500 → 17
5. STEP 1에서 처리한 두 샘플 모두에 대해 결과 출력
   - 샘플1 (260414 중성지방낮추는음식): char_count, slot_count, warning
   - 샘플2 (260427 중성지방관리): char_count, slot_count, warning

---

## 작업 후 멈춤
산출물(slot_decision.json) + 위 보고 항목을 보여주고 멈춘다. **단계 3으로 넘어가지 않는다.** 사람이 검증한 뒤 통과 여부를 알려준다.

테스트는 두 샘플 모두 돌려서 비교 보고한다. 단, `slot_decision.json`은 마지막 실행 결과로 덮어써도 됨(어느 샘플로 덮였는지 보고에 명시).
