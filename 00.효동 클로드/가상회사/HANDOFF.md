## 🎮 GUI 사무실 (v1.4~v1.5)

### 화면 구성
```
┌─────────────────────────────────────────────┐
│  [민준방] [수진방] [현우방] [지은방] [태현방]   │  ← 직원 클릭 → 능력 팝업
│                                             │
│  [   회의실   ] [  프로젝트실  ] [  CEO실  ] │  ← 업무 시 이동
├─────────────────────────────────────────────┤
│  채팅 로그                                   │
│  [입력창                         ] [전송]    │
└─────────────────────────────────────────────┘
```

### 핵심 기능
- **직원 캐릭터**: 자기 방에 대기 (색상 원형 + 이름 + 상태등)
- **업무 이동**: 1명 → 프로젝트실, 2명+ → 회의실
- **능력 팝업**: 직원 클릭 → 전문 분야 + "대화하기" 버튼
- **상태 표시**: 대기(초록), 이동(노랑), 작업(주황), 검색(파랑)
- **말풍선**: 직원의 현재 작업 표시 (v1.5)
- **중단 버튼**: 전송→중단(빨간) 전환, 다음 직원 호출 건너뜀 (v1.5)
- **API 키 GUI 입력**: .api_key 파일 없으면 입력 화면 (v1.5)

### on_event 콜백 (company.py → GUI)
| 이벤트 | 내용 | GUI 동작 |
|--------|------|----------|
| `routing` | 민준 배정 결과 | 직원을 회의실/프로젝트실로 이동 |
| `agent_start` | 작업 시작 | 상태 "working", 말풍선 표시 |
| `agent_done` | 작업 완료 | 자기 방 복귀 |

## 📚 직원 이력 시스템 (v1.6)

### 자동 기록
- 직원이 작업 완료할 때마다 옵시디언 마크다운 파일에 자동 append
- 저장 위치: `옵시디언/효진의 창고/05.Automation_DB/가상회사/직원 이력/`
- 파일: `수진_콘텐츠.md`, `현우_트래픽.md`, `지은_리서치.md`, `태현_자동화.md`, `민준_팀장.md`
- 용량 제한 없음 — 영구 보관 (기존 `notes.json`의 100건 제한과 별도)
- 각 항목: 날짜, 업무 내용(200자), 결과(800자)

### 학습 (프롬프트 주입)
- 직원 호출 시 해당 직원의 이력 파일에서 **최근 10건** 자동 읽기
- 시스템 프롬프트 뒤에 `## 지금까지 내가 한 업무 이력` 섹션으로 주입
- 직원이 이전 작업을 참고해서 일관성 유지 + 개선
- 이력 없으면 주입 안 함 (비용 추가 없음)

### 흐름
```
직원 작업 완료
    ↓
_feed_manager() → _save_note()
    ↓                    ↓
민준 이력 저장     notes.json (100건 제한)
                         ↓
                  _append_history_md()
                         ↓
              옵시디언 MD 파일 (영구 보관)

다음 호출 시:
_call_agent() → _load_history_md() → 최근 10건 → 시스템 프롬프트에 주입
```

## ⚙️ 핵심 로직

### 민준 팀장 라우팅
- JSON으로 응답: `{"routing": [{"agent": "ID", "task": "지시"}], "message": "보고"}`
- routing 배열 순서대로 직원 순차 호출
- 건강기능식품 관련 → `research_support` 먼저 배정
- 단순 인사/잡담 → routing 빈 배열

### 민준의 전체 맥락 관리
- `_feed_manager()`: 모든 직원 작업 결과 → 민준 이력에 자동 저장 (API 0원)
- 직접 대화 결과도 민준에게 보고
- `/민준`으로 전체 현황 파악 가능

### 직원 간 협업
- 이전 직원 결과 → 다음 직원에게 `[이전 팀원 작업 결과]`로 전달
- 순차 실행 (병렬 아님)
