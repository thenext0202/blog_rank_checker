## 🔨 v1.0 — 초기 구축

### Before
- 없음 (신규 프로그램)

### After
- `config.py`: 5명 직원 정의 (시스템 프롬프트 포함)
- `company.py`: CEO → 민준 라우팅 → 직원 호출 → 결과 취합
- `main.py`: 터미널 채팅 인터페이스
- 모델 티어링: Haiku(fast) / Sonnet(standard)

## 🔨 v1.1 — 웹 검색 도구

### Before
- 직원들이 자체 지식만으로 응답

### After
- `tools.py` 신규: `web_search`, `news_search`, `naver_search`
- DuckDuckGo 기반 무료 검색
- Anthropic tool_use로 직원이 검색 도구 호출
- 도구 호출 최대 3회 제한 (비용 폭발 방지)
- 팀장 민준은 도구 사용 불가 (라우팅만)

---

## 🔨 v1.2 — CEO 구조 변경

### Before
- CEO도 AI 에이전트로 구현
- 사용자가 CEO를 거쳐 민준에게 전달

### After
- CEO = 치답이 (사용자 본인)
- 민준 = 팀장 (전체 맥락 관리자)
- `_feed_manager()`: 모든 직원 결과가 민준 이력에 자동 저장 (API 0원)
- `/민준` 명령어: 민준에게 직접 질문 (현황 파악)
- 직접 대화 결과도 민준에게 보고

---

## 🔨 v1.3 — 기억 시스템

### Before
- 프로그램 종료 시 대화 이력 소멸
- 뭘 시켰는지 기록 없음

### After
- `data/histories.json`: 대화 이력 영구 저장 (직원당 최근 20턴)
- `data/notes.json`: 업무 노트 영구 저장 (최근 100건)
- 매 대화 후 + 종료 시 자동 저장
- `/메모`: 최근 업무 노트 10건 조회
- `/리셋`: 대화 이력 초기화 (노트 유지)
- `run.bat`: 더블클릭 실행 배치 파일

---

## 🔨 v1.4 — pygame GUI 사무실

### Before
- 터미널 텍스트만

### After
- `gui_app.py` 신규 (pygame 기반)
- 사무실 2D 맵: 개인실 5개 + 회의실 + 프로젝트실 + CEO실
- 직원 캐릭터: 색상 원형 + 이름 + 상태등(초록/노랑/주황)
- 업무 시 이동 애니메이션: 1명→프로젝트실, 2명+→회의실
- 능력 팝업: 직원 클릭 → 전문 분야 + "대화하기" 버튼
- 작업 중 후광 효과 (깜빡임)
- `on_event` 콜백: company.py → GUI 이벤트 전달
- API 호출은 별도 스레드 (GUI 멈춤 없음)
- `run_gui.bat`: GUI 버전 실행 배치

---

## 🔨 v1.5 — 말풍선, 중단, 공통 규칙

### Before
- 직원이 뭘 하는지 시각적으로 안 보임
- 처리 중 중단 불가
- 규칙 수정 시 5명 프롬프트 각각 수정

### After
- **말풍선**: 직원 캐릭터 위에 현재 작업/검색어 표시
- **중단 버튼**: 전송→중단(빨간) 전환, `is_cancelled` 콜백으로 다음 직원 건너뜀
- **공통 규칙**: `config.py`의 `COMMON_RULES` → 전 직원 자동 적용
  - 전문용어 금지, 쉬운 존댓말, 마크다운 미사용
- **API 키 GUI 입력**: `.api_key` 파일 없으면 pygame 입력 화면
  - 마스킹 표시 (앞 8자만 보여주고 나머지 `*`)
  - Ctrl+V 붙여넣기 지원
- **가독성 개선**: 마크다운 기호(`**`, `##`, ` ``` `) 제거 후 표시
