# 🔧 개별 HANDOFF — v1.6 직원 이력 자동 기록 + 학습

> **날짜:** 2026-03-30
> **버전:** v1.5 → v1.6
> **작업:** 직원 업무 이력을 옵시디언에 영구 기록 + 다음 호출 시 프롬프트에 자동 주입

## 📋 작업 요약

| 항목 | 내용 |
|------|------|
| **요청** | 직원 이력을 쉽게 볼 수 있게 + 정보 쌓여서 직원이 발전하게 |
| **해결** | 옵시디언 MD 파일에 영구 기록 + API 호출 시 최근 이력 프롬프트 주입 |
| **수정 파일** | `company.py` |

## 🔨 변경 내용

### Before
- `data/notes.json`에 최근 100건만 저장 (초과 시 오래된 건 삭제)
- `data/histories.json`에 직원당 20턴만 저장
- 직원이 이전에 뭘 했는지 전혀 모르고 매번 새로 시작

### After
- **옵시디언 영구 기록**: 작업 완료 시 `직원 이력/수진_콘텐츠.md` 등에 자동 append
- **학습 주입**: 직원 호출 시 최근 10건을 시스템 프롬프트에 자동 추가
- 기존 `notes.json` 저장은 그대로 유지 (하위 호환)

## 🔨 코드 변경 상세 (`company.py`)

### 추가된 상수

```python
# 직원 이력 저장 폴더 (옵시디언)
HISTORY_MD_DIR = "~/Desktop/옵시디언/효진의 창고/05.Automation_DB/가상회사/직원 이력"

# 직원 ID → 이력 파일명
HISTORY_FILES = {
    "manager": "민준_팀장.md",
    "content_director": "수진_콘텐츠.md",
    ...
}

# 프롬프트 주입 시 최근 몇 건 참고할지
HISTORY_INJECT_COUNT = 10
```

### 추가된 메서드

| 메서드 | 역할 |
|--------|------|
| `_append_history_md()` | 작업 완료 시 옵시디언 MD 파일에 이력 항목 추가 |
| `_load_history_md()` | 최근 10건 이력을 읽어서 프롬프트 주입용 텍스트 반환 |

### 수정된 메서드

| 메서드 | 변경 |
|--------|------|
| `_save_note()` | 마지막에 `_append_history_md()` 호출 추가 |
| `_call_agent()` | 시스템 프롬프트에 `_load_history_md()` 결과 추가 |

## 💰 비용 영향

- 이력 10건 주입 시 시스템 프롬프트 길이 증가 → 입력 토큰 소폭 증가
- `cache_control: ephemeral` 적용되어 있어서 같은 직원 연속 호출 시 캐싱됨
- 이력 없으면 주입 안 함 → 첫 사용 시 비용 추가 없음
