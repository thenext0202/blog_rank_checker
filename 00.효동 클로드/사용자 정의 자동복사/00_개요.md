## 1. 시스템 목적

Café24 사용자정의채널 매출 Excel 파일을 읽어 **Google Sheets에 자동 업로드**하는 CLI 프로그램.
날짜별 안전한 데이터 교체(행 삽입/삭제)로 기존 데이터를 보호한다.

## 5. 전체 워크플로우

```
① config.json에서 엑셀 폴더 경로 로드 (없으면 폴더 선택 다이얼로그)
    ↓
② 폴더에서 오늘 수정된 "사용자정의채널_*.xlsx" 탐색
    ↓
③ 파일명에서 날짜 추출 (YYYY-MM-DD 또는 범위)
    ↓
④ pandas로 Excel 읽기 → 행 리스트 변환
    ↓
⑤ Google Sheets "테스트" 탭에 업로드
   ├── 같은 날짜 존재: 행 수 차이만큼 삽입/삭제 후 덮어쓰기
   └── 새 날짜: 끝에 추가 (100행 배치)
    ↓
⑥ 시트 링크 출력
```

| 단계 | 커버 여부 |

| ① 파일 탐색 (패턴+날짜) | ✅ 완전 |

| ③ Excel 읽기 | ✅ 완전 |

| ⑤ 기존 데이터 보호 (행 관리) | ✅ 완전 |

## 8. 종합 평가

### 잘된 점
- **스마트 행 관리** — 같은 날짜 교체 시 행 삽입/삭제로 뒷 데이터 보호
- **A:R 범위 제한** — T+ 컬럼의 기존 데이터 안전
- **append 미사용** — 피드백 반영, 범위 직접 지정 방식
- **config.json** — 폴더 경로 영속 저장, 재실행 시 자동 로드
- **배치 업로드** — 100행 단위로 API 효율화
- **bat 파일** — 비개발자도 더블클릭으로 실행

### 부족한 점
- **예외 처리 부족** — API/Excel 에러 시 try-except 없이 크래시
- **업로드 검증 없음** — 기록된 데이터 대조 미구현
- **탭명 하드코딩** (`"테스트"`) — 실제 운영 탭으로 변경 필요할 수 있음
- **GUI 없음** — 콘솔 출력만 (다른 프로그램과 달리)
- **pandas 의존** — requirements.txt에 누락
- **중복 실행 방지 없음** — 같은 파일을 실수로 여러 번 업로드 가능

## 9. 핵심 피드백 (기록됨)

> **append 금지** — `ws.append_rows()` 사용 시 T+ 컬럼 데이터와 위치 불일치.
> 반드시 `ws.update()` + 명시적 범위 지정으로 처리할 것.
