# 📢 댓글 알림 v2.2 — 비공개 조치 감지 기능

> **날짜:** 2026-03-23
> **버전:** v2.2
> **경로:** `댓글알림\main.py`

---

## 📖 사용 방법

### 기존과 동일하게 사용

1. 글 URL 추가
2. `▶ 모니터링 시작` 클릭
3. 30분마다 자동 체크

> 🚨 **새로 추가된 것:** 등록한 글이 비공개 조치되면 **자동으로 Slack 알림**이 옵니다.
> 별도 설정 없이 기존 모니터링에 포함됩니다.

### Slack 알림 예시

```
🚨 비공개 조치 감지
• 게시글: skyhealthinfo/224161198440
• 요청기관: 식품의약품안전처
• 요청 일자: 2026.03.12
• 내용: 이 게시물은 식품의약품안전처의 요청으로 비공개 조치 되었습니다.
```

### 알림 규칙

| 항목 | 동작 |
|------|------|
| **알림 횟수** | 모니터링 세션당 **1번만** (같은 글 반복 알림 없음) |
| **재감지** | 모니터링 중지 → 다시 시작하면 초기화되어 다시 감지 |
| **요청기관** | 식품의약품안전처, 방통위 등 어떤 기관이든 대응 |

## 🔧 구현 방법

### 체크 흐름

```
모바일 페이지 (m.blog.naver.com) 1회 요청
├─ commentCount 추출 → 댓글 증감 비교
├─ 비공개 패턴 감지 ("~의 요청에 따라 비공개 조치")
│   └─ 감지됨 → 데스크톱 PostView에서 요청기관 + 요청일자 상세 조회
│                → Slack 알림
│
└─ commentCount 없음 (None) 판단:
    ├─ 비공개 글이라서 → 정상 처리 (비공개 알림 발송)
    └─ 비공개 아닌데 실패 → Selenium으로 강제 댓글 확인
```

### 왜 2단계 (모바일 → 데스크톱)?

| 구분 | 모바일 페이지 | 데스크톱 PostView |
|------|-------------|-----------------|
| **패턴** | "~의 요청에 따라 비공개 조치" | "이 게시물은 ~의 요청으로 비공개 조치 되었습니다" |
| **요청기관** | ✅ 추출 가능 | ✅ 추출 가능 |
| **요청 일자** | ❌ 없음 | ✅ 있음 |
| **용도** | 비공개 여부 1차 판별 | 상세 정보 (일자) 조회 |

> 모바일 페이지는 **commentCount 조회와 동시에** 체크하므로 추가 요청 없음.
> 데스크톱은 비공개가 감지된 글만 추가 1회 요청.

### 댓글 누락 수정 (부수 버그 수정)

**문제:** 모바일 페이지에서 댓글 수 조회 실패 시 → 이전 값으로 대체 → "변동 없음" 판정 → 새 댓글 놓침

**원인:** 비공개 조치된 글은 `commentCount` 속성이 없어서 None 반환 → 이전 값(예: 5)으로 대체 → 증가 안 됨으로 판단

**수정:** 댓글 수 조회 실패 + 비공개 아닌 글 → Selenium으로 직접 댓글 확인

## ✅ 잘된 점

- **추가 요청 없이 감지** — 기존 모바일 페이지 요청에서 비공개 패턴도 함께 체크
- **모바일/데스크톱 패턴 모두 대응** — 모바일에서 1차 감지, 데스크톱에서 상세 정보
- **댓글 누락 버그 해결** — 댓글 수 조회 실패 시 Selenium fallback 추가
- **요청기관 유동 대응** — 정규식이 기관명을 동적으로 추출

---

## ⚠️ 아쉬운 점

- **비공개 외 다른 제재** (삭제, 접근 제한 등)는 아직 미감지
- **모바일 패턴 변경 리스크** — 네이버가 문구를 바꾸면 정규식 수정 필요
- **요청 일자를 위해 추가 요청 1회** — 데스크톱 PostView 조회 (비공개 글에 한해)
