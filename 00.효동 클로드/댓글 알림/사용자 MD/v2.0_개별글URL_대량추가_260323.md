| 항목 | 내용 |
|------|------|

## 🚀 사용법

### 1. 프로그램 실행
```
python main.py
```

### 2. 글 URL 등록
- **개별 추가:** 입력란에 URL 붙여넣고 `추가` 클릭
- **대량 추가:** `대량 추가` 클릭 → 팝업에 URL 한 줄씩 붙여넣기 → `추가`
- 중복 자동 제거, 잘못된 URL 무시
- URL은 `config.json`에 자동 저장 → **껐다 켜도 유지**

### 3. 모니터링 시작
- `▶ 모니터링 시작` 클릭
- **버튼 누른 시점**의 댓글 수를 기준점으로 저장 (수 초 소요)
- 이후 15분마다 댓글 수 비교 → 새 댓글 발견 시 Slack 알림

### 4. 중지/재시작
- `⏹ 모니터링 중지` → 타이머 정지
- 다시 `▶ 모니터링 시작` → **그 시점부터** 다시 추적 시작

## ⚡ 동작 원리 (2단계 최적화)

### 1단계: API로 댓글 수 확인 (빠름, ~수 초)
```
등록된 URL들 → blog_id별 그룹핑 → PostTitleListAsync API 호출
→ 각 글의 commentCount 가져옴 → 이전 기준점과 비교
```
- Selenium 안 씀, 순수 HTTP 요청
- 200개 글이어도 ~1분이면 완료

### 2단계: 변동 글만 Selenium 스크래핑 (필요할 때만)
```
댓글 수 증가한 글만 → headless Chrome으로 접속
→ mainFrame iframe 전환 → 댓글 목록 펼치기
→ 댓글 수집 → 이전 seen과 비교 → 새 댓글 Slack 알림
```
- 변동 없으면 Selenium **아예 안 씀**
- 10개마다 드라이버 자동 재시작 (안정성)

### 속도 비교

| 시나리오 | 소요 시간 |
|----------|-----------|
| 첫 실행 (기준점 저장) | ~8초 (API만) |
| 이후 (변동 없음) | ~2초 |
| 이후 (변동 3개 글) | ~20초 |

## 🔑 주요 클래스/함수

| 이름 | 역할 |
|------|------|
| `parse_post_url(url)` | URL에서 blog_id, log_no 추출 |
| `create_driver()` | headless Chrome 드라이버 생성 (안정성 옵션 포함) |
| `BlogMonitor` | 모니터링 엔진 |
| `._fetch_comment_counts(blog_id)` | API로 블로그 전체 commentCount 조회 |
| `.get_comments(blog_id, log_no)` | Selenium으로 특정 글 댓글 스크래핑 |
| `.check_all()` | 전체 체크 흐름 (API → 변동 감지 → 스크래핑 → 알림) |
| `App` | tkinter GUI (URL 관리, 시작/중지, 로그) |

### v1.0 → v2.0 핵심 변경 요약
1. **블로그 전체 → 개별 글 URL** 모니터링으로 변경
2. **2단계 최적화:** API commentCount로 변동 감지 → 변동 글만 Selenium
3. **첫 실행 속도:** 100분+ → 8초 (API만 사용)
4. **대량 추가 팝업:** 200개 URL 한번에 붙여넣기 가능
5. **탭 크래시 해결:** page_load_timeout, sleep 단축, 10개마다 드라이버 재시작
6. **Chrome 안정성 옵션:** `--disable-background-networking`, `--disable-renderer-backgrounding`
