# 📢 댓글 알림 프로그램 v2.1 — 모바일 API + 즉시 체크

> **날짜:** 2026-03-23
> **버전:** v2.1
> **파일:** `댓글알림\main.py`

---

## 📖 사용 방법

### 기본 사용 (30분 자동 체크)
1. 프로그램 실행
2. 글 URL 추가 (단건 또는 대량 추가)
3. `▶ 모니터링 시작` 클릭
4. → 기준점 저장 → 30분마다 자동 체크 → 새 댓글 Slack 알림

### ⚡ 즉시 체크 (새 기능!)
- `⚡ 즉시 체크` 버튼 클릭 → **바로 전체 글 댓글 확인**
- 모니터링 ON 상태: 대기 중이던 30분 타이머 취소 → 즉시 체크 → 다시 30분 타이머 시작
- 모니터링 OFF 상태: 1회만 체크하고 끝

### 체크 주기 변경
- `config.json`의 `"interval_minutes": 30` 값 수정
- 기본값: 30분

---

## 🔧 구현 방식

### 댓글 수 확인: 모바일 페이지 방식 (v2.1 변경)

```
글 1개당 → m.blog.naver.com/{blogId}/{logNo} 접속
→ HTML에서 commentCount="숫자" 추출
→ 이전 기준점과 비교 → 증가한 글만 Selenium 스크래핑
```

| 단계 | 방법 | 속도 |
|------|------|------|
| **1단계: 댓글 수 확인** | 모바일 페이지 HTTP 요청 | 글당 ~0.4초 |
| **2단계: 댓글 스크래핑** | Selenium (변동 글만) | 글당 ~5초 |

> **49개 글 기준:** 1단계 ~25초 + 2단계는 변동 글 수에 따라 다름

### 왜 모바일 페이지를 쓰나?

| 방식 | 문제 |
|------|------|
| PostTitleListAsync (구버전) | **일부 글이 API 목록에서 누락됨** → 댓글 변동 감지 불가 |
| cbox API | 네이버가 ticket 값 변경 → `"Wrong ticket"` 에러 |
| **모바일 페이지 (현재)** | **모든 글을 정확하게 조회 가능** ✅ |

---

## ✅ 잘된 점

1. **모든 글 감지 가능** — PostTitleListAsync에서 누락되던 글도 정확하게 댓글 수 확인
2. **즉시 체크** — 30분 기다리지 않고 바로 확인 가능
3. **Selenium 없이 댓글 수 확인** — 모바일 페이지는 순수 HTTP 요청으로 가벼움
4. **기존 구조 유지** — 변동 글만 Selenium 스크래핑하는 2단계 최적화는 그대로

---

## ⚠️ 아쉬운 점 / 주의사항

1. **속도** — PostTitleListAsync(2~3초)보다 느림 (49개 글 ~25초). 글 수가 많아지면 더 느려짐
2. **네이버 차단 가능성** — 30분마다 49회 HTTP 요청은 안전하지만, 글이 수백 개로 늘어나면 간격 조정 필요
3. **모바일 페이지 구조 변경 리스크** — 네이버가 `commentCount="숫자"` 형식을 바꾸면 파싱 실패
