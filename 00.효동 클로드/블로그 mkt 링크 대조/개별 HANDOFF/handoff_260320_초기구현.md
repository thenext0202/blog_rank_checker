---

## 🎯 작업 목표

블로그에 삽입된 MKT 상품 링크가 시트의 찐링크와 동일한 상품을 가리키는지 자동으로 대조

---

## 📋 수행 내용

### 1차 — 초기 구현 및 실행

| 항목 | 내용 |
|---|---|
| 파일 | `blog_link_extractor.py` (280줄) |
| 위치 | `프로그램 개발\블로그 mkt 링크 대조\` |
| 실행 결과 | 120개 블로그 전량 처리 성공 |

**구현한 기능:**
- 구글 시트 G열에서 블로그 링크 읽기
- Selenium headless로 블로그 방문 → MKT 링크 추출
- MKT 링크 리다이렉트 추적 → 최종 URL 확인
- 찐링크(F열)와 정규화 비교 → 일치/불일치 판정
- 결과를 H~K열에 자동 기입

### 2차 — 중복 수집 이슈 수정

| 문제 | 원인 | 해결 |
|---|---|---|
| 블로그에 MKT 링크 2개인데 1개만 수집 | `_collect_mkt()`에서 base URL 기준 중복 제거 | 중복 제거 로직 제거 → 동일 링크도 허용 |

**수정 전:**
```python
base = re.sub(r"\?.*", "", clean)
if base not in seen:
    seen.add(base)
    mkt_links.append(clean)
```

**수정 후:**
```python
mkt_links.append(clean)
```

**수정 후 실행 결과:**
- 120개 블로그 × MKT 링크 34개씩 수집 (3가지 추출 방식 중복 포함)
- H열(1번째), J열(2번째) 모두 "일치" 확인
- 시트 H~K열 정상 기입 완료
