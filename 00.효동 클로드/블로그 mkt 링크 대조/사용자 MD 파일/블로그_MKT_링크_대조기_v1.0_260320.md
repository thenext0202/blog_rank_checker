## 🔧 구현 방법 설명

### 전체 흐름

```
구글 시트 읽기 → 블로그 방문(Selenium) → MKT 링크 추출 → 리다이렉트 추적 → 찐링크와 비교 → 시트 기입
```

### 핵심 로직

| 단계 | 함수 | 설명 |
|---|---|---|
| 시트 연결 | `get_gsheet()` | 서비스 계정 인증 → gspread |
| 블로그 접속 | `create_driver()` | Selenium headless Chrome |
| 링크 추출 | `extract_mkt_links()` | 3가지 방식으로 MKT 링크 수집 |
| 리다이렉트 추적 | `resolve_mkt_link()` | HEAD/GET 요청으로 최종 URL 확인 |
| URL 비교 | `check_match()` | 정규화 후 전체 URL + 상품 ID 비교 |

### MKT 링크 추출 3가지 방식

1. **DOM `<a>` 태그 href** — 8개 CSS 셀렉터로 탐색
2. **`data-linkdata` 속성** — JSON 데이터에서 URL 추출
3. **페이지 소스 정규식** — JS 렌더링 후 소스에서 직접 추출

> 💡 중복 허용: 같은 링크가 2번 삽입되었으면 2개 모두 수집 (삽입 누락 체크 목적)

### URL 일치 판정

```
1차: 정규화(프로토콜·쿼리·www·m 제거) 후 전체 URL 비교
2차: products/{상품ID} 추출 후 ID만 비교
→ 둘 다 실패하면 "불일치"
```

## ✅ 잘된 점

- **완전 자동화**: 시트 읽기 → 블로그 방문 → 결과 기입까지 사람 개입 없음
- **3중 추출**: DOM, data-linkdata, 페이지 소스 3가지 방식으로 놓치지 않고 수집
- **2단계 비교**: URL 전체 비교 + 상품 ID 비교로 정확도 향상
- **봇 감지 우회**: webdriver 프로퍼티 숨김, UA 설정 등 적용
- **120개 블로그 전량 처리 성공**: 에러 없이 완주

## ⚠️ 아쉬운 점 / 문제점

| 문제 | 설명 |
|---|---|
| **34개 중복 수집** | 3가지 추출 방식이 같은 링크를 여러 번 잡음 → 실제 2개인데 34개로 보임 |
| **시트/탭 하드코딩** | `시트34`가 코드에 고정 → 다른 탭 쓰려면 코드 수정 필요 |
| **순차 처리** | 블로그 1개씩 순차 방문 → 120개에 약 10분 소요 |
| **ws.update() 경고** | gspread 인자 순서 변경 DeprecationWarning 발생 |
| **H/J열에만 기록** | MKT 링크가 3개 이상이면 3번째부터는 무시됨 |

## 🚀 다음 디벨롭 제안

### 우선순위 높음

| 개선 | 설명 |
|---|---|
| **중복 수집 정제** | 3가지 추출 방식 간 중복 제거 + "실제 삽입 위치 기준" 카운팅 |
| **ws.update() 경고 수정** | `ws.update(values=[[...]], range_name=f"H{row}:K{row}")` 형식으로 변경 |
| **시트/탭 설정 외부화** | 코드 상단 상수 or 명령줄 인자로 시트 ID/탭명 지정 |

### 우선순위 보통

| 개선 | 설명 |
|---|---|
| **병렬 처리** | Selenium 드라이버 2~4개 병렬로 속도 3~4배 향상 (발행검수처럼) |
| **GUI 추가** | tkinter GUI로 시트 선택, 진행률 표시, 결과 요약 |
| **이미 처리된 행 스킵** | H열에 값이 있으면 건너뛰기 → 재실행 시 시간 절약 |

## 💡 다른 프로그램 만들 때 반영하면 좋은 점

| 교훈 | 설명 |
|---|---|
| **중복 제거 전략을 먼저 설계** | "같은 URL이 여러 곳에서 발견될 때 어떻게 처리할지"를 추출 로직 전에 결정해야 함 |
| **추출 방식 분리 → 병합** | DOM/속성/소스 각각 추출 후, 최종 단계에서 "위치 기반 dedup"을 하는 게 좋음 |
| **gspread API 변경 주시** | `ws.update()` 인자 순서 변경처럼 라이브러리 업데이트 시 breaking change 체크 |
| **Selenium iframe 전환** | 네이버 블로그는 `mainFrame` iframe 안에 콘텐츠가 있으므로 항상 `switch_to.frame()` 필요 |
| **MKT 링크 리다이렉트 체인** | HEAD 요청 실패 시 GET 폴백 패턴이 유용 — 다른 링크 추적에도 적용 가능 |
