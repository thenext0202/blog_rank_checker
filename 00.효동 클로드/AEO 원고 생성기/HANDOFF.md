# AEO 원고 생성기 — HANDOFF

> 마지막 갱신: 2026-04-10<br>
> 현재 버전: v1.4.2<br>
> 배포: Railway (web 프로세스 안에서 워커를 백그라운드 데몬 스레드로 실행)

<br>

## 📦 제품 개요

AEO(Answer Engine Optimization) 친화적인 블로그·SNS 원고를 카테고리/구조 지침 + 제품 정보에 맞춰 자동으로 생성하고, 자체 채점표로 점수를 매기고, 결과를 구글 시트에 기입하는 도구. 워드 파일 저장과 제품별 링크 자동 삽입까지 포함.

- **입력**: 작성일 + 채널 + 제품명 + 카테고리 + 구조(A/B/C…) + 키워드
- **출력**: 원고 본문 + 제목 + AEO 점수 + 등급 + 영역별 점수 + 파일명 + 워드 다운로드 링크 + (필요 시) 포스트 분할 텍스트
- **운영**: Railway에 배포된 Flask 웹 + 같은 프로세스 안의 백그라운드 워커 스레드

<br>

## 🔄 운영 흐름

### A. 즉시 생성 (웹)
1. 사용자가 `https://{railway-url}/` 접속
2. 제품명·카테고리·구조·채널·키워드 + (선택) 모델 선택
3. POST `/generate` → app.py가 제품 링크 조회 → generator.generate_aeo() → write_aeo_result()
4. 시트 새 행에 A~N + P + Q 기입 (O 모델은 사용자 칸이라 안 건드림)
5. (선택) "워드 저장" 버튼 → POST `/download_docx` → 즉시 .docx 다운로드

### B. 대량 자동 생성 (워커)
1. 사용자가 시트 빈 행에 A~F 입력 (+ 필요 시 O 체크)
2. 워커가 60초마다 폴링 → A~F 채워졌고 J 비어 있는 행 탐색
3. 발견 시 J열에 `⏳ 생성 중...` 마커 박기 (진행 표시 + 중복 픽업 방지)
4. 제품 링크 조회 → generator.generate_aeo() → update_aeo_row(row=N)
5. 해당 행 I~N + P + Q만 업데이트 (A~H, O 보존)
6. 일시 실패 시 J열 비움 (다음 폴링 재시도) / 3회 영구 실패 시 ❌ 마커
7. (선택) 사용자가 Q열 "워드 다운로드" 클릭 → GET `/download_row/<row>` → 즉시 .docx

## 🔗 제품별 링크 자동 주입

원고 안에 자사몰/네이버 스마트스토어 URL이 들어가야 할 경우 자동으로 정확한 URL이 박힘.

### 일반 제품 (블러드싸이클 등)
- 시트 T~V열에서 그 제품 행의 (자사몰, 네이버) 한 쌍을 가져와 프롬프트에 주입
- "원고에 링크가 자연스러우면 두 URL 모두 노출, 어색하면 안 넣음" 지시

### ALL
- 시트 T~V 매핑 표 9개 전체를 프롬프트에 주입
- "본문에 등장하는 각 제품마다 그 제품의 개별 URL 사용" 지시
- 본문 마지막 "브랜드 전체" 줄에는 ALL의 메인 URL (자사몰 메인 + 네이버 브랜드관) 사용

| 채널 | 코드 |

글루코컷 `gc` / 멜라토닌 `mt` / 블러드싸이클 `bc` / 상어연골환 `sc` / 판토오틴 `pt` / 퓨어톤 부스트 `gt` / 활성엽산 `pf` / 헬리컷 `hc` / ALL `all`

## 🚦 운영 주의사항

- **워커는 반드시 단일 인스턴스로** — 여러 개 띄우면 같은 행 중복 처리 가능
- **시트 기입은 항상 명시적 범위** (`A:N`, `I:N`, 단일 셀) — `append_rows` 사용 금지
- **사전 설정된 드롭다운/색상 chip 보존** — 워커도 update 시 범위 외 셀 안 건드림. 특히 B열 채널 드롭다운은 코드가 절대 안 건드림 (효진님 수동 관리)
- **API 비용**: Sonnet 기본, Opus는 O열 체크된 행만
- **시트 T~V 매핑 fresh read** — 효진님이 자사몰 URL 수정하면 다음 원고부터 즉시 반영. 향후 URL 확정 시 캐싱 검토
- **B열 드롭다운에 "공식 SNS"가 남아 있으면 코드와 불일치** — v1.4.2부터 코드는 "공식 X" / "공식 스레드"만 인식. 효진님이 시트 드롭다운 갱신 필요

## 🚧 알려진 제약

1. 워커 단일 인스턴스 가정 (락 메커니즘 없음)
2. ⏳ 마커가 박힌 후 워커가 죽으면 마커가 영영 남음 (효진님이 수동으로 J열 비우기 필요)
3. 시트 onEdit 트리거 없음 — 폴링 주기만큼 대기 필요 (평균 30초)
4. ALL 원고에서 본문 풀네임 ↔ 시트 짧은 이름 매칭은 Claude 판단에 의존 (100% 보장 아님)
5. 새 제품 추가 시 코드(`PRODUCTS`/`PRODUCT_CODES`) + 시트(T~V 매핑 + B열 드롭다운) 두 곳 수정 필요
6. /download_row 인증 없음 — 행 번호만 알면 누구나 다운로드 가능
