# handoff 260410 — 제품명 도입 + 파일명 + 제품별 링크 (v1.4)

> 작업일: 2026-04-10<br>
> 버전: v1.3.1 → **v1.4**<br>
> 작업자: 효동(Claude Opus 4.6) + 효진

<br>

## 🎯 작업 목적

원고를 **어떤 제품 캠페인을 위해 만들었는지** 시트와 코드 양쪽에서 추적하고, 그 정보를 활용해서 (1) 워드 파일명을 자동 생성하고, (2) 본문에 들어가는 자사몰·네이버 스마트스토어 URL을 제품별로 자동 매칭하도록 만든다.

배경: 기존엔 제품 구분 없이 카테고리·구조·키워드만으로 원고가 만들어졌고, 본문에 링크가 들어갈 때도 도메인 수준(`meditial.co.kr`)만 적혀서 효진님이 매번 정확한 URL로 수동 교체해야 했다.

<br>

## 🔁 Before / After

### 시트 구조 (A~P)

**Before (v1.3.1)**

```
A 발행일 | B 채널 | C 카테고리 | D 구조 | E 키워드 | F 담당자 | G 발행 링크
H 제목 | I 본문 | J 문단구분 | K 점수 | L 등급 | M 영역별 | N 모델
```

**After (v1.4)**

```
A 작성일 | B 채널 | [C 제품명] | D 카테고리 | E 구조 | F 키워드
G 담당자 | H 발행 링크 | I 제목 | J 본문 | K 문단구분 | L 점수 | M 등급
N 영역별 | O 모델 | [P 파일명]
```

- C에 **제품명** 신규 삽입 → 기존 C~N이 D~O로 한 칸씩 우측 이동
- P에 **파일명** 자동 생성 (블로그/지식인/티스토리 채널만)
- A 헤더 명칭 통일 (발행일 → 작성일)

<br>

### 워커 픽업 조건

| | Before | After |
| --- | --- | --- |
| 필수 입력 칸 | A~E (5칸) | A~F (6칸, 제품명 포함) |
| 본문 위치 | I열 (idx 8) | J열 (idx 9) |
| 모델 위치 | N열 (idx 13) | O열 (idx 14) |

<br>

### 본문 안의 링크

**Before**: 도메인만 텍스트로 노출. 제품별 구분 없음.
```
공식 자사몰 meditial.co.kr에서는...
네이버 스마트스토어 brand.naver.com/meditial...
```

**After**: 시트 T~V열의 매핑 표를 매번 fresh로 읽어서 정확한 URL 주입.
```
메디셜 블러드싸이클: https://meditial.co.kr/product/detail.html?product_no=333&...
네이버 스마트스토어 메디셜 브랜드관: https://mkt.shopping.naver.com/link/...
```

특히 제품명이 **ALL**이면 9개 제품 매핑 표 전체를 프롬프트에 주입해서, 본문에 등장하는 각 제품마다 개별 URL이 들어가도록 처리.

### 2. 파일명 빌더 — P열용 + 워드 다운로드용 두 가지

```python
# P열용: 블로그/지식인/티스토리만 — 그 외 채널은 빈 문자열
def build_filename(channel, date_str, structure, category, product):
    ch_code = CHANNEL_CODES.get(channel.strip())
    if not ch_code:
        return ""
    return _filename_core(ch_code, date_str, structure, category, product)

# 워드 다운로드용: 모든 채널 fallback (한국어 채널명 공백 제거)
def build_filename_for_docx(channel, date_str, structure, category, product):
    raw = channel.strip()
    ch_code = CHANNEL_CODES.get(raw) or raw.replace(" ", "")
    return _filename_core(ch_code, date_str, structure, category, product)
```

포맷: `AEO_{채널코드}_{YYMMDD}{구조}{카테고리숫자}_{제품코드}`

| 입력 | 결과 |
| --- | --- |
| 네이버 블로그 / 260407 / A / 1.1 브랜드 소개글 / ALL | `AEO_blog_260407A11_all` |
| 공식 SNS / 260410 / B / 2.1.1 제품 비교 / 글루코컷 | `AEO_공식SNS_260410B211_gc` |

### 3. 제품별 링크 조회 — load_product_links()

```python
def load_product_links(spreadsheet):
    """T~V열 매핑 → {제품명: (자사몰URL, 네이버URL)}"""
    ws = spreadsheet.worksheet(TAB_NAME)
    rows = ws.get("T2:V20")
    return {r[0].strip(): (r[1].strip(), r[2].strip())
            for r in rows if r and r[0].strip()}
```

**캐싱 안 함** — 효진님이 자사몰 URL을 나중에 수정할 거라서 매번 fresh로 읽음. 원고 1건당 시트 read 1회 추가 (분당 60회 무료 한도 안에서 부담 없음).

### 4. prompt_builder — ALL/단일 제품 분기

```python
def build_aeo_prompt(..., own_url="", naver_url="", product_links_map=None):
    if product_links_map:
        # ALL: 9개 제품 표 + 메인 URL 분리
        product_table = "\n".join(f"- {n}: 자사몰={u[0]} | 네이버={u[1]}" 
                                  for n, u in product_links_map.items() if n != "ALL")
        link_section = f"""=== 링크 사용 규칙 ===
이 원고는 메디셜 브랜드의 여러 제품이 본문에 등장하는 글입니다.
본문에서 특정 제품을 언급할 때는 반드시 아래 표에서 해당 제품의 URL을 사용하세요.

{product_table}

브랜드 전체를 가리킬 때만 사용:
- 자사몰 메인: {own_url}
- 네이버 브랜드관: {naver_url}
"""
    elif own_url and naver_url:
        # 단일 제품: URL 한 쌍만
        link_section = f"...자사몰: {own_url}\n네이버: {naver_url}..."
```

조건부 지시: "원고 종류상 링크가 자연스럽지 않다면 아예 넣지 마세요. 링크를 넣는다면 반드시 두 URL을 모두 노출하세요."
