# handoff 260410 — 워드(.docx) 저장 (v1.4.1)

> 작업일: 2026-04-10<br>
> 버전: v1.4 → **v1.4.1**<br>
> 작업자: 효동(Claude Opus 4.6) + 효진

<br>

## 🎯 작업 목적

원고 결과를 효진님이 워드 파일로 받아 사용할 수 있게 한다. **두 경로 모두에서** 가능해야 한다.

1. **웹 UI** — 원고 생성 직후 우측 상단의 "워드 저장" 버튼 클릭
2. **시트** — 워커가 처리한 행의 Q열에 자동으로 박힌 "워드 다운로드" 링크 클릭

별도 저장소(Google Drive 등) 없이, 시트의 본문(J열)을 단일 진실의 원천으로 두고 다운로드 시점에 즉석 변환.

<br>

## 🔁 Before / After

### Before

- 결과 박스에서 **복사** 버튼만 있음 → 효진님이 워드 직접 만들어서 붙여 넣어야 함
- 시트 행은 워드 출력 경로 없음

### After

- 웹 결과 박스에 **워드 저장** 버튼 추가 → 클릭 즉시 .docx 다운로드
- 시트 P열 옆에 **Q열 "원고 다운로드"** 헤더 추가, 워커가 행 처리 후 `=HYPERLINK(...)` 수식 자동 기입
- 두 경로 모두 **동일한 docx 빌더 + 동일한 파일명 규칙** 사용

<br>

## 🛠️ 구현 변경

### 1. docx_writer.py 신설

`python-docx` 기반의 단순 빌더. 메모리에서 .docx 바이트 생성.

```python
def build_docx_bytes(text):
    doc = Document()
    style = doc.styles['Normal']
    style.font.name = '맑은 고딕'
    style.font.size = Pt(11)

    blocks = re.split(r'\n\s*\n', text.strip())
    if blocks:
        doc.add_heading(blocks[0].strip(), level=1)  # 첫 단락 = 제목
        for block in blocks[1:]:
            if block.strip():
                doc.add_paragraph(block.strip())  # 빈 줄 기준 단락

    buf = BytesIO()
    doc.save(buf)
    return buf.getvalue()
```

기존 `manuscript_generator/docx_formatter.py`는 ㄴ 서식 지시어 파싱 등 무거운 로직이라 재사용 안 함. AEO 원고는 plain text라 단순 빌더로 충분.

<br>

### 2. /download_docx 엔드포인트 (웹 버튼용)

```python
@app.route("/download_docx", methods=["POST"])
def download_docx():
    data = request.get_json(force=True)
    manuscript = data.get("manuscript", "")
    filename = data.get("filename", "AEO_원고")
    docx_bytes = build_docx_bytes(manuscript)
    return _docx_response(docx_bytes, filename)
```

웹 폼에서 `buildDocxFilename()` JS로 파일명을 클라이언트에서 미리 만들어 전송.

<br>

### 3. /download_row/<int> 엔드포인트 (시트 링크용)

```python
@app.route("/download_row/<int:row>", methods=["GET"])
def download_row(row):
    spreadsheet = connect_sheet(SHEET_ID)
    ws = spreadsheet.worksheet(TAB_NAME)
    rng = ws.get(f"A{row}:P{row}")
    r = rng[0] + [""] * 16
    
    manuscript = r[9]    # J열 본문
    p_value = r[15].strip()  # P열 파일명
    
    filename = p_value or build_filename_for_docx(
        channel=r[1], date_str=r[0], structure=r[4],
        category=r[3], product=r[2]
    )
    
    docx_bytes = build_docx_bytes(manuscript)
    return _docx_response(docx_bytes, filename)
```

P열에 파일명이 없으면 (워드프레스/카페/공식 X 등) `build_filename_for_docx`로 한국어 채널명 fallback.

<br>

### 4. 한글 파일명 응답 (RFC 5987)

```python
def _docx_response(docx_bytes, filename):
    if not filename.lower().endswith(".docx"):
        filename += ".docx"
    encoded = quote(filename, safe="")
    return Response(
        docx_bytes,
        mimetype="application/vnd.openxmlformats-officedocument.wordprocessingml.document",
        headers={"Content-Disposition": f"attachment; filename*=UTF-8''{encoded}"},
    )
```

`AEO_공식SNS_260407A11_all.docx` 같은 한글 파일명이 브라우저에서 깨지지 않도록.

<br>

### 5. 시트 Q열 자동 하이퍼링크

```python
def _build_download_hyperlink(row):
    base = os.environ.get("APP_BASE_URL", "").strip().rstrip("/")
    if not base:
        return ""
    return f'=HYPERLINK("{base}/download_row/{row}", "워드 다운로드")'
```

`write_aeo_result`(웹 경로)와 `update_aeo_row`(워커 경로) 둘 다 본문 기입 직후 Q열에 수식 박음. 환경변수 `APP_BASE_URL`이 비어 있으면 안 박음 (graceful fallback).

<br>

### 6. 웹 폼 워드 저장 버튼 (templates/index.html)

```html
<button class="docx-btn" onclick="downloadDocx()">워드 저장</button>
```

```javascript
async function downloadDocx() {
  const manuscript = document.getElementById('manuscript-text').textContent;
  const filename = buildDocxFilename();  // 폼 값으로 클라이언트 측 생성
  const res = await fetch('/download_docx', {
    method: 'POST', 
    headers: {'Content-Type':'application/json'},
    body: JSON.stringify({manuscript, filename})
  });
  const blob = await res.blob();
  // <a download> 트릭으로 즉시 다운로드
}
```
