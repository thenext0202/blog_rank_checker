## 🎯 작업 목적

운영하면서 효진님이 불편했던 두 가지를 한 번에 해결.

1. **공식 SNS 채널 분리** — X(트위터)와 스레드(Threads)는 글자 수 제한이 다른데, 기존엔 둘을 "공식 SNS" 하나로 묶어서 모두 150자로 분할되고 있었음
2. **워커 진행 상황 깜깜이** — 시트에 입력하고 나면 30~60초 동안 워커가 도는지 죽었는지 알 수가 없어서 효진님이 답답해함

## 🔁 Before / After

### 1. 채널 분리

**Before**

```python
CHANNEL_INSTRUCTIONS = {
    ...
    "공식 SNS": "공식 SNS(X, 쓰레드 등)에 게시할 글입니다...",
    ...
}
SPLIT_CHANNELS = {"공식 SNS"}
```

- "공식 SNS" 하나만 존재 → 분할 시 **무조건 150자**
- X와 스레드를 구분 못 함

**After**

```python
CHANNEL_INSTRUCTIONS = {
    ...
    "공식 X": "공식 X(트위터)에 게시할 글입니다...",
    "공식 스레드": "공식 스레드(Threads)에 게시할 글입니다...",
    ...
}
SPLIT_CHANNEL_LIMITS = {
    "공식 X": 150,
    "공식 스레드": 500,
}
```

- 두 채널로 분리, 채널별 max_chars dict로 관리
- generator가 분할 시 채널별 글자수로 호출

<br>

### 2. 워커 진행 마커

**Before**

| 시점 | J열에 보이는 것 |
| --- | --- |
| 입력 직후 | (빈 칸) |
| 워커 처리 중 (30~60초) | (빈 칸 — 깜깜이) |
| 완료 | (본문) |
| 일시 실패 | (빈 칸 — 다음 폴링에 또 처리) |
| 영구 실패 | ❌ 마커 |

**After**

| 시점 | J열에 보이는 것 |
| --- | --- |
| 입력 직후 | (빈 칸) |
| 워커 픽업 직후 | `⏳ 생성 중... (14:23:15 시작)` |
| 처리 중 (30~60초) | (위 마커 그대로) |
| 완료 | (본문이 마커 덮어씀) |
| 일시 실패 | (빈 칸으로 자동 복구 → 다음 폴링 재시도) |
| 영구 실패 | ❌ 마커 |

마커가 박히면 효진님이 시트만 봐도 "지금 처리 중이구나" 알 수 있고, 동시에 같은 워커의 다음 폴링 사이클에서 중복 픽업 방지 효과도 있음.

<br>

## 🛠️ 구현 변경

### 1. SPLIT_CHANNELS set → SPLIT_CHANNEL_LIMITS dict

```python
# prompt_builder.py
SPLIT_CHANNEL_LIMITS = {
    "공식 X": 150,
    "공식 스레드": 500,
}

def needs_post_split(channel):
    return channel in SPLIT_CHANNEL_LIMITS

def get_split_max_chars(channel):
    return SPLIT_CHANNEL_LIMITS.get(channel, 0)
```

```python
# generator.py
if needs_post_split(channel):
    max_chars = get_split_max_chars(channel)
    posts = split_into_posts(manuscript, max_chars=max_chars)
```

`post_processor.split_into_posts`는 이미 `max_chars` 인자를 받는 구조라 그쪽은 변경 없음.

<br>

### 2. 워커 진행 마커 함수 (worker.py)

```python
def _mark_in_progress(row):
    """행 픽업 직후 J열에 ⏳ 마커"""
    spreadsheet = connect_sheet(SHEET_ID)
    ws = spreadsheet.worksheet(TAB_NAME)
    start_time = datetime.now(KST).strftime("%H:%M:%S")
    marker = f"⏳ 생성 중... ({start_time} 시작)"
    ws.update(values=[[marker]], range_name=f"J{row}",
              value_input_option="USER_ENTERED")

def _clear_in_progress(row):
    """일시 실패 시 J열 비움 → 다음 폴링 재시도 가능"""
    spreadsheet = connect_sheet(SHEET_ID)
    ws = spreadsheet.worksheet(TAB_NAME)
    ws.update(values=[[""]], range_name=f"J{row}",
              value_input_option="USER_ENTERED")
```

<br>

### 3. process_row 흐름 변경

```python
def process_row(api_key, item):
    row = item["row"]
    _mark_in_progress(row)  # ← 신규: 픽업 직후 마커
    
    try:
        result = generate_aeo(...)
    except Exception:
        _record_failure(row, ...)  # 한도 미만이면 _clear_in_progress 호출
        return False
    
    update_aeo_row(...)  # 성공 시 본문이 마커를 덮어씀
```

```python
def _record_failure(row, error_msg):
    count = _failure_counts.get(row, 0) + 1
    _failure_counts[row] = count
    if count >= FAILURE_LIMIT:
        _mark_permanent_failure(row, error_msg)  # ❌ 마커
    else:
        _clear_in_progress(row)  # ← 신규: 빈 칸으로 복구
```

## 🛡️ 주의사항

### 시트 B열 채널 드롭다운

코드는 `공식 SNS` 옵션을 더 이상 인식 안 함. 그러나 시트 B열은 효진님이 사전 설정한 색상 드롭다운이라 코드가 안 건드림. 효진님이 직접 시트에서 다음 작업 필요:

1. B열 드롭다운에서 "공식 SNS" 옵션 제거
2. "공식 X", "공식 스레드" 두 옵션 추가
3. 기존 행에 "공식 SNS"로 입력된 게 있으면 둘 중 하나로 변경

### ⏳ 마커가 5분 이상 그대로 있을 때

워커가 generate_aeo 도중에 진짜 죽으면 (Railway 컨테이너 재시작 등) ⏳가 영영 남음. 이 경우 효진님이 J열을 직접 비우면 다음 폴링에서 재시도.
