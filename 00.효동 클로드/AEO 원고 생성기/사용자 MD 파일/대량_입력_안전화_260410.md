# 대량 입력해도 안전 — 워커 안정화 v1.3.1

> 2026-04-10 / AEO 원고 생성기 v1.3.1<br>
> 한 번에 100건 넣어도 OAuth 만료/무한 재시도 걱정 없게 만들었어요.

<br>

## 📌 한 줄 요약

워커가 매 행마다 구글 시트에 새로 인증하고, 같은 행이 3번 실패하면 자동으로 ❌ 마킹해서 스킵해요. 이제 한 번에 수백 건 넣어 둬도 안심.

<br>

## ⚠️ 이전 버전(v1.3)의 두 가지 함정

### 1. 구글 인증 1시간 만료
- 구글 OAuth 토큰은 발급 후 **1시간 지나면 자동 만료**
- 이전 워커는 한 루프 시작할 때 **딱 한 번만 인증** → 100건이 1.5시간 걸리면 후반부터 시트 기입이 401 에러로 다 실패
- 앞쪽 50건은 OK, 뒤쪽 50건은 본문이 비어 있는 채로 남음

### 2. 영구 실패 무한 재시도
- 카테고리 오타 같이 영원히 실패하는 행이 있으면 → 본문 비어 있으니 다음 폴링에서 또 시도 → 또 실패 → 무한 반복
- API 비용만 계속 빠져나감

> 참고로 Railway 300초 timeout은 web 프로세스(웹 페이지)에만 걸리는 거고, 워커는 별개라서 무관해요.

<br>

## 🛡️ v1.3.1에서 한 일

### 1. 매 행마다 새로 인증
워커가 한 행 처리할 때마다 구글 시트 클라이언트를 새로 만들어요. **OAuth 토큰이 항상 신선** → 1시간 만료 무관.

> 추가 비용: 1행당 약 0.5~1초. 100건이면 약 1.5분 추가. 전체 처리 시간(1.5시간)의 1.6%라 거의 안 느껴짐.

### 2. 3회 실패 시 자동 ❌ 마킹
같은 행이 연속 3번 실패하면 I열(본문)에 이런 식으로 마킹해요.

> ❌ 자동 처리 실패 (3회 시도): 카테고리 '1.5 비교글' 없음

I열에 뭐가 들어있으면 워커가 "처리 완료된 행"으로 보고 다음부터 안 건드려요. **무한 재시도 차단.**

<br>

## 🔧 실패한 행 복구하는 법

ㅠ 카테고리 잘못 입력해서 ❌ 마킹된 경우:

1. C열 카테고리 수정
2. **I열에 적힌 ❌ 텍스트를 지움**
3. 다음 폴링(60초 이내)에 워커가 다시 처리해요

> 워커를 재시작하면 메모리상 실패 카운트는 리셋되지만, 시트의 ❌ 마킹은 그대로라 자동 스킵됨.

<br>

## ⚙️ 환경변수로 조절 가능

| 변수 | 기본값 | 의미 |
| --- | --- | --- |
| `WORKER_POLL_INTERVAL` | 60 | 시트 폴링 주기(초) |
| `WORKER_FAILURE_LIMIT` | 3 | 영구 실패로 간주할 연속 실패 횟수 |

> 예: `WORKER_FAILURE_LIMIT=5`로 두면 5번 실패까지 재시도 후 마킹.

<br>

## 👍 잘된 점

1. **수백 건 안전 처리** — OAuth 만료 걱정 없음
2. **자동 격리** — 문제 행은 알아서 스킵, 정상 행은 계속 처리
3. **사용자 복구가 직관적** — 시트의 ❌ 텍스트만 지우면 재처리
4. **추가 비용 거의 0** — 인증 추가 호출은 1초도 안 됨

<br>

## 🤔 아쉬운 점 / 한계

1. **여전히 단일 인스턴스만 권장** — 락 메커니즘은 안 만듦. 워커를 2개 띄우면 같은 행 동시 처리 가능
2. **실패 알림 없음** — ❌ 마킹된 게 있어도 사용자가 시트를 봐야 알 수 있음
3. **부분 실패 추적 안 됨** — 원고는 생성됐는데 시트 기입에서 실패한 경우, 원고 데이터는 메모리에서 그냥 사라짐
