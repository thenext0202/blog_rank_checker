# 시트만 채우면 원고가 알아서 뽑히는 자동 워커

> 2026-04-10 / AEO 원고 생성기 v1.3<br>
> 웹과 별개로 돌아가는 백그라운드 일꾼이 추가됐어요.

<br>

## 📌 한 줄 요약

구글 시트 **AEO 발행리스트** 탭에 `발행일 / 채널 / 카테고리 / 구조 / 키워드` 다섯 칸만 채워 두면, 워커가 알아서 그 행을 잡아 원고를 뽑아서 H~M열(제목·본문·점수)에 채워 넣습니다.

<br>

## 🎯 왜 만들었나

| 기존 방식 | 새 방식 |
| --- | --- |
| 웹 페이지 열기 → 카테고리·구조·키워드 입력 → 생성 버튼 → 결과 대기 → 다른 키워드 또 입력 | 시트에 키워드 여러 개 한꺼번에 적어 두기 → 자리 비워 둬도 알아서 채워짐 |
| 한 번에 1건씩 | 대량 입력 가능 (워커가 순차 처리) |

<br>

## ✏️ 사용 방법

### 1. 시트에 입력

`AEO 발행리스트` 탭에서 **빈 행**에 다음 다섯 칸만 채워 넣으세요.

| 열 | 항목 | 예시 |
| --- | --- | --- |
| A | 발행일 | 2026-04-15 |
| B | 채널 | 네이버 블로그 |
| C | 카테고리 | 1.1 브랜드 소개글 |
| D | 구조 | A |
| E | 키워드 | 메디셜 세럼 |

> F(담당자), G(발행 링크)는 자유. 워커가 안 건드립니다.

<br>

### 2. 모델 선택 (N열)

`N열 = 모델` 칸을 새로 만들어 두세요.

- **빈 칸** → Sonnet으로 생성 (기본)
- **체크 표시 (TRUE/체크박스/O)** → Opus로 생성

> 한 행만 Opus로 돌리고 싶으면 그 행의 N열만 체크하면 됩니다.

<br>

### 3. 워커가 알아서 처리

- 워커는 60초마다 시트를 둘러보며 **A~E가 다 채워졌는데 본문(I)이 비어 있는 행**을 찾습니다.
- 찾으면 원고 생성 → 채점 → 포스트 분할까지 다 돌리고, **H~M열**에 결과를 채워 넣습니다.
- 처리 끝난 행은 본문이 채워졌으니 다음 폴링에서 다시 잡지 않습니다.

<br>

## 🛡️ 안전 장치

- **A~G는 절대 안 건드림** — 사용자가 입력한 발행일·채널·카테고리·구조·키워드·담당자·발행링크 그대로 보존
- **N열도 안 건드림** — 모델 체크는 사용자 의지대로
- **시트 기존 드롭다운/색상 chip 그대로 보존** — 워커도 H~M만 정확히 범위 지정해서 update
- **실패해도 다음 행 처리 계속** — 한 행 실패한다고 워커가 멈추지 않음

<br>

## 🔧 워커 실행 방법

### 로컬 (테스트용)
```bash
cd aeo_writer
python worker.py
```

콘솔에 `[worker] 행 5 시작 — 1.1 브랜드 소개글 / A / 메디셜 세럼 (Sonnet)` 같은 로그가 찍혀요.

### Railway (운영)
`Procfile`에 `worker:` 항목이 추가되어 있어, Railway 대시보드에서 worker 프로세스를 활성화하면 백그라운드로 계속 돕니다.

### 폴링 주기 조절
환경변수 `WORKER_POLL_INTERVAL`(초 단위, 기본 60)로 조정 가능.

<br>

## 👍 잘된 점

1. **클릭 0번** — 시트만 채우면 끝. 웹 안 켜도 됨.
2. **모델 선택이 직관적** — 체크박스 하나로 Sonnet/Opus 토글
3. **웹과 충돌 없음** — 웹은 즉시 생성용, 워커는 대량 백그라운드용. 둘 다 같은 시트에 안전하게 기입
4. **재시작에 강함** — 워커 죽었다 살아나도, 본문 빈 행만 다시 찾아 처리하면 되니 상태 관리 불필요
5. **코드 재사용** — `generator.py` 모듈로 핵심 로직 분리해서 웹과 워커가 같은 함수를 호출

<br>

## 🤔 아쉬운 점 / 한계

1. **단일 인스턴스만 권장** — 워커를 2개 이상 띄우면 같은 행을 동시에 처리할 수 있음. (현재는 마커/락 없음)
2. **실패 시 재시도 없음** — 한 행이 API 오류로 실패하면 본문이 비어 있는 채로 남아, 다음 폴링에서 또 시도. 무한 루프 가능 (영구 실패하는 케이스)
3. **진행 상황 알림 없음** — 시트에서 "지금 처리 중"인지 알 수 없음. 콘솔 로그만 확인 가능
4. **폴링 60초** — 시트 입력 후 즉시 시작 안 됨. 평균 30초 대기
