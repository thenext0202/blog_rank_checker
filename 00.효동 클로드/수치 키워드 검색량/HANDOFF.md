### 시트 탭 구조

**수치키워드_설정:**
| A: 키워드 | B: 시작 | C: 끝 | D: 간격 | E: 시작2 | F: 끝2 | G: 간격2 |
|-----------|---------|-------|---------|----------|-------|---------|
| 혈압 | 40 | 300 | 1 | | | |
| 혈압 | 80 | 200 | 10 | 40 | 130 | 10 |

- E~G 비어있으면 → 단일 숫자: "혈압 120"
- E~G 채워져있으면 → 두 숫자 조합: "혈압 140 90"

**수치키워드_결과:**
| A: 기본 키워드 | B: 수치 키워드 | C: 검색량 |

## ✅ 완료된 작업

### 1. 기본 기능 — 완료 (v1.0, 2026-03-27)
- 구글 시트 설정 읽기 → 숫자 조합 생성 → API 배치 조회 → 결과 기록
- 5개씩 배치, 0.5초 간격
- 검색량 > 0인 것만 기록, 키워드 그룹별 내림차순 정렬

### 2. 소수점 지원 — 완료 (v1.0)
- `Decimal` 사용으로 부동소수점 오차 방지
- 당화혈색소 3.0~15.0 (0.1 단위) 등

### 3. 두 숫자 조합 — 완료 (v1.0)
- E~G열로 두 번째 숫자 범위 지정
- "혈압 140 90", "혈압 150 100" 등 수축기/이완기 조합

### 4. 중복 조회 방지 — 완료 (v1.0)
- 결과 탭에서 기존 데이터 읽어 `done_keys` set 구성

- 기존 결과 + 신규 결과 합쳐서 시트에 기록

### API 호출
- `call_keyword_api()`: 네이버 검색광고 API HMAC-SHA256 인증
- `hintKeywords`에 공백 제거한 키워드 전달
