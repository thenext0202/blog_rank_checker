# 03-1. 페르소나 생성 (Step 1)

> main.py: `_on_generate()`, `build_persona_prompt()`, `_show_persona_dialog()`

---

## 1. 기능 요약

Claude API에 페르소나 프롬프트를 전송하여 **3개 후보 페르소나**를 생성하고, 사용자가 선택/편집한다.

---

## 2. 처리 흐름

```
① _on_generate() — GUI 입력값 수집
   ├── 제품, 유형, 스타일, 키워드, 설정값 등
   └── 유효성 검증 (API 키, 시트 연결)
    ↓
② build_persona_prompt() — 페르소나 프롬프트 구성
   ├── 일반: PERSONA_PROMPT_TEMPLATE
   └── 제3자 관찰형: PERSONA_PROMPT_3RD (별도 템플릿)
   포함 정보: 제품, 키워드, 유형, 스타일
    ↓
③ call_claude_api() — API 호출
   ├── 모델: claude-sonnet-4-20250514
   ├── max_tokens: 2048
   └── 스레드 실행 (UI 비차단)
    ↓
④ _show_persona_dialog() — 선택 다이얼로그
   ├── 3개 라디오버튼 카드 ([페르소나 A/B/C])
   ├── 텍스트 편집 영역 (선택 후 수정 가능)
   └── 버튼: "선택 완료" / "페르소나 없이 생성" / "취소"
    ↓
⑤ → Step 2 (제목 생성)으로 전달
```

---

## 3. 페르소나 출력 형식

```
[페르소나 A]
· 나이·성별·직업: 45세 여성, 사무직
· 생활 습관: 운동 부족, 야식 습관
· 계기: 건강검진에서 혈압 경고
· 성격/문체 톤: 걱정 많고 꼼꼼, 반말 일기체

[페르소나 B]
...

[페르소나 C]
...
```

---

## 4. 사용자 선택 옵션

| 버튼 | 동작 |
|------|------|
| 선택 완료 | 선택된 페르소나 → Step 2로 |
| 페르소나 없이 생성 | 페르소나 빈 채로 → Step 2로 |
| 취소 | 생성 중단 |

편집: 라디오 선택 후 텍스트 영역에서 자유 수정 가능.

---

## 5. 평가

**잘된 점:**
- 3개 후보 → 다양성 확보 + 사용자 최종 결정권
- 제3자 관찰형 별도 템플릿으로 시점 차이 반영
- 편집 가능 → 미세 조정 용이

**부족한 점:**
- 페르소나 재생성(다시 3개 뽑기) 기능 없음
- 이전 생성에서 좋았던 페르소나 재사용 기능 없음
