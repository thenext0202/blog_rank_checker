# 03-3. 원고 생성 + 후처리 (Step 3)

> main.py: `_generate_final()`, `build_prompt()`

---

## 1. 기능 요약

14섹션 프롬프트로 원고를 생성하고, 후처리 → 화면 표시 → 자동 저장 → Drive 업로드까지 실행한다.

---

## 2. 처리 흐름

```
① _generate_final() — 전체 프롬프트 조립 (build_prompt)
   14+ 섹션 포함 (02_프롬프트_조립.md 참조)
    ↓
② call_claude_api()
   ├── max_tokens: 8192
   └── 스레드 실행 (30초~2분)
    ↓
③ 후처리
   ├── 마크다운 ** 마커 제거
   ├── 이미지 설명 ㄴ(...) 제거
   └── 이미지 번호 변환 (00→0, 01→1, …)
    ↓
④ 화면 표시
   ├── result_text에 내용 삽입
   └── 태그 하이라이팅:
       ├── "annotation" (초록, 굵게) — ㄴ서식 줄
       ├── "img_num" (파란, 굵게) — 이미지 번호
       └── "blogger_req" (빨간, 굵게) — ★블로거 요청사항★
    ↓
⑤ 스코어보드 업데이트
   ├── 글자수: X,XXX (공백포함 Y,YYY)
   ├── 키워드: [메인]: N회
   └── 이미지: N장
    ↓
⑥ 자동 저장
   ├── output/ 폴더에 .txt 저장
   └── 파일명: 작가_날짜키워드_유형_제품코드.txt
    ↓
⑦ Drive 업로드 (백그라운드)
    ↓
⑧ generation_log.json 기록 (최대 200건 유지)
    ↓
⑨ 배치 모드면 → _batch_next() 호출
```

---

## 3. 배치 모드 (연속 생성)

**입력 형식** (한 줄에 하나):
```
메인키워드 / 연관키워드1,연관키워드2 / #해시1,#해시2
메인키워드2 / 연관키워드3 / #해시3
```

**동작:**
1. 키워드 목록 파싱 → `batch_keywords` 리스트
2. `batch_current = 0`부터 시작
3. 각 키워드로 GUI 필드 업데이트 → `_on_generate(is_batch=True)`
4. 완료 콜백에서 `_batch_next()` → 다음 키워드
5. `batch_current >= batch_count`이면 종료

## 5. 검수 + 수정 재생성

```
① "검수" 버튼 → 검수 프롬프트 구성
   ├── 소구점 반영 체크
   ├── 어색한 표현 체크
   └── 서식 누락 체크
    ↓
② call_claude_api(max_tokens=4096)
    ↓
③ 검수 결과 다이얼로그 (✅/❌/💡 표시)
    ↓
④ "수정 재생성" 선택 시:
   ├── 원본 원고 + 검수 결과 + 7개 수정 규칙
   └── Claude API → 수정된 원고로 교체
```

## 7. 평가

**잘된 점:**
- 후처리로 불필요한 마크다운/이미지 설명 자동 정리
- 스코어보드로 글자수/키워드/이미지 즉시 확인
- 자동 저장 + Drive 업로드 → 수동 작업 최소화
- 배치 모드로 대량 생성 가능
- 검수 → 수정 재생성 루프로 품질 보장

**부족한 점:**
- 생성 실패 시 재시도 없음 (처음부터 다시)
- 배치 모드에서 개별 키워드 skip/재시도 불가
- 검수 재생성이 1회만 — 반복 수정 루프 없음
