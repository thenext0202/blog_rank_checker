## 1. 시스템 목적

건강/의학 블로그 원고를 **Claude AI로 자동 생성**하는 tkinter GUI 프로그램.
페르소나 → 제목 → 원고 3단계 생성, Google Sheets 연동, Word 파일 출력.


## 3. 기술 스택

```
Python 3 + tkinter GUI
├── anthropic (Claude Sonnet 4 API)
├── python-docx (Word 파일 생성)
├── gspread + google-auth (Google Sheets)
├── googleapiclient (Google Drive 동기화)
├── PyMuPDF (PDF 읽기)
└── threading (비동기 API 호출)
```

## 6. MECE 분석

### 6-1. 원고 생성 파이프라인

| 단계 | 커버 여부 | 담당 |
|------|-----------|------|
| ① 데이터 준비 (시트/참고자료) | ✅ 완전 | load_all_from_sheet + Drive sync |
| ② 프롬프트 조립 (14섹션) | ✅ 완전 | build_prompt() |
| ③ 페르소나 생성 | ✅ 완전 | 3개 후보 + 편집 |
| ④ 제목 생성 | ✅ 완전 | 3개 후보 + 편집 |
| ⑤ 원고 생성 | ✅ 완전 | Claude API (max 8192토큰) |
| ⑥ 서식 변환 (→ DOCX) | ✅ 완전 | ㄴ파싱 + python-docx |
| ⑦ 검수 | ✅ 완전 | AI 검수 + 수정 재생성 |
| ⑧ 배포 (Drive 업로드) | ✅ 완전 | 자동 업로드 |
| ⑨ 연속 생성 (배치) | ✅ 완전 | 키워드 목록 → 순차 생성 |

**Gap 없음** — 생성부터 배포까지 전 과정 커버.

### 6-2. 입력 소스

| 소스 | 용도 | 관리 위치 |
|------|------|-----------|
| Google Sheets | 프롬프트/스타일/지침/제품 | 웹 (비개발자 수정 가능) |
| references/ 폴더 | 참고 논문/자료 (.txt, .md, .pdf) | 로컬 + Drive 동기화 |
| samples/ 폴더 | Few-shot 예시 원고 (.docx) | 로컬 + Drive 동기화 |
| GUI 입력 | 키워드/설정/추가지침 | 실행 시 |

### 6-3. 출력 형식

| 형식 | 지원 | 비고 |
|------|------|------|
| Word (.docx) | ✅ | 서식 완전 적용 |
| 텍스트 (.txt) | ✅ | 서식 마크업 포함 |
| 시트 기록 | ✅ | generation_log.json |
| Drive 업로드 | ✅ | output 폴더 자동 |

## 7. 종합 평가

### 잘된 점
- **14섹션 프롬프트 체계**: 빈틈 없는 지시로 일관된 품질
- **3단계 생성 (페르소나→제목→원고)**: 단계별 사용자 개입으로 방향 제어
- **ㄴ서식 시스템**: 텍스트 지시 → Word 포맷 자동 변환
- **시트 기반 설정**: 비개발자가 프롬프트/지침 직접 수정 가능
- **배치 모드**: 키워드 목록으로 연속 자동 생성
- **Drive 동기화**: 팀 협업용 자동 업/다운로드
- **검수+재생성**: AI 품질 검증 → 자동 수정 루프

### 부족한 점
- **main.py 3,482줄 단일 파일**: 모듈 분리 없이 모든 로직이 한 파일
- **API 키 평문 저장**: `.api_key` 파일에 그대로 저장
- **HTML 파싱 없음**: 네이버 관련 기능 없이 순수 생성 전용
- **ㄴ서식 파싱 복잡도**: 멀티라인, 지연 적용 등 엣지 케이스 많음
- **테스트 코드 없음**: 프롬프트 조립, 서식 파싱 등 핵심 로직 검증 부재
- **모델 하드코딩**: `claude-sonnet-4-20250514` 변경 시 코드 수정 필요
