> main.py Lines 1141~1638

---

## 1. 기능 요약

생성된 텍스트 원고를 **ㄴ서식 지시**를 파싱하여 Word(.docx) 파일로 변환한다.

---

## 2. 처리 순서 (DOCX 생성)

```
① ★블로거 요청사항★ 블록
   → 빨간 테두리 + 연한 노란 배경 테이블
   → 빨간 굵은 글씨 (#CC0000), 11pt
    ↓
② ㄴ서식 지시 줄
   → 초록 글씨, 24pt, 형광 초록 하이라이트
   → 실제 서식 적용 (이전 문단에)
    ↓
③ 이미지 번호 (00→0 변환)
   → 파란색, 가운데정렬, 14pt
    ↓
④ 마크다운 제목 (# ## ###)
   → Word 기본 Heading 1/2/3 스타일
    ↓
⑤ 일반 텍스트
   → 마크다운 처리: **굵게**, *기울임*, ***둘다***
```

---

## 3. ㄴ서식 파싱 (parse_annotation)

`ㄴ`으로 시작하는 줄에서 서식 정보 추출:

| 파싱 항목 | 예시 | 결과 |
|----------|------|------|
| 폰트 크기 | `ㄴ 글씨크기 15` | font_size=15 |
| 굵게 | `ㄴ 볼드` | bold=True |
| 기울임 | `ㄴ 이탤릭` | italic=True |
| 색상 | `ㄴ 파란색 글씨` | color=RGB(0,112,192) |
| 형광펜 | `ㄴ 노란 형광펜` | highlight=YELLOW |
| 인용구 | `ㄴ 인용구 3` | quote_style=3 |
| 링크 | `ㄴ 링크 삽입` | has_link=True |
| 멀티라인 | `ㄴ 두 줄 모두` | multi_line=2 |
| 이미지 설명 | `ㄴ (이미지 설명)` | image_desc=... |

## 6. 유효 폰트 크기

→ 범위 밖 입력 시 가장 가까운 값으로 매핑

---

## 7. 멀티라인 적용

| 지시 | 동작 |
|------|------|
| `ㄴ 두 줄 모두` | 직전 2개 문단에 서식 적용 |
| `ㄴ 세 줄 모두` | 직전 3개 문단에 서식 적용 |
| `ㄴ 네 줄 모두` | 직전 4개 문단에 |
| `ㄴ 다섯 줄 모두` | 직전 5개 문단에 |

---

## 8. 지연 적용 (Deferred Application)

색상 단어 매칭이 바로 위 문단에 없을 때:
1. 최대 5개 이전 문단까지 역방향 검색
2. 매칭되면 해당 문단에 적용
3. 못 찾으면 큐에 저장 → 이후 텍스트에서 매칭 시도

---

## 9. 평가

**잘된 점:**
- ㄴ서식 → Word 변환으로 AI 생성 텍스트를 바로 발행 가능한 문서로
- 멀티라인, 지연 적용 등 복잡한 엣지 케이스 처리
- 색상/형광펜 매핑이 풍부 (10색 + 6형광)
- 마크다운 + ㄴ서식 혼합 지원

**부족한 점:**
- 파싱 로직이 regex + 문자열 매칭 혼합 → 디버깅 어려움
- 서식 지시가 정확하지 않으면 무시됨 (사용자에게 피드백 없음)
- Word 스타일 커스터마이징 제한 (Heading 스타일 고정)
- 인용구 6종 스타일이 하드코딩
