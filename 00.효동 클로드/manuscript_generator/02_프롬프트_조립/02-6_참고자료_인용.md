## 1. 기능 요약

`references/` 폴더의 파일 + 시트 "참고논문" 탭의 논문 데이터를 프롬프트에 삽입하고, **3가지 인용 방식**을 안내한다.

---

## 2. 참고자료 소스

| 소스 | 내용 | 최대 길이 |
|------|------|-----------|
| `references/공통/` | 모든 제품 공통 자료 | 8,000자/파일 |
| `references/{제품명}/` | 제품별 전용 자료 | 8,000자/파일 |
| 시트 "참고논문" 탭 | 제품별 논문 요약 | 제한 없음 |

```python
# 파일 참고자료
for fname, content in selected_refs.items():
    if len(content) > 8000:
        content = content[:8000] + "\n... (이하 생략)"
    parts.append(f"\n--- {fname} ---\n{content}")

# 시트 참고논문
if papers:
    for i, paper in enumerate(papers, 1):
        parts.append(f"\n[논문 {i}]\n{paper}")
```

---

## 3. 인용 방식 3가지

프롬프트에서 Claude에게 아래 3가지 중 선택하여 자연스럽게 인용하도록 지시:

### 1) 커뮤니티 발견형 (권장, 가장 자연스러움)

```
"맘카페에서 누가 정리해놓은 글을 봤는데, ~대학교 연구에서 ~라는 결과가 있었대요."
"블로그 돌아다니다가 발견한 건데, ~연구소에서 ~를 확인했다더라고요."
```

→ 온라인 커뮤니티에서 우연히 본 정보처럼 표현

### 2) 검색/탐색형 (본인이 찾아본 경우)

```
"이게 진짜 효과가 있나 싶어서 논문까지 찾아봤는데, ~에 따르면..."
"불안해서 밤새 검색했는데, ~학회지에서 발표된 연구가 있더라고요."
```

→ 적극적으로 검색한 맥락

### 3) 전문가/의사 언급형 (병원/상담 맥락)

```
"의사 선생님이 말씀하시길, ~연구에서도 ~라는 결과가 있다고..."
"영양사분이 추천하면서 ~연구 결과를 보여주시더라고요."
```

→ 전문가로부터 들은 정보

## 4. 금지 사항

```
※ 딱딱한 학술 톤('~대학교 연구팀에 따르면')은 피하세요.
   블로그 글답게 자연스럽게.
※ 참고자료에 없는 논문이나 연구를 지어내지 마세요.
   참고자료에 있는 내용만 인용합니다.
```

---

## 5. 시트 "참고논문" 탭 구조

| 열 | 내용 |
|----|------|
| A | 제품명 |
| B | 연구명 |
| C | 출처 (저널/기관) |
| D | 대상 (피험자) |
| E | 핵심 결과 |
| F | 수치 데이터 |

→ 로딩 시 B~F를 하나의 요약 문자열로 결합

---

## 6. 평가

**잘된 점:**
- 3가지 인용법으로 블로그 톤에 맞는 자연스러운 근거 제시
- 할루시네이션 방지: "있는 내용만 인용" 명시
- 파일 + 시트 이중 소스로 참고자료 관리 유연

**부족한 점:**
- 참고자료가 많으면 프롬프트가 매우 길어짐 (토큰 비용)
- 참고자료 중 어떤 것을 인용할지 Claude 판단에 의존
- 인용 횟수 가이드 없음 (너무 많거나 적을 수 있음)
