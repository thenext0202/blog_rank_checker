## 3. 일반 페르소나 템플릿 (PERSONA_PROMPT_TEMPLATE)

**입력 변수:** product_name, product_guide, prompt_type, keywords, style_desc, tone

**출력 형식:**
```
[페르소나 A]
· 나이·성별·직업:
· 생활 습관 (건강 문제가 생긴 이유):
· 계기 (제품을 알게 된 배경):
· 성격/문체 톤:

[페르소나 B] ...
[페르소나 C] ...
```

**설계 규칙:**
- 제품과 키워드에 맞는 실존감 있는 인물
- 3개 제안, 다른 설명 불필요

---

## 4. 제3자 관찰형 템플릿 (PERSONA_PROMPT_3RD)

**차이점:** 페르소나가 블로거가 아니라 **관찰 대상**

```
이 페르소나는 블로그 글의 주인공이 아니라,
블로거(관찰자)가 곁에서 지켜보는 '대상 인물'입니다.
```

**출력 형식:**
```
[페르소나 A]
· 관찰 대상: 나이·성별·직업 (블로거와의 관계: 예-엄마, 남편, 직장 동료)
· 생활 습관 (건강 문제가 생긴 이유):
· 계기 (제품을 알게 된 배경):
· 관찰자 시점: (예: 딸이 엄마의 변화를 지켜봄)
```

parts.append(f"===== 블로거 페르소나 =====\n{persona_text}")

    # 관찰자 ≠ 대상 구분 지시
    parts.append("위 페르소나는 '관찰 대상'입니다.")
    parts.append("'나'는 관찰자이고, 위 페르소나는 관찰 대상입니다.")
else:
    parts.append("위 페르소나의 시점에서 원고를 작성해주세요.")

## 7. 평가

**잘된 점:**
- 일반/제3자 관찰형 분리로 시점 차이 정확히 반영
- 4개 항목(나이직업/습관/계기/톤)으로 구체적이면서 간결한 페르소나
- 관찰자-대상 관계를 페르소나에 포함 (딸↔엄마 등)

**부족한 점:**
- 템플릿이 코드에 하드코딩 → 시트로 이동하면 유연성 확보
- 제품 소구점을 3,000자로 자름 → 긴 소구점 정보 손실 가능
- 이전 대화에서 생성된 페르소나 재활용 기능 없음
