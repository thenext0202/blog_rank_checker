# 02-2. 레퍼런스 원고 (Few-Shot 샘플)

> build_prompt() 섹션 1-1, `load_sample_for_type()`

---

## 1. 기능 요약

잘 작성된 실제 원고 샘플을 프롬프트에 삽입하여, Claude가 **톤/구조/서식/흐름을 모방**하도록 유도한다.

---

## 2. 샘플 매칭 로직

```python
sample_fname, sample_text = load_sample_for_type(prompt_type, product_name)
```

### 매칭 우선순위

| 순위 | 조건 | 예시 |
|------|------|------|
| 1순위 | 같은 유형 + 같은 제품 | `_시나리오형(수치)_ml.docx` |
| 2순위 | 같은 유형 + 다른 제품 | `_시나리오형(수치)_bc.docx` |
| 없음 | 매칭 실패 | 샘플 섹션 생략 |

### 파일명 규칙

```
{작가명}_{날짜키워드}_{유형}_{제품코드}.docx
```

예: `강보람_240101수면_시나리오형(수치)_ml.docx`

---

## 3. 프롬프트 삽입 방식

```
===== 참고 원고 예시 (톤/구조/서식 참고용) =====
아래는 같은 유형으로 잘 작성된 실제 원고 예시입니다.
이 원고의 톤, 문장 길이, 줄 끊기, 서식 지시(ㄴ), 이미지 배치, 전체 흐름을 참고하여 작성해주세요.
단, 내용은 그대로 베끼지 말고 이번 키워드/제품/페르소나에 맞게 새로 작성합니다.

--- 예시 원고 시작 ---
{sample_text (최대 4,000자)}
--- 예시 원고 끝 ---
```

---

## 4. 토큰 절약

```python
if len(sample_text) > 4000:
    sample_text = sample_text[:4000] + "\n... (이하 생략)"
```

- DOCX → 텍스트 변환 후 최대 4,000자만 사용
- 전체 원고가 아닌 앞부분(도입~중반) 위주로 참고

---

## 5. 샘플 관리

| 항목 | 내용 |
|------|------|
| 저장 위치 | `samples/` 폴더 |
| 파일 형식 | `.docx` (Word) |
| Drive 동기화 | 프로그램 시작 시 자동 다운로드 |
| 추가 방법 | Drive 폴더에 업로드 또는 로컬 직접 배치 |

---

## 6. 지시 포인트

프롬프트에서 Claude에게 명시하는 참고 방향:
1. **톤** — 문장의 분위기, 감정 표현 수준
2. **문장 길이** — 줄 끊기 패턴
3. **서식 지시(ㄴ)** — ㄴ 사용 빈도와 위치
4. **이미지 배치** — 이미지 번호 삽입 패턴
5. **전체 흐름** — 도입→전개→전환→마무리 구조

**금지:** 내용 복사. 구조만 참고하고 새 키워드/제품/페르소나에 맞게 재작성.

---

## 7. 평가

**잘된 점:**
- Few-shot으로 일관된 품질 유지 (특히 ㄴ서식 패턴)
- 유형+제품 매칭으로 가장 관련 높은 샘플 자동 선택
- 4,000자 제한으로 토큰 비용 관리

**부족한 점:**
- 샘플이 없는 유형+제품 조합은 품질 편차 발생
- 샘플 품질 관리 기준/프로세스 없음 (어떤 원고가 '좋은 샘플'인지)
- 여러 샘플 중 랜덤/최신 선택 로직이 명확하지 않음
