## 1. 프롬프트 템플릿 (섹션 1)

시트 "프롬프트" 탭에서 선택된 유형의 프롬프트를 통째로 삽입한다.

```python
prompt_template = sheet_data["prompts"].get(prompt_type, "")
parts.append(prompt_template)
```

### 지원 유형 (시트에서 관리)

| 유형명 | 설명 |
|--------|------|
| 후기형(ver3) | 체험 후기 서사 (최신) |
| 후기형(ver2) | 체험 후기 서사 (이전) |
| 에어서치 | 정보 탐색형 |
| 원료기반형 | 성분/원료 중심 |
| 시나리오형(수치) | 수치 비교 시나리오 |
| 시나리오형(질병) | 질병 시나리오 |
| 1인칭 경험담 | 본인 경험 서사 |
| 공감 정보형 | 공감 → 정보 → 솔루션 |

+ 5개 분류 유형: 수치 충격형, 돌발 증상형, 증상 악화 진행형, 정보 탐색 큐레이션형, 제3자 관찰형

**핵심:** 프롬프트 내용은 코드에 없고 **시트에서 관리**한다. 유형 추가/수정 시 시트만 편집하면 된다.

---

## 2. 제품 소구점 (섹션 4)

```python
product_guide = sheet_data["products"].get(product_name, "")
parts.append(f"===== 제품 정보: {product_name} =====\n{product_guide}")
```

시트 "제품소구점" 탭 B열의 소구점 가이드를 통째로 삽입.

---

## 3. 목차 규칙 (섹션 4-1)

고정 텍스트:
```
목차(소제목)에는 제품명, 배합명, 허브키워드를 넣지 마세요. 광고처럼 보입니다.
목차는 독자의 궁금증/감정/상황 중심으로 작성하세요.
```

---

## 4. 평가

**잘된 점:**
- 유형별 프롬프트를 시트에서 관리 → 코드 수정 없이 유형 추가/개선 가능
- 소구점 가이드로 제품별 핵심 메시지 일관성

**부족한 점:**
- 프롬프트 템플릿이 매우 길 수 있음 (3000자+) → 토큰 비용
- 목차 규칙이 하드코딩 → 시트로 이동하면 유연성 확보
