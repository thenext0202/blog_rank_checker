## 🐛 무엇이 문제였나

검수 결과에 **"요청 문구 노출"이 200건 이상** 표시되었지만, 실제로는 요청 문구가 노출된 건이 없었음.

### 원인

원고에 `★블로거님 요청사항★` 이후 `---` 구분선 없이 바로 본문이 이어지면, **본문 전체를 요청 문구로 잘못 인식**.

```
★블로거님 요청사항★         ← 여기서 "요청 문구 수집 모드" 시작
- 글자크기 15/ 왼쪽 정렬    ← 실제 요청 문구 ✓
- 인용구, 스티커 적용        ← 실제 요청 문구 ✓
제목 : 식후 2시간 혈당...    ← 실제 요청 문구 ✓
0                           ← ❌ 기존: 요청 문구로 인식 → 수정 후: 이미지 번호
식후2시간혈당을 낮추려면     ← ❌ 기존: 요청 문구로 인식 → 수정 후: 본문
가장 먼저 손봐야 하는 건     ← ❌ 기존: 요청 문구로 인식 → 수정 후: 본문
```

블로그에는 당연히 본문이 있으니 → 전부 "요청 문구 노출"로 오탐 발생

## ✅ 수정 결과

이제 `-`, `ㄴ` 시작 / 키워드 포함 / `부탁`·`요청` 포함 등 **지시사항 특성이 있는 줄만** 요청 문구로 인식하고, 본문이 시작되면 자동으로 수집을 종료합니다.

- 요청 문구 10개 → 정상 10개로 인식
- 본문 → content로 정상 분류
- `ㄴ 인용구 3번 글자크기 24` 같은 서식 지시는 계속 정상 인식
