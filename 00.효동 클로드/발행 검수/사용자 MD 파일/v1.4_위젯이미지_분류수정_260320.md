## 📋 무엇을 수정했나?

블로그 검수 시 **mkt 링크의 삽입 위치**(이미지/위젯)가 **반대로 표시**되던 버그를 수정했습니다.

### 수정 전 (버그)

| 표시 | 실제 |
|------|------|
| `[이미지]` 위젯 링크 | ❌ 실제로는 위젯(oglink)에 삽입된 링크 |
| `[위젯]` 이미지 링크 | ❌ 실제로는 이미지에 삽입된 링크 |

### 수정 후 (정상)

| 표시 | 실제 |
|------|------|
| `[위젯]` 위젯 링크 | ✅ 정확히 위젯으로 분류 |
| `[이미지 N/M번째]` 이미지 링크 | ✅ 정확히 이미지로 분류 |

---

## 🔍 왜 뒤바뀌었나? (원인)

네이버 블로그의 **oglink 위젯**(링크 도구)은 내부에 **썸네일 이미지(`<img>`)** 를 포함합니다.

```
oglink 위젯 구조:
<div class="se-component se-oglink">
  <a href="mkt링크">           ← 이 <a> 안에
    <img src="썸네일.jpg">     ← 썸네일 이미지가 있음!
    <span>상품명...</span>
  </a>
</div>
```

프로그램이 블로그의 모든 `<a>` 태그를 스캔할 때, `<a>` 안에 `<img>`가 있으면 "이미지 링크"로 분류합니다.
→ **oglink 위젯의 썸네일 때문에 위젯 링크가 "이미지"로 잘못 분류**된 것입니다.

---

## 🛠️ 어떻게 수정했나? (구현 방법)

### 수정 1: `_add_link()` — 위젯 확인 시 이미지 목록에서 제거

> 핵심: step 2(위젯 셀렉터)에서 위젯으로 확인되면, step 1에서 잘못 넣은 `image_links` 항목을 제거

```python
# 수정 전
elif link_type == "widget" and href not in widget_links:
    widget_links.append(href)

# 수정 후
elif link_type == "widget":
    if href not in widget_links:
        widget_links.append(href)
    # oglink 썸네일 <img>로 인해 image로 오분류된 경우 제거
    if href in image_links:
        image_links.remove(href)
```

### 수정 2: `_show_detail()` — 태그 우선순위 변경

> 핵심: 위젯을 일반 이미지보다 먼저 체크 (image_link_map은 여전히 1순위)

```
수정 전: image_link_map → image_links → widget_links → 텍스트
수정 후: image_link_map → widget_links → image_links → 텍스트
                          ^^^^^^^^^^^^ 위젯이 먼저!
```

### 수정 3: `_link_found_in_blog()` — 반환 우선순위 변경

> 핵심: 검수 결과에서도 위젯을 이미지보다 먼저 판정

```
수정 전: image → widget → text → missing
수정 후: widget → image → text → missing
```

## 📖 사용 방법

기존과 동일합니다. 프로그램 실행 후 검수하면 링크 목록에 정확한 태그가 표시됩니다:

| 태그 | 의미 |
|------|------|
| `[이미지 3/7번째]` | 블로그 7개 이미지 중 3번째에 링크가 걸림 |
| `[이미지]` | 이미지에 링크가 걸렸으나 순번 미확인 |
| `[위젯]` | 링크 도구(oglink)로 삽입됨 |
| `[텍스트]` | 텍스트에 하이퍼링크로 삽입됨 |

---

## ✅ 잘된 점

- **근본 원인 해결**: 단순 우선순위 변경이 아니라, `_add_link()`에서 오분류 자체를 제거
- **3곳 일관 수정**: 수집(`_add_link`), 표시(`_show_detail`), 검수(`_link_found_in_blog`) 모두 동일한 우선순위 적용
- **기존 기능 유지**: `image_link_map`(JS 기반 정밀 이미지 추적)은 여전히 최우선

---

## ⚠️ 주의할 점

- `image_links.remove(href)`는 **정확한 URL 문자열 매칭**에 의존 — URL 인코딩이 다르면 제거 실패 가능
- oglink 외에 다른 위젯 유형(`se-link` 등)에서도 썸네일 `<img>`가 있을 수 있음 — 추가 케이스 모니터링 필요
