# 🔗 v1.6 — 배너 링크 원본 URL 수집 개선

> 📅 2026-03-20
> 📁 `발행검수\main.py`

---

## 📌 무슨 문제였나?

상품 링크와 광고 링크가 **같은 MKT ID**를 공유하고, 쿼리 파라미터(`_bc05` 등)로만 구분되는 경우:

| 구분 | URL 예시 |
|------|---------|
| 상품 | `mkt.../link/ABC?nt_detail=260313홍국영양제&nt_keyword=bc` |
| 광고 | `mkt.../link/ABC?nt_detail=260313홍국영양제_bc05&nt_keyword=bc` |

- 블로그에 두 링크 모두 존재하는데, 프로그램이 `_bc05`를 인식 못함
- 결과: **"광고 링크 미삽입"** 오탐 발생
- 추가로 같은 배너가 **2건으로 중복 표시**되는 문제도 있었음

---

## 🔧 어떻게 수정했나?

### 수정 1: `data-linkdata`에서 원본 URL 추출

> **원인:** 네이버 블로그 JS가 oglink 배너의 `<a>` href를 렌더링할 때 파라미터를 수정/제거함.
> 하지만 `data-linkdata` HTML 속성에는 블로거가 붙여넣은 **원본 URL이 보존**되어 있음.

| 기존 | 변경 |
|------|------|
| `a.href`로 URL 수집 | 배너인 경우 `data-linkdata`에서 MKT URL 우선 추출 |
| `_bc05` 등 파라미터 소실 | 원본 파라미터 보존 |

### 수정 2: oglink 컴포넌트 단위 중복 제거

> **원인:** 하나의 oglink 배너 안에 `<a>` 태그가 여러 개 (썸네일, 제목 등) → 같은 배너가 2~3건으로 수집

| 기존 | 변경 |
|------|------|
| 모든 `<a>` 개별 수집 | `processedBanners` 배열로 처리된 oglink 추적 |
| 1개 배너 → 2~3건 | 1개 배너 → 1건 |

변경 사항 없음 — 기존과 동일하게 사용하면 됨.
이전에 "광고 링크 미삽입"으로 오탐되던 건이 정상 인식됩니다.

---

## 👍 잘된 점

- `data-linkdata` 활용으로 네이버 블로그 JS의 URL 수정 문제를 근본적으로 해결
- oglink 컴포넌트 단위 중복 제거로 링크 목록이 실제 배너 수와 일치
- 기존 이미지/텍스트 링크 수집에는 영향 없음

## ⚠️ 주의할 점

- `data-linkdata`가 없는 구형 블로그 에디터(SE2 이하)에서는 기존 `a.href` 방식으로 fallback
- `data-linkdata`에 MKT URL이 없는 경우에도 `a.href`로 fallback
