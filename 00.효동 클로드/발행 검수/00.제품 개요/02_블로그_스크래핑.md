## 1. Selenium 설정 (setup_driver)

### Chrome Headless 옵션

```python
--headless=new
--no-sandbox
--disable-dev-shm-usage
--disable-blink-features=AutomationControlled
excludeSwitches: ["enable-automation"]
useAutomationExtension: False
User-Agent: Chrome 131.0
CDP: navigator.webdriver = undefined
```

→ 탐지 방지 설정 적용

## 2. 병렬 스크래핑 (4 workers)

```
items를 4등분 → 각 chunk에 Chrome 드라이버 1개
    → threading.Thread(target=crawl_worker)
    → Lock으로 진행 카운터 공유
    → root.after()로 프로그레스 업데이트
```

## 3. 블로그 콘텐츠 추출 (scrape_blog)

```

```

## 4. 링크 수집 (3단계)

### Stage 1: 일반 `<a>` 태그

```css
div.se-main-container a
div#postViewArea a
```
- `<img>` 포함 → image_links
- 나머지 → links

### Stage 2: 네이버 위젯 링크

```css
a.se-oglink-info          # SE3 링크 도구
a.se-module-oglink         # oglink 모듈
div.se-module-oglink a
a[data-linkdata]           # data-linkdata 속성
div.se-section-oglink a
a.se-link
```
→ widget_links

### Stage 3: data-linkdata 파싱

`data-linkdata` 속성에서 URL regex 추출

### 링크 정리

```python
_clean_href(href):   # NaPm 추적 파라미터 제거, URL 디코딩
_skip(href):         # 네이버 내부링크, 앵커, 비HTTP 제외
_add_link(href):     # 중복 제거 후 적절한 리스트에 추가
```

## 6. 평가

**잘된 점:**
- 4개 병렬 드라이버로 스크래핑 속도 최적화
- 3단계 링크 수집으로 다양한 삽입 방식 커버
- 탐지 방지 설정으로 차단 위험 최소화
- JavaScript 실행으로 서식 정보까지 추출

**부족한 점:**
- SE3 에디터 버전에 따라 DOM 셀렉터가 달라질 수 있음
- iframe 내부 링크 미검출
- 스크래핑 실패 시 빈 결과 반환 (재시도 없음)
- 드라이버 종료 타이밍 이슈 가능 (예외 시 좀비 프로세스)
