## 1. 링크 매칭 흐름

```
① 대상 링크 (시트의 product_link / ad_link)
    ↓
② _extract_link_id(url) — 핵심 ID 추출
   ├── mkt.shopping.naver.com/link/{code} → code
   ├── smartstore.naver.com/{store} → store ID
   └── fallback: base URL
    ↓
③ _resolve_url(url) — 리다이렉트 최종 URL 추적
   ├── requests.head() 시도
   └── requests.get() fallback
    ↓
④ _normalize_url(url) — 정규화
   └── scheme/query/trailing slash 제거, 소문자
    ↓
⑤ 블로그 3개 링크 리스트에서 매칭 검색
   ├── image_links → "image"
   ├── widget_links → "widget"
   ├── links (텍스트) → "text"
   └── 없음 → "missing"
```

## 5. 평가

**잘된 점:**
- 리다이렉트 추적으로 단축 URL도 매칭 가능
- ID 추출 + 정규화로 URL 변형(파라미터 차이 등) 대응
- image/widget/text 3단계 분류로 삽입 형태까지 파악
- 이미지 삽입 필수 여부를 DOCX 지시사항에서 자동 판단

**부족한 점:**
- 모든 링크에 HTTP 요청 → 속도 저하 (캐싱 없음)
- mkt.shopping 이외의 쇼핑 링크 패턴 누락 가능
- 블로그 링크 수집 자체가 누락되면 정상 삽입도 "missing" 판정
