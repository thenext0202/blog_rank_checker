# 03-3. 서식 검증

> main.py: `check_publication()` 검수 7, `_parse_format_info()`, `_color_matches()`

---

## 검수 ⑦: 서식 매칭

DOCX의 서식 요구사항(format_reqs)과 블로그의 실제 서식(format_info)을 대조한다.

---

## 1. DOCX 서식 요구 (format_reqs)

ㄴ서식 줄에서 파싱:

```python
{
    "text": "본문 텍스트",      # 서식 적용 대상 텍스트
    "quote": 1,                # 인용구 번호 (None이면 체크 안 함)
    "font_size": 24,           # 글자크기 (None이면 체크 안 함)
    "bold": True,              # 굵게 여부
    "color": "red",            # 색상 (None이면 체크 안 함)
}
```

### 서식 파싱 regex (_parse_format_info)

| 패턴 | 추출 |
|------|------|
| `인용구\s*(\d+)` | 인용구 번호 |
| `글자\s*크기\s*(\d+)` | 폰트 크기 |
| `글꼴\s*두껍게` | bold=True |
| `빨간색` | color="red" |
| `파란색` | color="blue" |
| `초록색` | color="green" |
| `주황색` | color="orange" |
| `보라색` | color="purple" |

---

## 2. 블로그 서식 정보 (format_info)

JavaScript로 수집:

```python
{
    "text": "문단 텍스트",
    "in_quote": True,          # .se-quotation 내부 여부
    "font_size": 24,           # se-fs-fs{N} 클래스
    "bold": False,             # <b>, <strong> 존재
    "colors": [[255,0,0], ...] # span[style*="color"] RGB
}
```

---

## 3. 매칭 로직

```
DOCX format_req 각각에 대해:
  ① 텍스트 정규화 후 첫 20자로 블로그 format_info 검색
  ② 매칭된 문단에서 서식 비교:
     ├── quote: DOCX 요구 있으면 blog in_quote==True 확인
     ├── font_size: 정확히 일치 확인
     ├── bold: DOCX 요구 있으면 blog bold==True 확인
     └── color: _color_matches() RGB 범위 확인
  ③ 불일치 시 이슈 생성
```

---

## 4. 색상 매칭 (_color_matches)

RGB 범위로 판별:

| 색상 | R | G | B |
|------|---|---|---|
| red | >150 | <100 | <100 |
| blue | <100 | <100 | >150 |
| green | <100 | >150 | <100 |
| orange | >200 | >100 | <50 |
| purple | >100 | <100 | >150 |

---

## 5. 평가

**잘된 점:**
- DOCX ㄴ서식 → 블로그 실제 서식 자동 대조
- JavaScript로 SE3 에디터의 클래스 기반 서식 추출
- RGB 범위 매칭으로 미세한 색상 차이 허용

**부족한 점:**
- 텍스트 매칭이 첫 20자 기준 → 짧은 텍스트나 중복 텍스트에서 오매칭
- 형광펜(highlight) 검증 미구현
- 인용구 번호(1~6) 구분 없이 in_quote 여부만 체크
- SE3 에디터의 font-size 클래스가 변경되면 파싱 깨짐
