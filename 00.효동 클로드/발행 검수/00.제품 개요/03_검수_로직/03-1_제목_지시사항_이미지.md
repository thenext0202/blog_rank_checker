| 항목 | 내용 |
|------|------|
| 비교 대상 | 시트 제목 vs 블로그 제목 |

| 항목 | 내용 |
|------|------|
| 소스 | DOCX `instructions[]` |
| 대상 | 블로그 `body` |
| 조건 | 지시사항 길이 ≥5자 AND 정규화 후 블로그 본문에 포함 |

| 심각도 | High |

→ ★블로거 요청사항★ 블록 등 편집자 지시가 블로그에 그대로 노출된 경우

## 검수 ④: 이미지 번호 노출

| 항목 | 내용 |
|------|------|
| 소스 | DOCX `image_numbers[]` |
| 대상 | 블로그 `body` (줄 단위) |
| 조건 1 | DOCX 이미지 번호가 블로그 줄에 정확히 매칭 |
| 조건 2 | regex `^0\d$` (00~09 형태) 블로그 줄에 존재 |
| 판정 | `"이미지 번호 노출: {number}"` |
| 심각도 | High |

→ "01", "02" 등의 이미지 번호가 블로그에 텍스트로 남은 경우

---

## 평가

**잘된 점:**
- 텍스트 정규화로 유니코드/공백 차이에 의한 오탐 방지
- 서식 키워드 14개로 주요 지시어 커버
- 문맥 표시(±30자)로 오탐/정탐 판단 용이
- 이미지 번호는 DOCX 기반 + 패턴 기반 이중 체크

**부족한 점:**
- 서식 키워드가 본문 내용으로 정당하게 사용된 경우도 오탐 (예: "글씨 크기가 작아서")
- 이미지 번호 "0"은 본문에서 자연스럽게 나올 수 있어 오탐 가능
