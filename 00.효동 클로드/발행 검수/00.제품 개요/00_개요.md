## 1. 시스템 목적

발행된 네이버 블로그를 **원본 DOCX와 자동 대조**하여 문제를 검출하는 tkinter GUI 프로그램.
Selenium 4개 병렬 브라우저로 블로그를 스크래핑하고, 7가지 검수 항목을 체크한다.

## 3. 기술 스택

```
Python 3 + tkinter GUI
├── selenium + webdriver-manager (Chrome headless, 4 병렬)
├── gspread + google-auth (Google Sheets)
├── googleapiclient (Google Drive)
```

## 8. 종합 평가

### 잘된 점
- **Selenium 4개 병렬**로 스크래핑 속도 최적화
- **7가지 검수 항목**으로 주요 발행 오류 자동 검출
- **링크 매칭 로직** — 리다이렉트 추적 + ID 추출 + 정규화로 정확도 높음
- **발행처별 메시지 자동 생성** → 바로 복사해서 전달
- **상세 편집** → 검수 결과를 수동으로 조정 가능
- **DOCX 비교 팝업** → 원본 확인 용이

### 부족한 점
- **DOCX 다운로드 실패** 빈번 — 파일명 매칭 로직 개선 필요
- **블로그 링크 수집 누락** — SE3 에디터 구조에 따른 DOM 셀렉터 갭
- **URL 리졸브 속도** — 링크마다 HTTP 요청 (캐싱 없음)
- **검수 결과 이력 없음** — 과거 검수 기록 미저장
- **main.py 단일 파일** 1,433줄 — 모듈 분리 필요
- **설정 하드코딩** — Sheet ID, Drive ID, 키워드 목록 모두 코드 내
