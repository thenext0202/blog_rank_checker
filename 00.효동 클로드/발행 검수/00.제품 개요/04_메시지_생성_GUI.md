# 04. 메시지 생성 + GUI 구조

> main.py: `generate_message()`, GUI 클래스

---

## 1. 메시지 생성 (generate_message)

### 입력

체크된 (item, issues) 목록

### 출력 형식

```
[발행처1]
안녕하세요, 아래 2건 내용 수정 요청 부탁드립니다.

https://blog.naver.com/...
ㄴ 이슈1
ㄴ 이슈2

https://blog.naver.com/...
ㄴ 이슈3

감사합니다 : )

========================================

[발행처2]
안녕하세요, 아래 1건 내용 수정 요청 부탁드립니다.

https://blog.naver.com/...
ㄴ 이슈1

감사합니다 : )
```

- 발행처별 그룹핑
- 건수 자동 카운트
- ㄴ 접두사로 이슈 나열

## 2. GUI 레이아웃 (1100×850)

```
┌─────────────────────────────────────────┐
│ 발행 원고 검수 (헤더)                     │
├─────────────────────────────────────────┤
│ [날짜 선택] [검수시작] [중단]              │

│ 진행률 ▓▓▓░░░ 30%                       │
├────────────────────┬────────────────────┤
│ 검수 결과 트리뷰     │ 수정 요청 메시지    │

│   param  발행처  제목│                    │
│ ─────────────────  │                    │
│ 상세 정보           │                    │
│ (편집 가능 텍스트)   │                    │
│ ─────────────────  │                    │
│ 블로그 내 링크 목록  │                    │
└────────────────────┴────────────────────┘
│ 통과 8건 / 문제 2건 (이슈 5개)            │
└─────────────────────────────────────────┘
```

---

## 3. 트리뷰 (Treeview)

| 컬럼 | 폭 | 내용 |
|------|-----|------|
| chk | 30 | ☑/☐ 체크박스 |
| status | 80 | 통과 / 문제 N건 |
| param | 150 | 파라미터 |
| publisher | 80 | 발행처 |
| title | 250 | 제목 |

- `ok` 태그: 초록 (통과)
- `err` 태그: 빨강 (문제)
- 클릭: 상세 표시, 더블클릭: DOCX 비교 팝업

---

## 4. 상세 편집

상세 텍스트 영역에서 이슈를 직접 수정 가능:

```
제목: 고혈압 관리법
파라미터: 효동_고혈압
발행처: A업체
링크: https://blog.naver.com/...

검수 결과:
1. 상품 링크 미삽입
2. 이미지 번호 노출: 02
```

→ 번호를 추가/삭제하면 `_save_detail_edits()`가 파싱하여 반영
→ 이슈 변경 시 트리뷰 상태도 자동 업데이트

---

## 5. DOCX 비교 팝업

트리뷰 더블클릭 시:
- 새 창 (700×700)
- DOCX 전체 텍스트 표시 (읽기 전용)
- "블로그 열기" 버튼

---

## 6. 키보드/인터랙션

| 동작 | 결과 |
|------|------|
| 트리뷰 클릭 (1열) | 체크 토글 |
| 트리뷰 클릭 (기타) | 상세 표시 |
| 트리뷰 더블클릭 | DOCX 비교 팝업 |
| 트리뷰 Enter | 블로그 열기 |
| 트리뷰 ↑↓ | 상세 전환 |
| 링크 리스트 더블클릭 | 해당 URL 열기 |

---

## 7. 평가

**잘된 점:**
- 발행처별 메시지 자동 그룹핑 → 바로 복사해서 전달
- 상세 편집으로 오탐 제거 / 추가 이슈 기입 가능
- DOCX 비교 팝업으로 원본 즉시 확인
- 블로그 내 링크 목록으로 삽입 상태 한눈에 파악

**부족한 점:**
- 메시지 템플릿 고정 (발행처별 커스터마이징 불가)
- 검수 결과 이력 저장 없음 (매번 새로 실행)
- 대량 체크 토글 (전체 선택/해제) 기능 없음
