## 🐛 문제

`★블로거 요청사항★` 이후 `in_req = True`가 설정되면, `---` 구분선이 없는 원고에서 **본문 전체가 instructions 리스트에 추가**됨.

- 본문 줄이 모두 "요청 문구"로 분류
- 블로그에 당연히 존재하는 텍스트 → **200건+ "요청 문구 노출" 오탐 발생**
- 원인: 기존 종료 조건이 `---` 구분선뿐이었고, 빈 줄은 `in_req` 체크 전에 `continue`로 넘어감

### 원고 구조 예시

```
★블로거님 요청사항★

- 글자크기 15/ 왼쪽 정렬 / 줄 간격 250%
- 인용구, 스티커, 글자크기, 강조 등 꼭 적용해주세요
- 해시태그(5개 이상): #식후혈당 ...
제목 : 식후 2시간 혈당 낮추는 방법 관리 후기

0                              ← 여기서 in_req 종료
식후 2시간 혈당 낮추는 방법...   ← 본문 (content)
ㄴ 세 줄 모두 글자크기 11...    ← 일반 ㄴ 파싱으로 처리
1. 인슐린 저항성과의 관계       ← content
ㄴ 인용구 3번 글자크기 24...    ← 일반 ㄴ 파싱으로 처리
```

> 본문도 짧은 줄 단위로 작성되어 있어 글자 수 기반 판별 불가

## ✅ 수정 내용

`in_req` 모드에서 **모든 줄을 무조건 캡처**하던 방식 → **지시사항 특성이 있는 줄만 캡처**, 없으면 `in_req=False` 후 일반 파싱으로 전환:

```python
is_instr_like = (
    text.startswith(("ㄴ", "-", "·", "*", "•"))
    or any(kw in text for kw in INSTRUCTION_KEYWORDS)
    or "★" in text
    or "부탁" in text or "요청" in text
    or text.startswith(("제목", "해시태그"))
    or re.match(r"^https?://", text)
)
if is_instr_like:
    instructions.append(text)
    continue
else:
    in_req = False
    # 아래 일반 파싱 로직으로 계속 진행
```

### 판별 기준

| 조건 | 예시 |
|------|------|
| `ㄴ`, `-` 등으로 시작 | `- 글자크기 15/...`, `ㄴ 인용구 3번...` |
| INSTRUCTION_KEYWORDS 포함 | `글자크기`, `인용구`, `줄 간격`, `배너 삽입` 등 |
| `★`, `부탁`, `요청` 포함 | `★블로거 요청사항★`, `적용 부탁드립니다` |
| `제목`/`해시태그`로 시작 | `제목 : 식후...`, `해시태그 10개` |
| URL | `https://mkt.shopping...` |

### `in_req` 종료 후

- `ㄴ` 서식 지시줄은 일반 파싱의 `ㄴ` 처리 블록(242행~)에서 정상 캡처
- 이미지 번호(`0`, `1` 등)는 일반 파싱의 `^\d{1,2}$` 매칭으로 처리
- 본문 텍스트는 content로 정상 분류
