## 1. 기능 요약

키워드 창고(전환 키워드 + 잠재 키워드)에서 **전환 70% + 잠재 30%** 비율로 작가별 키워드를 배정한다.

---

## 2. 실행 방법

```bash
python keyword_assign.py
# → CLI 대화형: 최대 건수 입력 → 제품 선택 → 미리보기 → 확인
```

---

## 3. 처리 흐름

```
① 전환 키워드 탭 → 키워드 창고 갱신
   (전환 키워드 유지 + 시트1에서 잠재 키워드 추가, 검색량 ≥ 1000)
    ↓
② 원고 작성 건수 탭 → 오늘 작가별 quota (M/D 형식 날짜 매칭)
    ↓
③ 사용자 입력: 최대 배정 건수 + 사전 확정 키워드(선택) + 배정 제품
    ↓
④ 기존 배정 확인 (최근 2주 중복 방지, 2주 초과 자동 삭제)
    ↓
⑤ 배정 실행
   ├── PASS 0: 사전 확정 키워드
   ├── PASS 1: 전환 키워드 70% (작가별 제품 효율 순서)
   └── PASS 2: 잠재 키워드 30% (전환 중 낮은 금액 → 잠재로 교체)
    ↓
⑥ 미리보기 + 사용자 수정 (번호로 제외/교체, 대체 후보 5개 제시)
    ↓
⑦ 네이버 자동완성 띄어쓰기 보정 (ac.search.naver.com)
    ↓
⑧ 네이버 스마트블록/인기글 체크 (search.naver.com HTML 파싱)
    ↓
⑨ 키워드 배정 탭에 11열 기록
```

---

## 4. 시트 탭 구조

### 전환 키워드 탭 (4열 단위 반복)
| 열 | 내용 |
|----|------|
| 1 | 키워드 |
| 2 | 파라미터 |
| 3 | 작가명 |
| 4 | 금액 |

### 키워드 창고 탭 (5열)
| 열 | 내용 |
|----|------|
| A | 제품명 |
| B | 키워드 |
| C | 검색량 |
| D | 평균전환금액 |
| E | 분류 (전환/잠재) |

### 키워드 배정 탭 (11열, 출력)
| 열 | 내용 |
|----|------|
| A | 날짜 (YYYY-MM-DD) |
| B | 작가명 |
| C | 제품 |
| D | 키워드 (띄어쓰기 보정) |
| E | 구분 (전환/고잠재) |
| F | 검색량 |
| G | 평균금액 |
| H | 점수 |
| I | 배정 사유 |
| J | 스마트블록 (O/X) |
| K | 스마트블록 타이틀 |

## 5. 전환점수 산출 (calc_conversion_score)

| 항목 | 배점 | 기준 |
|------|------|------|
| 검색량 | 30점 | 10000+ ~ 500+ 구간별 |
| 구매의도 키워드 | 50점 | 추천, 가격, 후기, 효능 등 포함 여부 |
| 카테고리 매칭 | 15점 | 제품 카테고리 키워드 포함 |
| 전환금액 | 20점 | 100,000원+ |
| **최대** | **115점** | |

## 9. 평가

**잘된 점:**
- 전환/잠재 비율로 실적 안정성과 신규 키워드 발굴을 균형
- 작가별 제품 전환 이력 → 전문 분야 우선 배정
- 미리보기 + 대체 후보 5개 제시 → 수동 개입 용이
- 전환점수 115점 체계로 잠재 키워드 정량 평가

**부족한 점:**
- 전환/잠재 비율(70/30) 고정 — 상황별 조절 불가
- 키워드 창고 갱신 시 기존 잠재 키워드 전부 교체 (누적 안 됨)
- 배정 중간 실패 시 롤백 없음
