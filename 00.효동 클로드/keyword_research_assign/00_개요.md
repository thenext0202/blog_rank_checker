## 1. 시스템 목적

건강/의학 블로그 마케팅을 위한 **키워드 리서치 → 배정** 파이프라인.

| 파일 | 역할 | 상세 문서 |
|------|------|-----------|
| `main.py` | 키워드 리서치 | `01_키워드_리서치.md` |
| `keyword_assign.py` | 창고 기반 배정 | `02_창고기반_배정.md` |
| `assign.py` | 전광판 기반 배정 | `03_전광판기반_배정/` |

## 6. MECE 분석

### 6-1. 키워드 라이프사이클 커버리지

| 단계 | 커버 여부 | 담당 |
|------|-----------|------|
| ① 키워드 발굴 (리서치) | ✅ 완전 | main.py |
| ② 키워드 평가 (검색량/전환금액) | ✅ 완전 | main.py + keyword_assign.py |
| ③ 키워드 분류 (전환/잠재) | ✅ 완전 | keyword_assign.py |
| ④ 키워드 배정 (작가 매칭) | ✅ 완전 | keyword_assign.py / assign.py |
| ⑤ 경쟁도 분석 (스마트블록 등) | ✅ 부분 | 배정 시점에만 체크 |
| ⑥ 성과 피드백 (전환 → 리서치) | ⚠️ 간접 | 전환 키워드 탭 존재, 자동 연동 없음 |
| ⑦ 키워드 폐기/교체 | ❌ 없음 | 수동 관리 |

**Gap:** ⑥→① 피드백 루프 미자동화. ⑦ 폐기 프로세스 부재.

### 6-2. 검색 플랫폼 커버리지

| 플랫폼 | 연관검색어 | 검색량 | 순위 체크 |
|--------|-----------|--------|----------|
| Naver | ✅ | ✅ | ✅ |
| Google | ✅ | ❌ | ❌ |
| Daum | ✅ | ❌ | ❌ |

타겟이 네이버 블로그이므로 합리적이지만, Naver API 의존도 100%.

### 6-3. 두 배정 프로그램의 경계

| 구분 | keyword_assign.py | assign.py |
|------|-------------------|-----------|
| 데이터 소스 | 키워드 창고 (자체 관리) | 키워드 전광판 (외부 입력) |
| 분류 기준 | 전환/잠재 비율 (70/30) | 순위/경과일 우선순위 3단계 |
| 시트 | `1_rytQ…` | `1xJAo…` |
| 출력 | 11열 | 8열 |

**Gap:** 언제 어떤 걸 쓸지 사용 기준이 명시되지 않음.

## 7. 종합 평가

### 잘된 점
- 전환/잠재 비율 배정으로 안정성+성장 동시 추구
- 작가-제품 매칭 최적화 (전환 이력 기반)
- 멀티소스 키워드 수집 (Google/Naver/Daum)
- 중복 방지 체계 (norm_keyword + 기간 제한)
- 사용자 개입 포인트 (사전확정, 미리보기, 수동교체)

### 부족한 점
- CLI 전용 — 다른 프로그램과 달리 GUI 없음
- 배정 프로그램 2개 중복 존재, 사용 기준 불명확
- HTML regex 파싱 — 네이버 UI 변경 시 깨짐
- API 키 하드코딩 (보안)
- 네이버 검색 순차 처리 (병렬화 미구현)
- 중간 실패 복구 / 테스트 코드 없음
