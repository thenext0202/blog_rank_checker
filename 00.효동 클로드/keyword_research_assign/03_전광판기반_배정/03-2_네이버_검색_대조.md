# 03-2. 네이버 검색 + 대조

> 담당 파일: `assign.py`

---

## 1. 기능 요약

전광판에서 로드한 키워드를 **우선순위 분류**하고, 네이버 통합검색에서 **스마트블록/인기글/브랜드** 존재 여부를 체크한 뒤, 최적의 작가에게 매칭한다.

---

## 2. 처리 흐름

```
① 우선순위 분류 (prioritize_keywords)
   ├── 1차: 전환금액 > 0 AND 순위 밖 → 전환 실적 있지만 밀려난 키워드
   ├── 2차: 전환금액 > 0 AND 경과일 ≥ 10 → 오래된 전환 키워드
   └── 3차: 순위 밖 (나머지) → 신규/비전환 키워드
   각 우선순위 내: 전환금액 내림차순 정렬
    ↓
② 배정 실행 (assign_to_writers)
   ├── 사전 확정 키워드 먼저 배정
   ├── 1차 → 2차 → 3차 순서대로
   ├── 5일 중복 제외
   └── 건수 부족 시 4일 초과 키워드 재배정 허용
    ↓
③ 작가 매칭 (pick_best_writer)
   점수 = 해당 제품 전환금액
         - 이번 배정 같은 제품 수 × 100억 (분산 페널티)
         + 남은 건수 가산
   → max() 선택
    ↓
④ 네이버 검색 체크 (check_all_keywords → check_naver_keyword)
   각 배정 키워드에 대해:
   ├── 네이버 통합검색 HTML 요청 (0.3초 간격)
   ├── "subjectTitle" regex 추출
   ├── 일반 섹션 필터링 (GENERIC_SECTIONS 제외)
   └── 판별:
       ├── 스마트블록: 일반/FAQ 아닌 섹션 존재 여부
       ├── 인기글: "인기글" 포함 섹션 존재 여부
       └── 브랜드: "브랜드" 포함 섹션 존재 여부
    ↓
⑤ 미리보기 + 수정 (preview_and_confirm)
   ├── 전체 배정 결과 테이블 출력
   ├── 번호 입력 → 해당 키워드 제외
   └── find_next_keyword() → 우선순위 기반 자동 대체
```

---

## 3. 우선순위 분류 기준

| 우선순위 | 조건 | 의미 | 정렬 |
|----------|------|------|------|
| 1차 | 전환금액 > 0 AND 순위 밖 | 돈 벌었는데 밀림 → 복구 급함 | 전환금액↓ |
| 2차 | 전환금액 > 0 AND 경과일 ≥ 10 | 실적 있지만 오래됨 → 갱신 | 전환금액↓ |
| 3차 | 순위 밖 (기타) | 나머지 → 신규 진입 시도 | 전환금액↓ |

---

## 4. 작가 매칭 알고리즘 (pick_best_writer)

```python
score = (해당 제품 전환금액 합계)
      - (이번 배정에서 같은 제품 받은 수 × 10,000,000,000)  # 분산 페널티
      + (남은 quota)
```

- 전환금액이 높은 작가 = 해당 제품 전문가
- 분산 페널티 = 같은 제품 몰아주기 방지
- 남은 quota = 배정 균등화

---

## 5. 네이버 검색 파싱 상세 (check_naver_keyword)

**요청:** `https://search.naver.com/search.naver?where=nexearch&query={keyword}`

**파싱 대상:** `"subjectTitle":"([^"]*)"`

**제외 목록 (GENERIC_SECTIONS):**
```
브랜드 콘텐츠, 이미지, 뉴스, 동영상, 지식iN, 쇼핑,
지도, 장소, 사전, 웹사이트, 카페, 학술정보,
어학사전, 도서, 뮤직, 영화, TV, 플레이스
```

**판별 결과:**

| 항목 | 판별 기준 | 의미 |
|------|-----------|------|
| 스마트블록 | 일반 섹션/FAQ 아닌 타이틀 존재 | 블로그 노출 제한 가능 |
| 인기글 | "인기글" 포함 타이틀 존재 | 인기글 탭 존재 |
| 브랜드 | "브랜드" 포함 타이틀 존재 | 브랜드 콘텐츠 영역 존재 |

---

## 6. 핵심 함수

| 함수 | 역할 |
|------|------|
| `prioritize_keywords(board_kws)` | 3단계 우선순위 분류 |
| `assign_to_writers(priorities, quotas, ...)` | 우선순위별 배정 실행 |
| `pick_best_writer(product, quotas, efficiency, ...)` | 최적 작가 선택 |
| `check_naver_keyword(keyword)` | 네이버 검색 → 스마트블록/인기글/브랜드 |
| `check_all_keywords(assignments)` | 전체 키워드 순차 체크 |
| `preview_and_confirm(assignments)` | 미리보기 + 수동 수정 |
| `find_next_keyword(priorities, used)` | 대체 키워드 자동 추천 |

---

## 7. 평가

**잘된 점:**
- 3단계 우선순위가 비즈니스 로직(전환복구 → 갱신 → 신규)과 정확히 맞음
- 분산 페널티로 특정 제품/작가 쏠림 방지
- 스마트블록/인기글/브랜드 사전 체크 → 경쟁도 파악 후 배정

**부족한 점:**
- 네이버 검색 순차 처리 (0.3초 × N) → 키워드 많으면 느림
- HTML regex 파싱 → 네이버 UI 변경 시 즉시 깨짐
- 분산 페널티 가중치(100억)가 매직넘버 — 조절 불가
- 미리보기 수정이 1개씩만 → 대량 수정 불편
