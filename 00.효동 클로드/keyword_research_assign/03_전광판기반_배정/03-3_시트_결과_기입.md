# 03-3. 시트 결과 기입

> 담당 파일: `assign.py`

---

## 1. 기능 요약

배정 확정된 키워드를 **원고 배정 탭**에 기록한다. 기존 데이터 아래에 append 방식으로 추가한다.

---

## 2. 처리 흐름

```
① 기존 행 수 파악
   └── 원고 배정 탭의 마지막 데이터 행 확인
    ↓
② 각 배정 건별 데이터 구성
   ├── 날짜: YYYY-MM-DD (datetime.now())
   ├── 작가명
   ├── 제품명
   ├── 키워드
   ├── 배정 사유 (우선순위 + 전환금액 정보)
   ├── 스마트블록 (O/X) — 03-2에서 체크한 결과
   ├── 인기글 (O/X)
   └── 브랜드 (O/X)
    ↓
③ 시트에 batch update
   └── 기존 마지막 행 + 1부터 연속 기록
```

---

## 3. 출력 시트 구조 — 원고 배정 탭 (8열)

| 열 | 내용 | 예시 |
|----|------|------|
| A | 날짜 | 2026-03-19 |
| B | 작가명 | 남경아 |
| C | 제품명 | 블러드싸이클 |
| D | 키워드 | 고혈압 낮추는 방법 |
| E | 배정 사유 | 1차(전환+순위밖) 평균 45,000원 |
| F | 스마트블록 | O / X |
| G | 인기글 | O / X |
| H | 브랜드 | O / X |

---

## 4. 배정 사유 생성 규칙

| 우선순위 | 사유 형식 |
|----------|-----------|
| 1차 | `1차(전환+순위밖) 평균 {금액}원` |
| 2차 | `2차(전환+경과{일}일) 평균 {금액}원` |
| 3차 | `3차(순위밖)` |
| 사전확정 | `사전확정` |

## 6. 시트 기록 방식

```python
# 기존 데이터 끝 다음 행부터 기록
start_row = len(ws.get_all_values()) + 1
range_str = f"A{start_row}:H{start_row + len(assignments) - 1}"
ws.update(range_str, rows)
```

- **append 방식**: 기존 데이터를 건드리지 않음
- **batch update**: 한 번의 API 호출로 전체 기록

---

## 7. 평가

**잘된 점:**
- append 방식으로 기존 데이터 안전 보존
- batch update로 API 호출 최소화
- 스마트블록/인기글/브랜드 체크 결과를 함께 기록 → 사후 분석 가능

**부족한 점:**
- 기록 실패 시 복구/재시도 없음 (네트워크 오류 시 데이터 유실)
- 기록 후 검증 없음 (실제 기록된 데이터 vs 의도된 데이터 대조 없음)
- 날짜 형식이 시트 포맷에 의존 — 시트 설정에 따라 표시 차이 가능
