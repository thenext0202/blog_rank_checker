# 03-1. 시트 키워드 읽기

> 담당 파일: `assign.py`
> 시트: `1xJAogt0alaQ8A5OctxltPaF3kg_0PFSF5Z0ePxMw3tY`

---

## 1. 기능 요약

키워드 전광판 시트에서 키워드 목록을 로드하고, 작가별 quota와 전환 이력을 수집하여 배정 준비 데이터를 구성한다.

---

## 2. 처리 흐름

```
① 키워드 전광판 탭 읽기 (load_board_keywords)
   ├── Row 0: 날짜 헤더
   ├── Row 1: 컬럼 헤더
   └── Row 2+: 데이터
    ↓
② 원고 작성 건수 탭 → 오늘 작가별 quota (get_today_quotas)
   └── M/D 형식 날짜 컬럼 매칭
    ↓
③ 최대 배정 건수 적용 (apply_max_quota)
   └── 시트 건수 < max_count → 시트 건수 유지
    ↓
④ 전환 키워드 탭 → 작가별 제품 효율 분석 (get_writer_product_efficiency)
   └── {작가: {제품: {amount, count}}}, {작가: [효율 순 제품]}
    ↓
⑤ 최근 배정 이력 조회 (get_recent_assignments)
   ├── 최근 5일: 중복 제외용
   └── 최근 4일: 재배정 기준
    ↓
⑥ 사전 확정 키워드 입력 (get_pre_assignments)
   └── 형식: "작가명, 제품, 키워드"
```

---

## 3. 키워드 전광판 탭 구조

| 열 | 내용 | 파싱 방법 |
|----|------|-----------|
| A | 제품명 | 문자열 |
| B | 키워드 | 문자열 |
| C | 기존 전환금액 | `parse_amount()` → 숫자 |
| D | 순위 | "순위 밖" 판별 |
| E-H | 추가 데이터 | 컬럼별 |
| I | 연속일(경과일) | `parse_days()` → 숫자 |

---

## 4. 전환 키워드 탭 구조 (5열 단위 반복)

```
[작가명 헤더]
키워드 | 파라미터 | 제품 | 금액 | (빈열)
키워드 | 파라미터 | 제품 | 금액 | (빈열)
...
[다음 작가명 헤더]
```

---

## 5. 핵심 함수

| 함수 | 입력 | 출력 |
|------|------|------|
| `load_board_keywords()` | 전광판 시트 | [{제품, 키워드, 전환금액, 순위, 경과일}] |
| `get_today_quotas()` | 작성건수 시트 | {작가: 건수} |
| `apply_max_quota(quotas, max)` | quota dict | {작가: min(건수, max)} |
| `get_writer_product_efficiency()` | 전환 키워드 시트 | 작가별 제품 효율 dict |
| `get_recent_assignments()` | 배정 시트 | (5일 set, 4일 set) |
| `get_pre_assignments()` | CLI 입력 | [(작가, 제품, 키워드)] |

---

## 6. 평가

**잘된 점:**
- 전광판 데이터를 직접 활용 → 순위/경과일 등 실시간 상태 반영
- 작가별 제품 효율을 전환금액+건수로 정량 분석
- 중복 방지 기간(5일)과 재배정 기준(4일)을 분리

**부족한 점:**
- 전광판 시트 구조(Row 0/1/2 하드코딩)가 변경되면 파싱 깨짐
- 전환 키워드 탭의 5열 단위 구조가 복잡 — 작가 헤더 위치 파싱에 의존
