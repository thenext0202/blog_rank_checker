## 🏗️ 아키텍처 패턴 (9건)

### 1. 2단계 최적화 패턴 (가벼운 감지 → 무거운 처리)

> **대표 사례:** 댓글 알림 v2.0+

| 지표 | Before | After |
|------|--------|-------|
| 첫 실행 | 100분+ | ~8초 |
| 변동 없을 때 | 100분+ | ~2초 |

- 1단계: API/HTTP로 변동 감지 → 2단계: 변동 있는 것만 Selenium
- **적용 가능:** 대량 모니터링, 순위 체크, 변동 감지 전반

### 2. 시트 기반 설정 관리

- Google Sheets에 프롬프트/스타일/지침/제품 데이터 관리
- 코드 수정 없이 비개발자가 직접 수정 가능 — 원고 제작기 8개 탭이 대표

### 3. JSON 영구 저장 패턴

- `config.json`, `exclude_keywords.json`, `manual_pass.json` 등 EXE 옆에 저장
- 폴더 압축 전달 시 설정도 함께 전달

### 4. EXE 배포 + 경로 자동 감지

```python
if getattr(sys, 'frozen', False):
    BASE_DIR = os.path.dirname(sys.executable)
else:
    BASE_DIR = os.path.dirname(os.path.abspath(__file__))
```

- EXE 옆 `credentials.json` 우선 → 없으면 기존 경로 fallback

### 5. 모듈 분리 설계 (이미지 원고 제작기)

- 기존 원고 제작기 3,482줄 단일 파일의 교훈
- 6개 모듈: `main.py`, `lib_common.py`, `image_selector.py`, `image_metadata.py`, `catalog_images.py`, `substitution.py`
- **교훈:** 새 프로그램은 처음부터 모듈 분리

### 6. 파라미터 조인 패턴 (발행 검수)

- 자사 발행리스트(H열) + 원고리스트(J열)를 파라미터 키로 자동 조인
- 서로 다른 시트 탭의 데이터를 하나의 키로 연결

### 7. 환경변수 우선 + 파일 fallback 인증

- `GOOGLE_CREDENTIALS_BASE64` 환경변수 우선, 없으면 credentials.json
- 서버/로컬 양쪽 배포 가능

### 8. 단일 파일 CLI 패턴

- 매출 업로더, 댓글 검수, MKT 링크 대조 등 — 단일 .py + bat 실행
- **교훈:** 간단한 자동화는 GUI 없이 CLI가 더 실용적

### 9. 옵시디언 MD 변환기 — graceful degradation

- 20+ 확장자를 마크다운으로 변환
- 미설치 패키지는 안내 메시지만 표시, 프로그램은 정상 동작

## 🔍 검수/매칭 패턴 (15건)

### 10. 링크 매칭 3단계 (발행 검수 v1.5+)

```
① 전체 URL 비교 (percent-decode + 쿼리 정렬)
   ↓ 실패 시
② MKT ID만 비교 (/link/ABC123)
   ↓ 실패 시
③ 최종 리다이렉트 URL 비교
```

### 11. `data-linkdata` 원본 URL 추출 (발행 검수 v1.6)

- 네이버 JS가 `a.href` 수정 → `data-linkdata` 속성에서 원본 MKT URL 우선 추출
- **교훈:** 네이버 DOM에서는 `a.href`보다 `data-*` 속성이 신뢰도 높음

### 12. 10자 부분 일치 매칭 (댓글 검수)

- 슬라이딩 윈도우 + 공백 전부 제거 후 비교
- 띄어쓰기/특수문자 차이도 커버

### 13. 독립 if/if 구조 vs if/elif

- 상품/광고가 같은 MKT ID 공유 시 `if/elif`는 하나만 잡힘
- **독립 `if/if`로 둘 다 매칭** — 검수에서 "하나만 잡히면 나머지 스킵" 패턴 주의

### 14. URL 정규화 체계

- percent-decode + 쿼리 파라미터 정렬 + scheme/trailing slash 제거 + 소문자
- `_mkt_full()`, `_mkt_id()`, `_normalize_url()` 3단계
- **교훈:** URL 비교 시 정규화 없이 직접 비교하면 실패

### 15. 텍스트 정규화 후 비교

- 유니코드 따옴표→ASCII, 화살표 제거, 공백 정리, 소문자 변환
- 블로그/시트 간 문자 차이에 의한 오탐 방지

### 16. 서식 검수 점수 기반 매칭

- DOCX format_reqs와 블로그 format_info를 첫 20자 텍스트 매칭
- 인용구/글자크기/볼드/색상 각각 대조, RGB 범위 매칭으로 미세 색상 차이 허용

### 17. 지시사항 특성 기반 파싱 (v1.8)

- `in_req` 모드에서 `-`, `ㄴ` 시작, INSTRUCTION_KEYWORDS, `부탁`/`요청` 등 특성 줄만 캡처
- 특성 없으면 즉시 모드 종료 → **무분별한 캡처가 오탐의 원인**

### 18. 중복 보고 방지 패턴 (v1.9)

- `req_seen` set으로 동일 요청문구 중복 방지
- `req_exposed` 리스트로 요청문구↔서식키워드 겹침 방지
- **교훈:** 검수 로직이 여러 경로에서 같은 문제를 잡을 때 중복 필터 필요

### 19. 47개 INSTRUCTION_KEYWORDS 분류

- 글씨, 폰트, 색, 볼드, 형광, 인용, 정렬 등 편집자 지시 키워드 47개
- ㄴ지시사항/서식을 정확히 분류

### 20. 2단계 파일 검색 + 날짜 패턴 제거

- Drive 검색 1차: 전체 파일명 → 2차: `_strip_pub_date()`로 날짜 패턴 제거 후 재검색
- 파일명 변형 패턴에 대응

### 21. 전환/잠재 비율 배정 (키워드 배정)

- 전환 70% + 잠재 30% — 안정성(검증된 전환) + 성장(신규 잠재) 동시 추구

### 22. 작가-제품 효율 기반 매칭

- 작가별 제품 전환 이력 분석 → 전문 분야 우선 배정
- 분산 페널티(100억)로 같은 제품 쏠림 방지

### 23. 3단계 우선순위 분류 (키워드 배정)

- 1차: 전환실적+순위밖(복구 급함) → 2차: 전환실적+경과10일(갱신) → 3차: 순위밖(신규)

### 24. 전환점수 115점 체계

- 검색량(30점) + 구매의도키워드(50점) + 카테고리매칭(15점) + 전환금액(20점)
- 잠재 키워드의 정량 평가

## ⚡ 병렬 처리 패턴 (3건)

### 25. Selenium N개 병렬

```python
with ThreadPoolExecutor(max_workers=N) as executor:
    futures = [executor.submit(process_chunk, chunk) for chunk in chunks]
```

- 발행 검수(4개), 댓글 검수(4개), 원고 배정 검수(8개)
- `Lock`으로 카운터 공유, `root.after()`로 tkinter 프로그레스 업데이트
- 스레드에서 직접 UI 접근 금지

### 26. URL 캐시 + 고유 URL 추출 (댓글 검수)

- 동일 URL이 여러 블록에 있어도 1번만 스크래핑, 캐시에서 재사용

### 27. 10개마다 드라이버 자동 재시작

- Chrome 메모리 누수 방지 + `page_load_timeout(15)` 함께 적용

## 📝 프롬프트 엔지니어링 — 원고 제작기 (15건)

### 55. 14섹션 프롬프트 체계

- 템플릿→샘플→스타일→톤→소구점→목차규칙→공통지침→키워드→연관키워드→해시태그→글자수→줄끊기→ㄴ배치→이미지→참고자료→인용법→서식→페르소나→제목→추가지시→체크리스트→자연스러움→유형별재강조
- 빈틈 없는 지시로 일관된 품질

### 56. Few-shot 샘플 매칭

- 같은 유형+같은 제품 우선, 같은 유형+다른 제품 fallback
- 4,000자 제한으로 토큰 비용 관리 — 톤/구조/서식/흐름 모방 유도

### 57. 공통지침 앞+뒤 배치

## 🌐 Selenium/스크래핑 베스트 프랙티스 (8건)

### 70. 탐지 방지 설정 세트

```python
options.add_argument('--headless=new')

```

### 71. mainFrame iframe 전환 필수

- 네이버 블로그 = iframe 구조 → `switch_to.frame('mainFrame')` 필수
- 없으면 콘텐츠 접근 불가

### 72. 더보기 버튼 반복 클릭

- `expand_all_comments()`: 더보기 버튼 최대 30회 반복 클릭으로 모든 댓글 로드

### 73. find_elements vs find_element

- `find_element`는 implicit wait 병목 유발
- `find_elements` → 요소 없으면 빈 리스트 반환, 대기 없음

### 74. 삭제된 게시글 alert 처리

- `switch_to.alert.accept()` 먼저 처리 → JavaScript alert 에러 방지

### 75. --single-process 제거

- 오히려 불안정 유발
- 대신 `--disable-background-networking`, `--disable-renderer-backgrounding` 추가

### 76. PostView URL 변환 패턴

```
blog.naver.com/username/123456
```
