# 🔧 이미지번호 오인식 수정 & 기인정 기능

> **날짜**: 2026-03-24
> **프로그램**: 원고 배정 프로그램 v1.4+

---

<br>

## 📌 변경 요약

| 항목 | 내용 |
|:-----|:-----|
| 이미지 번호 오인식 수정 | 원고 본문의 큰 숫자(연도, 수치 등)를 이미지 번호로 잘못 인식하는 문제 해결 |
| 기인정 기능 | 수동 통과 처리한 원고를 영구 저장 → 다음 실행 시 자동 통과("기인정") |

---

<br>

## 🎯 사용 방법

### 1. 이미지 번호 오인식 — 수정 후 자동 적용
- 별도 조작 없음. **30 이하 숫자만** 이미지 번호로 인식합니다.
- 원고에 "2024", "100" 등 큰 숫자가 있어도 더 이상 이미지 번호로 잡히지 않습니다.

### 2. 기인정 — 수동 통과 영구 저장
- Step 2에서 **"선택 항목 검수 통과 처리"** 버튼을 누르면 기존처럼 통과 처리됩니다.
- 이때 해당 폴더명이 `manual_pass.json`에 **자동 저장**됩니다.
- 다음에 프로그램을 다시 실행하면, 저장된 원고는 검수 실패여도 **"기인정"**으로 자동 통과됩니다.

### 검수 열 상태값 구분

| 상태 | 의미 |
|:-----|:-----|
| **통과** | 검수 자동 통과 (문제 없음) |
| **통과(수동)** | 이번에 수동으로 통과 처리 |
| **기인정** | 이전에 수동 통과 저장된 원고 (자동 복원) |
| **실패** | 검수 실패 (수동 통과 가능) |

## ⚙️ 구현 방법

### 이미지 번호 30 제한
- `review_manuscript()` 함수에서 문단 숫자를 이미지 번호로 판별할 때 `int(text) <= 30` 조건 추가
- 단독 숫자(`^\d+$`)와 나열 숫자(`^\d+[\s,、]+\d+`) 모두 동일하게 제한

### 기인정 영구 저장
- **저장 파일**: `manual_pass.json` (EXE와 같은 폴더에 자동 생성)
- **저장 시점**: `_manual_pass()` 실행 시 폴더명을 JSON에 추가
- **복원 시점**: Step 2 검수 완료 후, 검수 실패 + JSON에 존재 → `passed=True`, `pre_approved=True` 설정
- **표시**: `_display_step2()`에서 `pre_approved` 플래그로 "기인정" 표시

```json
// manual_pass.json 예시
[
  "bc_홍길동_블러드싸이클_원고",
  "gc_김철수_혈당컷_원고"
]
```

## ✅ 잘된 점
- 이미지 번호 30 제한은 간단하면서 실용적 — 이미지 30장 초과 원고는 거의 없음
- `exclude_keywords.json`과 동일한 패턴으로 구현 → 일관성 있음
- 폴더 압축 전달 시 `manual_pass.json`도 함께 전달되어 팀 공유 가능

---

<br>

## ⚠️ 아쉬운 점
- `manual_pass.json`에 배정 완료된 폴더명이 계속 쌓임 (성능 영향은 없지만 정리 필요 시 수동 요청)
- 이미지 30장 초과 원고가 만약 있다면 상한값 수정 필요
