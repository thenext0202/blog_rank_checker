# 🚀 자동 연쇄 실행 기능 가이드

> **적용 버전**: v1.4+ (2026-03-20)
> **목적**: 불필요한 클릭을 줄여 배정 작업을 빠르게 완료

---

<br>

## 📋 변경 요약

| 단계 | 이전 (수동) | 이후 (자동) |
|:----:|:------------|:------------|
| **Step 1** | "시트에 기입" 클릭 → "발행리스트 불러오기" 클릭 | "시트에 기입" 클릭 **한 번이면 끝** |
| **Step 2** | 화면 진입 → "매칭+검수 실행" 클릭 | 화면 진입하면 **자동 실행** |
| **Step 4** | "배정 실행" → "시트 파라미터 업데이트" → "키워드 시트 기입" (3번 클릭) | "배정 실행" 클릭 **한 번이면 끝** |

> 💡 **총 6번 클릭 → 3번으로 절반 감소!**

## 🔧 사용 방법

### Step 1. 배정 건수 설정
1. 업체별 건수 입력
2. **"시트에 기입"** 클릭
3. ✅ 시트 기입 완료 → **자동으로** 발행리스트 불러오기 실행 (0.5초 후)

### Step 2. 원고 매칭 + 검수
1. **"다음 ▶"** 버튼으로 Step 2 진입
2. ✅ **자동으로** 매칭+검수 실행 (진입 즉시)
3. 필요 시 "매칭+검수 실행" 버튼으로 재실행 가능

### Step 4. 배정 실행
1. **"배정 실행"** 클릭
2. ✅ 파일 복사/ZIP/삭제 완료 후 **자동으로**:
   - 시트 파라미터 업데이트
   - 3초 대기 (수식 반영)
   - 키워드 시트 기입
3. 로그에서 각 단계 진행 상황 확인 가능

## 🔧 구현 방식

| 기능 | 구현 로직 |
|:-----|:---------|
| Step 1 자동 불러오기 | `_write_plan_to_sheet` worker 완료 시 `root.after(500)` → `_load_publish_data` 호출 |
| Step 2 자동 매칭 | `_build_step2` 마지막에 `root.after(100)` → `_match_and_review` 호출 |
| Step 4 자동 시트 기입 | `_execute_assignment` worker 완료 시 `root.after(500)` → `_auto_update_sheets` 호출 |
| Step 4 키워드 대기 | `_auto_update_sheets` 내부에서 파라미터 기입 후 `time.sleep(3)` → 키워드 시트 기입 |

> **핵심**: `_auto_update_sheets`는 확인 팝업 없이 실행되는 전용 메서드.
> 기존 `_update_sheet_dates`, `_update_keyword_sheet`는 수동 버튼용으로 유지 (확인 팝업 있음).

## ✅ 잘된 점

- **클릭 수 절반 감소** — 반복 작업에서 체감 효과 큼
- **기존 수동 버튼 유지** — 자동 실행 실패 시 개별 재실행 가능
- **수식 반영 대기** — 키워드 시트 기입 전 3초 대기로 시트 수식 안정성 확보
- **진행 상황 표시** — `exec_progress_var`로 현재 단계 안내

---

<br>

## ⚠️ 아쉬운 점

- **수식 반영 대기 시간 고정** — 3초가 항상 충분한지 보장 어려움 (시트 부하에 따라 다를 수 있음)
- **Step 1 기존 데이터 경고 시** — 경고 팝업 후에도 발행리스트 불러오기는 실행 안 됨 (의도된 동작이나, 사용자가 혼동할 수 있음)
