# 🖼️ 이미지 변형 기능 (manuscript_generator_v2)

> 📅 2026-03-25 | 원고 생성 후 이미지를 AI로 변형하는 기능

---

## 📖 사용 방법 가이드

### 1단계: OpenAI API Key 설정
```
설정 탭 → "OpenAI API Key (이미지 변형)" → 키 입력 → 저장
```

### 2단계: 원고 생성 (기존과 동일)
```
제품/키워드/유형 설정 → 이미지 사용 체크 → 이미지 자동 선택 → 원고 생성
```

### 3단계: 이미지 검토
```
원고 생성 완료 → [이미지 검토] 버튼 클릭
```

### 4단계: 검토 다이얼로그에서
| 영역 | 설명 |
|------|------|
| **이미지 리스트** | 각 이미지의 썸네일 + 적합/불일치 뱃지 + AI 제안 |
| **추가 지시란** | 사용자가 직접 프롬프트 입력 |
| **[이미지 생성/변형]** | 불일치 이미지를 AI로 변형 |
| **[Word 저장 (이미지 포함)]** | 변형된 이미지가 포함된 Word 저장 |

### 5단계: 추가 지시 예시
```
"3개월차 컷과 1개월차 컷을 다시 만들어줘"
"관리 전 이마/볼 클로즈업 컷 2장 추가"
"복용하는 장면도 필요해"
"여자가 체중계 재는 모습, 화면에는 54KG"
```

## 🔧 구현 방법 설명

### 전체 흐름
```
원고 생성 완료
    ↓
[이미지 검토] 클릭
    ↓
Claude가 원고 맥락(이미지 앞뒤 텍스트) vs 이미지 메타데이터 비교
    → 적합/불일치 판정 + 변형 제안
    ↓
사용자 확인 + 추가 지시 입력
    ↓
OpenAI gpt-image-1 API 호출
    → 참조 이미지(Drive) + 변형 프롬프트 → 새 이미지 생성
    ↓
Word 저장 시 이미지 임베드
```

### 핵심 모듈

| 파일 | 역할 |
|------|------|
| `image_transformer.py` | 이미지 변형 엔진 (신규) |
| `main.py` | GUI + save_as_docx 수정 |

### image_transformer.py 주요 함수

| 함수 | 역할 |
|------|------|
| `extract_image_contexts()` | 원고에서 이미지 번호 앞뒤 150자 맥락 추출 |
| `analyze_image_fit()` | Claude로 적합도 판정 (fit/reason/suggestion) |
| `transform_image()` | OpenAI로 참조 이미지 변형 |
| `generate_image()` | 참조 없이 새 이미지 생성 |
| `build_transform_prompt()` | 원고 맥락 + AI 제안 + 사용자 지시 → 프롬프트 조합 |

### 비용 (OpenAI gpt-image-1)

| 품질 | 1장당 | 10장 |
|------|-------|------|
| medium (기본) | ~$0.04 | ~$0.40 |
| low | ~$0.01 | ~$0.10 |

## ✅ 잘된 점

1. **기존 원고 제작기를 복사본(v2)으로 작업** — 원본 보호
2. **자동 + 수동 이중 구조** — AI 자동 분석으로 불일치 발견 + 사용자가 직접 지시 가능
3. **기존 코드 최소 수정** — `image_transformer.py`에 핵심 로직 분리, main.py는 GUI + 연결만
4. **참조 이미지 기반 변형** — Drive에서 기존 이미지 다운 → 스타일 유지하면서 변형
5. **HAS_TRANSFORMER 플래그** — import 실패해도 기존 기능 정상 동작

## ⚠️ 아쉬운 점 / 주의사항

1. **아직 테스트 미완료** — OpenAI API 실제 호출 테스트 필요
2. **이미지 품질 불확실** — AI 생성 이미지가 블로그 퀄리티에 맞는지 실사용 확인 필요
3. **한글 텍스트 렌더링** — AI 이미지에 한글 숫자(54KG 등) 정확도 제한적일 수 있음
4. **비용 누적** — 이미지 변형마다 OpenAI API 비용 발생
5. **Drive 연결 필요** — 참조 이미지 다운로드에 Drive 서비스 필요 (미연결 시 새 이미지만 생성)
