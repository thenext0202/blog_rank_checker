## 💡 이 프로그램은 뭐야?

기존 원고 제작기는 **원고를 먼저 쓰고 이미지 번호를 넣는 방식**이었는데,
이 프로그램은 **이미지를 먼저 골라놓고, 그 이미지에 맞는 원고를 쓰는 방식**이야.

> 이미지를 보고 글을 쓰는 게 AI가 훨씬 잘하거든.

---

## 🚀 사용 순서

### 1단계: 이미지 준비 (1회만)

```
이미지_정리/이미지_라이브러리/ 폴더를 구글 드라이브에 업로드
    ↓
python catalog_images.py --drive-folder-id <ID> --sheet-id <ID> --tag
    ↓
시트에 "이미지 메타데이터" 탭 생성 → AI가 각 이미지 분석
```

### 2단계: 프로그램 실행

```bash
python main.py
```

### 3단계: 원고 생성

| 순서 | 할 일 |
|------|-------|
| ① | **설정 탭**에서 API Key + Sheet ID 입력 → 저장+연결 |
| ② | **신규 생성 탭**에서 제품/키워드/유형 선택 |
| ③ | **[자동 선택]** 클릭 → 이미지 프리뷰 확인 |
| ④ | 필요하면 이미지 교체 (클릭 → 검색 → 선택) |
| ⑤ | **[▶ 생성]** 클릭 → 페르소나 → 제목 → 원고 자동 생성 |
| ⑥ | 결과 확인 → Word 저장 |

### 치환 모드

| 순서 | 할 일 |
|------|-------|
| ① | **치환 모드 탭**에서 원본 원고 불러오기 (DOCX/텍스트) |
| ② | 대상 제품 + 키워드 입력 |
| ③ | **[자동 매칭]** → 이미지 매칭 확인 |
| ④ | **[▶ 치환 생성]** → 완료 |

## 🧩 프로그램 파일 설명

| 파일 | 한 줄 설명 |
|------|-----------|
| `main.py` | GUI 앱 — 신규생성/치환/설정/이력 4탭 |
| `lib_common.py` | 기존 원고 제작기에서 뽑은 공용 함수 (시트/Drive/API/서식) |
| `image_selector.py` | 이미지 자동 선택 — 후킹→제품컷→AI추천→폴백 |
| `image_metadata.py` | 시트 메타데이터 로딩 + 썸네일 캐시 |
| `catalog_images.py` | Drive 이미지 스캔 → 시트 등록 + AI 태깅 |
| `substitution.py` | 치환 모드 — 원본 파싱→장면추론→매칭→생성 |
