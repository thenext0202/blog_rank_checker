# ㄴ 주석 표준 문법 (manuscript_web v2.1~)

> 일주일간 버그의 뿌리였던 "같은 주석이 여러 형식으로 공존"을 단일 표준으로 통합.
> 2026-04-24 확정.

---

## 🎯 결정: 표준은 `ㄴ '단어' 서식명`

```
ㄴ '홍국' 노란 형광펜, 볼드      ← 이 형식으로 통일
ㄴ '블러디션 배합' 검정 형광펜, 볼드
ㄴ '모나콜린K' 파란색
```

대신 이 형식은 **사라짐:**

```
ㄴ 노란 형광펜, 볼드 — "홍국"    ← ❌ 더 이상 생성 안 함
ㄴ 검정 형광펜, 볼드 — "블러디션 배합"  ← ❌
```

---

## 🧭 왜 이 형식인가

1. **편집창 UI가 이미 이 형식을 만들고 있음.** 버튼 눌러 생성되는 모양이 표준이 되는 게 가장 자연스러움.
2. **사람이 읽기 쉬움.** `— "단어"`는 em-dash(`—`) 입력이 불편하고 낯선 기호.
3. **수정 범위 최소.** `docx_formatter.py`의 `color_names` 리스트에 `'하늘색', '노란색'` 2개만 추가하면 모든 색상이 이 형식으로 동작.

---

## 📋 지원 서식

### 색상 (글자색)
`빨간색, 파란색, 청록색, 초록색, 보라색, 주황색, 회색, 하늘색, 노란색`
+ 음영: `옅은 회색, 많이 옅은 회색, 진한 회색`

### 형광펜
`노란 형광펜, 하늘 형광펜, 검정 형광펜, 파란 형광펜, 빨간 형광펜, 초록 형광펜, 청록 형광펜`

### 스타일
`볼드, 밑줄, 이탤릭`

### 인용구
`인용구 1번` ~ `인용구 6번` (자동으로 19pt + 볼드)

### 글자 크기
`글자 크기 11` / `12` / `13` / `15` / `16` / `19` / `24` / `28`

---

## 🧩 주석 구조

```
ㄴ ['단어']? 서식1[, 서식2, 서식3...]
```

- **`'단어'` 있음** → 그 단어에만 서식 적용
- **`'단어'` 없음** → 해당 본문 줄에 적용 (엔진 수정 후, 윗줄 흡수 금지)

### 예시

| 목적 | 표준 주석 |
|---|---|
| "홍국"만 노란 형광펜+볼드 | `ㄴ '홍국' 노란 형광펜, 볼드` |
| 이 줄 전체 파란색 | `ㄴ 파란색` |
| 이 줄 인용구 1번 | `ㄴ 인용구 1번, 글자 크기 19, 볼드` |
| 링크 (단어에) | `ㄴ '자세히' 링크 도구로 삽입` |

## 🚧 레거시 원고 호환

기존 원고에 `— "단어"` 형식이 남아 있을 수 있습니다.
파이프라인 진입 시 **자동 변환**합니다 (사용자 수동 작업 불필요):

```
ㄴ 노란 형광펜 — "홍국"   →   ㄴ '홍국' 노란 형광펜
```

변환은 `output_parser.parse()` 진입 직후 한 번만 수행.
