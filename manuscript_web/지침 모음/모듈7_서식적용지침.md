# [참조 문서 6] 블로그 원고 서식 적용 지침

---

## 0. 대원칙: 원고는 수정하지 않는다

이 지침은 **서식 적용 지침**이다. 원고의 텍스트는 한 글자도 바꾸지 않는다.

### 허용: 서식 작업만
- 줄바꿈, 빈 줄 추가
- 색상·볼드·형광펜·밑줄·글자크기 적용
- 편집 지시(초록 형광펜) 삽입

### 금지: 원고 내용 변경
- 원고에 없는 목차·대제목·소제목을 만들어 넣지 않는다
- 원고의 문장·어미·표현을 바꾸지 않는다

### 원고에 특정 구조가 없을 때
서식 규칙을 적용하려고 없는 구조를 만들지 않는다.
예: 원고에 소제목이 없으면 "소제목 아래 초록 지시" 규칙은 적용 대상이 없는 것이다.

## 0-2. 마크다운 헤더 처리

원고에 포함된 마크다운 헤더 기호(`##`)는 **위계 식별용 메타 정보**이며, 최종 출력물에는 **반드시 제거**한다.

### 처리 절차
1. `## 소제목` → 기호(`## `) 제거 → 소제목 텍스트만 남김 → 해당 위계에 맞는 서식(볼드 + 지정 글자 크기) 적용 → 아래에 초록 지시 삽입
2. 글자 크기는 §4 및 §8의 규칙에 따른다 (19pt 또는 24pt 등, 상황에 따라 결정한다 — 고정값 아님)
3. 기호 제거는 **텍스트 변경이 아닌 서식 변환**으로 간주한다 (§0 "원고 수정 금지" 원칙과 충돌하지 않음)

### 주의
- 대제목은 본문에 등장하지 않고 항상 문서 최상단 `제목 : ~~` 형식으로 별도 배치되므로 `#` 헤더 기호는 사용하지 않는다
- 최종 원고에 `##` 기호가 한 글자라도 남아 있으면 안 된다
- 기호 제거 후 남은 텍스트는 원고와 **문장 단위로 완전히 일치**해야 한다

## 1. 문서 최상단 구조

아래 순서대로 최상단에 배치한다.

```
★블로거 요청사항★  ← 빨간색(FF0000), 볼드
- 글자크기 16/ 중간 정렬/ 줄 간격 210%
- 링크는 배너 삽입 부탁드려요!
- 인용구, 스티커, 글자크기, 강조 등 두 번 확인 부탁드려요!
- 이미지와 글 사이는 한 칸씩 띄워주세요!
───────────────── (구분선)


### 블로거 요청사항 블록 색상 규칙
- **"★블로거 요청사항★" 제목 줄에만** 빨간색(FF0000) + 볼드 적용
- 아래 4개 항목("- 글자크기 11/...", "- 링크는 배너...", 등)은 **검정 + 일반** (볼드 없음)

제목 : [제목 텍스트]  ← 일반(볼드 없음)

## 2. 줄바꿈 규칙

### 기본 원칙
- 한 줄에 **12~25자** 내외로 작성 (모바일 가독성 기준)
- **호흡 단위**에서 줄을 끊는다: 쉼표(,), 연결어미(-고, -서, -는데, -면서, -지만 등)
- **의미 단락이 전환되는 곳**에 빈 줄을 넣는다
- **한 문단은 최대 5줄**까지만 붙인다. 5줄을 넘으면 의미 단위에서 끊어 빈 줄을 넣는다

### 글자 수 vs 의미 단위 — 충돌 시 우선순위
12~25자는 **목표**이지 **절대 기준**이 아니다.
의미 단위가 25자를 조금 넘더라도 쪼개지 않고 한 줄에 둔다.

우선순위:
1순위: 의미 덩어리 보존 (조사·서술어 토막 금지)
2순위: 호흡 단위 (쉼표, 연결어미)
3순위: 12~25자 목표

### 금지 사항
- **2~5자짜리 토막이 한 줄에 혼자 남지 않게 한다**
  - ✗ "거기에" (3자) 단독 줄
  - ✗ "답니다." (4자) 단독 줄
  - ✗ "으로" (2자) 단독 줄
- 글자 수를 맞추려다 의미 덩어리를 쪼개지 않는다
- 짧은 토막이 생기면 앞줄이나 뒷줄에 합쳐서 의미가 이어지게 한다

### 짧은 줄 합치기 원칙 (매우 중요)
한 줄이 **10자 이하**면 원칙적으로 앞/뒷줄과 합쳐서 **12~25자 범위**로 만든다.
LLM은 호흡 단위(쉼표·연결어미)에서 기계적으로 끊는 경향이 있어, 결과적으로 한 문장을 3~5줄로 잘게 쪼개는 경우가 많다. 이 패턴이 누적되면 모바일에서 **스크롤만 길고 가독성이 떨어지는** 원고가 된다.

✗ 잘못된 예 (8자·12자·7자로 잘게 쪼갬):
```
솔직히 검진 결과지
열어보고
한동안 멍했다.
```

✓ 올바른 예 (14자·7자로 한 번만 끊기, 여전히 5줄 이내):
```
솔직히 검진 결과지 열어보고
한동안 멍했다.
```

또는 한 줄로도 충분 (21자):
```
솔직히 검진 결과지 열어보고 한동안 멍했다.
```

**합치기 원칙의 예외 (이런 경우에만 10자 이하 단독 줄 허용):**
1. **숫자·수치 독립 줄** — "공복혈당 108." 같이 강조 의도로 숫자를 독립시킨 경우
2. **단독 완결 감탄/강조 문장** — "그게 더 무섭더라." 같이 문단 마지막에서 여운을 남기는 짧은 문장
3. **원문 자체가 10자 이하로 완결** — 더 붙일 문맥이 없는 경우

이 3가지 예외가 **아니면** 반드시 앞/뒷줄과 합친다.

### 구체 금지 패턴
아래 패턴이 나오면 **무조건** 앞/뒷줄과 합친다:
- 조사 단독 줄: "에 많고,", "으로", "에서", "이라서"
- 짧은 서술어 단독 줄: "했어요.", "있어요.", "돼요.", "이에요."
- 연결어미 토막 단독 줄: "하고", "지만", "는데"

### 예시
```
✗ 잘못된 줄바꿈:
거기에
혈관 자체가 낡아지면
탄력까지 떨어진다
고
하더라고요.

✓ 올바른 줄바꿈:
거기에 혈관 자체가 낡아지면
탄력까지 떨어진다고 하더라고요.
```

```
✗ 잘못된 분할 (조사 단독 줄):
칼슘은
우유, 치즈, 멸치, 두부
에 많고,
  → "에 많고,"가 조사 단독 줄 (금지)

✓ 올바른 분할:
칼슘은 우유, 치즈,
멸치, 두부에 많고,
  → 쉼표 단위로 끊음, 조사가 앞 단어와 붙음
```

### 논문 인용 줄바꿈 규칙
논문 제목은 대부분 영문이라 한글 호흡 단위(쉼표·연결어미)가 적용되지 않는다. 그렇다고 한 줄로 길게 두면 본문의 다른 줄과 길이 차이가 커서 이상해 보인다. 아래 기준으로 끊는다.

- **쉼표(,) 기준으로 끊는다** — 논문 제목 안에 쉼표가 있으면 그 지점에서 줄바꿈
- **괄호 안 출처(저널명·연도)는 별도 줄로** 분리
- 쉼표가 없으면 **전치사(in, of, on, for) 앞**에서 끊거나, 의미 단위(주어구·목적어구)로 분리
- 각 줄은 가급적 **본문 줄과 비슷한 길이**를 유지한다

### 예시
```
✗ 잘못된 줄바꿈 (한 줄로 길게):
(Red yeast rice and omega-3 PUFAs combination, PMC 2024)

✓ 올바른 줄바꿈:
(Red yeast rice and
omega-3 PUFAs combination,
PMC 2024)
```

```
✗ 잘못된 줄바꿈:
(Effect of GutGard in Management of H. pylori, RCT 2020)

✓ 올바른 줄바꿈:
(Effect of GutGard
in Management of H. pylori,
RCT 2020)
  → 쉼표 기준 + 전치사 'in' 앞에서 끊음
```

### 5줄 제한 — 기계적 카운트
"최대 5줄"은 감으로 맞추지 않고 **반드시 센다**.

작업 절차:
1. 문단을 줄바꿈 처리한다
2. 연속된 줄(빈 줄 전까지)의 개수를 센다
3. 6줄 이상이면 의미 단락 전환점에서 빈 줄을 추가해 분리한다
4. 분리 후에도 각 블록이 5줄 이내인지 재확인한다

감으로 "이 정도면 되겠지"로 넘기지 않는다.

### 작업 절차 (필수)
각 문단을 쪼갠 직후 다음을 확인한다:
1. 각 줄을 읽어보고 조사/서술어 단독 줄이 있는지 확인
2. 있으면 앞뒷줄과 합쳐서 의미가 이어지게 수정
3. 수정 후 줄 수를 세어 5줄 초과 여부 확인

## 3. 볼드 규칙

- **본문 전체에 볼드를 넣지 않는다** — 일반 서술 문장은 볼드 없이 작성
- 볼드는 아래 경우에만 사용:
  - 색상이 적용된 텍스트 (빨간색, 파란색 등)
  - 목차 항목
  - 섹션 제목 (19pt 강조)
  - 밑줄 강조 문장
  - 형광펜이 적용된 텍스트
  - 제품 연결 문맥에서 강조가 필요한 부분 ("이 세 가지를 한 번에 챙길 수 있는", "복잡해 보이지만" 등)
  - bc 이미지 라벨

## 4. 글자 크기 규칙

- **기본 본문**: 12pt
- **강조 문장 (감정적 하이라이트, 소제목)**: 19pt, 볼드
- **편집 지시(초록 형광펜)**: 24pt, 볼드
- **URL**: 12pt
- 그 외 크기는 사용하지 않는다 (14pt, 20pt 등 사용 금지)

## 5. 색상 적용 규칙

### 색상 적용 범위 원칙
색상은 **구체적 핵심 내용**에만 적용하고, 그 앞의 도입·상황 설명·연결 문장은 일반으로 둔다. 전체를 칠하면 강조 효과가 희석되므로, 진짜 핵심만 칠해서 눈에 들어오게 한다.

```
✗ 잘못된 적용 (전부 파란색):
병원 다녀온 다음 날부터         ← 도입인데 파란색
생활 습관을 바꿨어요.           ← 도입인데 파란색
일단 엘리베이터 대신            ← 맥락인데 파란색
계단 올라가기는 유지하되         ← 맥락인데 파란색
내려가는 건 엘리베이터로 바꿨고요.  ← 핵심도 파란색

✓ 올바른 적용 (핵심만 파란색):
생활 습관을 바꿨어요.           ← 일반 (도입)
일단 엘리베이터 대신            ← 일반 (맥락)
계단 올라가기는 유지하되         ← 일반 (맥락)
내려가는 건 엘리베이터로 바꿨고요.  ← 파란색 (실천 핵심)
```

빨간색도 동일하게 적용: 상황 설명은 일반, 증상·경고의 핵심 내용만 빨간색.

### 빨간색 (FF0000) + 볼드
**부정적/경고/위기 상황** 문장에 적용. 키워드만이 아니라 **의미 문장 단위**로 넓게 칠한다.

적용 대상:
- 위험 수치, 진단명: "혈압이 158/96", "2기 고혈압 범위"
- 증상 묘사: "머리가 띵하고", "숨이 차더라고요", "심장이 쿵쿵쿅 뛰면서 식은땀"
- 부정적 상황 전체 문장: "혈압 재봤더니 수축기가 / 160 넘어가 있던 적, / 혹시 있으신가요?" (3줄 전체)
- 방치/악화: "고혈압 160대를 반년 넘게 / 방치한 거라고요." (전체)
- 원인 설명 블록: "중성지방이 높으면 / 피가 끈적해지고, / 콜레스테롤이 쌓이면 / 혈관이 좁아지고," (전체)
- 핵심 경고: "혈관 전체가 망가지고 있다는 / '체질의 신호'"
- 마무리 당부: "혈압 수치에만 매달리지 말고, / 근본 체질부터 들여다보라고요."

**핵심**: 키워드 1~2개만 칠하는 게 아니라, 해당 문장이 전달하는 감정/경고 전체를 칠한다.

### 파란색 (0000FF) + 볼드
**긍정적 변화, 행동, 효능** 문장에 적용. 역시 문장 단위로 넓게.

적용 대상:
- 마음가짐 전환: "근데 저는 이때 / 생각이 달랐어요."
- 실천 행동: "식단을 바꿨어요", "레몬즙이나 / 후추로 간을 했거든요.", "매일 저녁 30분씩 / 빠르게 걷기도 시작했어요."
- 개선 체감: "머리가 좀 덜 무거운 느낌", "요즘은 아침에 일어나면 / 머리가 맑아요.", "확실히 예전보다 / 나아진 느낌이에요.", "혈색이 좋아 보인다"
- 수치 개선: "수치도 안정적인 범위로 / 돌아왔답니다."
- 핵심 메시지: "몸이 스스로 조절할 수 있게 / 환경을 만들어주는 거였어요.", "중성지방, 콜레스테롤, / 혈관 노화를 한꺼번에 케어"
- 식약처 인정 문구: "'혈중 중성지질 개선과 혈행 개선에 도움을 줄 수 있음'" 등
- 정상 수치 언급: "정상이 120/80 이하"

### 빨간색 vs 파란색 판단 기준
| 문맥 | 색상 |
|------|------|
| 진단, 수치 이상, 증상, 악화, 방치 | 빨간색 |
| 행동 변화, 실천, 개선 체감, 효능 | 파란색 |
| 같은 수치라도 "높게 나왔다" = 빨간, "정상 범위" = 파란 | 문맥에 따라 |

### 수치 없는 증상 서술형 적용 예시
원고에 구체적 수치·진단명이 없고 **주관적 증상 서술·체감 변화**만 있는 경우에도 동일한 기준을 적용한다. 위 예시가 수치 기반이라고 해서 수치 없는 원고에 색상을 적게 칠하는 것은 잘못이다.

```
빨간색 대상 (주관적 증상·반복·악화):
- "명치 쪽이 갑자기 쥐어짜듯 아픈"
- "일주일에 두세 번씩 반복됐다"
- "밥 먹다가 숟가락을 내려놓고 / 한참을 가만히 있어야 할 때"
- "점심 먹고 사무실 의자에서 / 쪼그리고 앉아 명치를 누르고"

파란색 대상 (체감 개선·실천):
- "아침을 챙겨 먹는 거였다"
- "속이 가벼운 날이 꽤 생겨서"
- "명치 쪽이 불편해서 / 일을 못 하는 날이 확실히 줄었다"
- "예전처럼 쪼그리고 앉아 있는 일은 없어졌다"
```

**핵심:** 수치가 없어도 "전에는 이랬다(부정)" = 빨간색, "지금은 이렇다(긍정)" = 파란색으로 문장 단위로 적용한다.

## 6. 형광펜 규칙

### 하늘색(cyan) 형광펜 + 볼드
- 메인 검색 키워드: 독자가 검색해서 유입되는 핵심 키워드 (예: "관절 영양제 추천", "고혈압 160")

**메인 검색 키워드 반복 등장 시 처리 규칙:**
한 원고 안에 메인 검색 키워드가 여러 번 등장하는 경우, 모든 언급에 형광펜을 칠하지 않는다. 전부 칠하면 강조 효과가 희석되고 SEO용 반복처럼 보인다.

- **형광펜 적용 대상:** 원고 내 **최초 등장 1회만** 형광펜 적용
- **일반 처리 대상:** 두 번째 이후 모든 언급은 **형광펜 없이 일반 텍스트**로 둔다
- 합계 기준: 한 원고당 형광펜 처리 총 **1회**

예시 (메인 검색 키워드 "혈액순환영양제"가 5회 등장하는 경우):
```
1회차 (최초 등장) → 하늘색 형광펜 ✓
2회차 (중간 본문) → 일반 처리
3회차 (중간 본문) → 일반 처리
4회차 (중간 본문) → 일반 처리
5회차 (마무리) → 일반 처리
```

### 노란 형광펜 + 볼드
- 핵심 성분명 (소제목 역할): "알티지 오메가3", "홍국", "코엔자임Q10"
- 감정적으로 강조할 인용문: "이걸 평생 먹어야 하나"
- 목차 하단 성분 정리: "- 알티지 오메가3", "- 홍국", "- 코엔자임Q10"

### 검정 형광펜 + 볼드 (흰색 글자)
- 제품명/배합명: "블러디션 배합", "블러디션"

**허브키워드 반복 등장 시 처리 규칙:**
한 원고 안에 허브키워드가 여러 번 등장하는 경우, 모든 언급에 형광펜을 칠하지 않는다. 전부 칠하면 강조 효과가 희석되고 광고 냄새가 난다.

- **형광펜 적용 대상:** ① 원고 내 **최초 등장** 1회 + ② **마무리·CTA 구간의 결론 강조** 1~2회
- **일반 처리 대상:** 중간 본문에서 반복 언급되는 허브키워드는 **형광펜 없이 일반 텍스트**로 둔다
- 합계 기준: 한 원고당 형광펜 처리 총 **2~3회 이내**

예시 (허브키워드 "스토마이신 배합"이 5회 등장하는 경우):
```
1회차 (최초 등장) → 검정 형광펜 ✓
2회차 (중간 본문) → 일반 처리
3회차 (중간 본문) → 일반 처리
4회차 (중간 본문) → 일반 처리
5회차 (마무리 CTA) → 검정 형광펜 ✓
```

### 초록 형광펜 + 볼드
- 블로거 편집 지시사항 전용 (본문 내용이 아님)
- 인용구 번호, 글자크기, 링크도구 안내 등

## 8. 편집 지시 (초록 형광펜) 삽입 위치

### 인용구 번호 규칙
네이버 인용구는 1~6번 중 자유롭게 선택하되, **같은 위계는 같은 번호로 통일**한다.

- 대제목은 문서 내 모든 대제목이 같은 번호
- 소제목은 문서 내 모든 소제목이 같은 번호
- 대제목과 소제목은 서로 다른 번호를 사용
- 허브키워드는 인용구를 사용하지 않음

예시:
```
대제목 → 인용구 ○번 (문서 내 모든 대제목 동일)
소제목 → 인용구 ○번 (문서 내 모든 소제목 동일, 대제목과 다른 번호)
허브키워드 → 인용구 사용 안 함
```

### 목차 아래
```
[목차]  ← 19pt, 볼드
1. ...
2. ...
3. ...
ㄴ 글자 크기 19/볼드  ← 초록 형광펜
```

### 대제목 (섹션 제목) 아래
```
1. 괜찮겠지, 라는 말의 끝  ← 19pt, 볼드
ㄴ 인용구 ○번 글자 크기 24, 글꼴 두껍게  ← 초록 형광펜, 24pt
```

### 소제목 아래
```
그냥 넘긴 1개월  ← 19pt, 볼드
ㄴ 인용구 ○번 글자 크기 19, 글꼴 두껍게  ← 초록 형광펜, 24pt
```

### 감정 강조 문장 아래
```
앞으로 같은 출발선에서  ← 19pt, 볼드
6개월을 방치한 사람과 관리한 사람,
어떤 차이가 생기는지 보여드릴게요.
ㄴ 글자 크기 24/볼드  ← 초록 형광펜, 24pt
```

### 허브키워드
허브키워드에는 인용구를 적용하지 않는다. **글자 크기도 키우지 않는다** — 검정 형광펜 + 볼드만 적용하고, 본문과 동일한 기본 크기(12pt)를 유지한다. "ㄴ 글자 크기 24" 초록 지시는 허브키워드에 사용하지 않는다.

**허브키워드가 단독으로 쓰일 때:**
```
"블러디션"  ← 검정 형광펜 + 볼드 (기본 크기 유지)
```

**허브키워드가 문장 안에 포함될 때:**
별도의 초록 지시 없이 해당 단어에만 검정 형광펜 + 볼드를 적용한다.
```
그게 나노카틸이었어요.
  → '나노카틸'에만 검정 형광펜 + 볼드, 글자 크기는 본문과 동일

그래서 운동이랑 식단,
나노카틸 루틴은
당분간 쭉 가져가려고 해요.
  → '나노카틸'에만 검정 형광펜 + 볼드, 글자 크기는 본문과 동일
```

### 마지막 링크
```
[URL]  ← 12pt, 파란색(1155CC), 밑줄
ㄴ 링크 도구로 삽입  ← 초록 형광펜
```

## 10. 체크리스트

### 작업 완료 직전 필수 점검 (먼저 확인)

- [ ] **원고 대조**: 최종 결과물의 텍스트가 원고와 문장 단위로 완전히 일치하는가
      (목차·제목·문장이 추가되거나 수정되지 않았는가)
- [ ] **마크다운 헤더 잔존 검사**: 최종 결과물에 `##` 기호가 한 글자도 남아 있지 않은가 (§0-2)
- [ ] **줄 수 카운트**: 모든 연속 줄 블록을 실제로 세어 5줄 이내인가
- [ ] **조사/서술어 단독 줄 검사**: 아래 패턴이 단독 줄로 있는가
      - "에 많고,", "으로", "에서" 등 조사 토막
      - "했어요.", "있어요." 등 짧은 서술어 토막
- [ ] **블로거 요청사항 색상 범위**: 제목 줄에만 빨간 볼드, 항목은 검정 일반

이 다섯 항목을 통과하지 못하면 다른 체크로 넘어가지 않는다.

### 일반 점검

서식 적용 완료 후 아래 항목을 점검한다.

- [ ] 도입부를 포함한 모든 영역에서 색상·볼드 규칙이 일관되게 적용되었는가
- [ ] 본문 전체에 불필요한 볼드가 없는가
- [ ] 2~5자짜리 토막이 혼자 한 줄을 차지하지 않는가
- [ ] 빨간색이 키워드만이 아닌 문장 단위로 적용되었는가
- [ ] 색상이 도입·연결 문장까지 과도하게 칠해지지 않고 핵심 내용에만 적용되었는가
- [ ] 빈 줄 없이 연속되는 줄이 5줄을 넘지 않는가
- [ ] 파란색이 긍정/행동/개선 문장에 빠짐없이 적용되었는가
- [ ] 형광펜(노랑/검정/초록)이 올바른 대상에만 적용되었는가
- [ ] 강조 글자 크기가 19pt 또는 24pt만 사용되었는가
- [ ] 모든 대제목/소제목 아래에 초록 형광펜 편집 지시가 있는가
- [ ] 인용구 번호가 위계별로 통일되어 있는가 (대제목끼리 동일, 소제목끼리 동일, 서로 다른 번호)
- [ ] 제목 반복 3줄에 볼드가 없는가
- [ ] URL이 12pt, 파란색(1155CC), 밑줄로 처리되었는가
