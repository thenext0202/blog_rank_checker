# 네이버 블록 체커 — 설계 문서

작성일: 2026-06-04

## 1. 목적

네이버에서 키워드를 검색했을 때, 그 통합검색 결과(SERP)에 **블로그 계열 블록**이 어떻게 뜨는지 정리한다.

확인 대상 3종:
- **인기글** — 헤더가 "인기글"(또는 "건강·의학 인기글"처럼 분야 테마)인 묶음 블록
- **스마트블록(스블)** — "[키워드] 인기글 / [키워드] 추천 / [키워드] 순위 / [키워드] 인기 카페글" 등 주제 묶음 블록
- **통검 블로그** — 블록으로 묶이지 않고 통합검색에 낱개로 흩뿌려진 블로그 글

각 블록 안에서 네이버가 기본 노출하는 글들의 **발행일**을 함께 수집한다. PC 화면 기준.

비범위(이번엔 안 함): 파워링크·가격비교·플러스스토어·네이버클립·인플루언서·지식iN·지식백과·뉴스·웹사이트·나무위키 등 블로그 계열 아닌 블록. 카페 낱개 글도 제외(카페 묶음 스블은 스블로 기록).

## 2. 검증된 분류 규칙 (실데이터 5건)

캡처·추출로 검증한 키워드: `오메가3 영양제`(인기글), `코큐텐 영양제`(분야 인기글), `콘드로이친`(스블 3블록), `오메가3 추천`(스블 3블록), `고혈압 수치`(통검 낱개 5건). 전부 정답 일치, 오탐 0.

```
main_pack 전체에서 섹션 박스를 넓게 수집(조상 중복 제거).
유닛별 신호: 헤더 텍스트 / _fe_view_root 클래스 유무 / 블로그·카페 링크 수 / 날짜 토큰 수
판정:
  n_posts = 유닛 안 날짜 토큰 수 ≈ 묶인 글 수
  grouped = (n_posts >= 2)           ← 묶음 블록 여부의 결정적 신호

  블로그·카페 링크 0개                         → 제외
  헤더가 제외 목록(광고/가격비교/스토어/클립/AI/인플루언서/지식백과/
       지식iN/나무위키/위키백과/뉴스/도메인문자열/관련 브랜드 콘텐츠) → 제외
  _fe_view_root + 헤더 "…인기글" + grouped      → 인기글  (분야명 있으면 함께 기록)
  grouped (그 외 묶음)                          → 스블    (헤더=주제명 기록)
  낱개(글 1개)이고 블로그 링크 有               → 통검 블로그
  낱개 카페 글                                  → 제외
```

핵심 교훈:
- **링크 수로 묶음을 판정하면 안 됨** — 낱개 글 1개도 제목·프로필·댓글 링크로 링크가 10개 넘게 잡힌다(고혈압 "예쁜카페" 오탐 사례). **날짜 토큰 수(=글 수)** 가 묶음/낱개를 가르는 신뢰 신호.
- **`_fe_view_root` + "…인기글"** 이 인기글의 신호. 키워드 접두 "[키워드] 인기글"은 `_fe_view_root`가 없고 스블이다.
- 헤더가 "인기글"로 끝나도 키워드가 앞에 붙으면 스블(예: "'오메가3추천' 인기글").

## 3. 발행일 처리

- 유닛 텍스트에서 날짜 토큰을 정규식으로 추출: `N초/분/시간/일/주/개월 전`, `어제`, `그제`, `YYYY.MM.DD`.
- 상대 날짜는 캡처 시각(KST 기준 오늘) 기준 **절대일(YYYY.MM.DD)로 환산**해 저장.
- 글 수 ≈ 날짜 토큰 수(네이버가 기본 펼친 글만; 스블은 보통 2~3개 + 더보기).
- 셀 표기: `N건: 최신순 날짜 나열`. 인기글/통검은 한 줄, 스블은 블록마다 한 줄(셀 내 줄바꿈).

- 탭이 없으면 프로그램이 생성하고 헤더(A~I) + B열 체크박스 유효성을 기입. `ws.update()` + 명시 범위만 사용(append 금지).
- 키워드는 사용자가 A열에 직접 입력. B열 체크박스로 그 행만 실행(순위체커식 트리거).

| 열 | 이름 | 내용 |
|---|---|---|
| A | 키워드 | 사용자 입력 |
| B | 실행 | 체크박스 트리거 |
| C | 인기글 | `✅`/`❌`, 분야 인기글이면 `✅ 건강·의학` |
| D | 인기글 날짜 | `7건: 06.03, 06.02, …` (최신순) |
| E | 스블 | `✅ N블록`/`❌` |
| F | 스블 주제·날짜 | 블록마다 한 줄(줄바꿈): `맥스콘드로이친(3): 05.14, 2026.03.25, 2025.09.18` |
| G | 통검블로그 | `✅ N건`/`❌` |
| H | 통검 날짜 | `5건: 05.30, 05.21, …` |
| I | 상태 | `완료 06/04 11:02` / `오류: (메시지)` / 빈칸(대기) |

## 5. 구성 (모듈 분리)

```
블로그 블록 체커/
├── main.py          # 실행 흐름 + 모드(1회/감시)
├── sheets.py        # 별도 시트 생성·연결·결과 기록 (서비스계정, 공유)
├── serp_parser.py   # SERP 캡처 JS + 블록 분류 + 날짜 추출/환산
└── config.json      # 시트 ID 등 영구 저장
```

- Selenium 드라이버는 순위체커 `create_driver`(headless, 자동화 탐지 우회) 패턴 재사용.
- 인증: `blog_management_hub/credentials.json`(서비스계정 `blog-checking@…`) 또는 환경변수 `GOOGLE_CREDENTIALS_BASE64`. (CLAUDE.md의 `manuscript_generator/credentials.json`는 현재 없음 — 빌드 시 실제 존재 경로로 확정)
- 실행 모드: `python main.py`(체크된 행 1회 처리), `python main.py watch`(주기 감시, 선택). EXE/bat 배포 가능.

## 6. 빌드 시 추가 검증 (실데이터로 확정할 것)

- 날짜 토큰 정규식이 모든 블록 유형에서 글 수와 맞는지 재확인(특히 "더보기" 뒤 숨은 글 제외가 의도대로인지).
- 제외 헤더 목록이 새 키워드에서 다른 비블로그 블록을 놓치지 않는지(예: 새 서비스 블록 등장 대비).
- 분야 인기글의 분야명 파싱("건강·의학" 등) 정확도.
- 스블/통검 낱개의 날짜 마크업이 인기글과 다른 경우 대비(현재는 유닛 텍스트 정규식이라 클래스 변화에 강건).
