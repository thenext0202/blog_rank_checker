# STATE

## 1. 프로젝트 개요

- **목적**: 네이버에서 키워드를 검색하면, 그 통합검색 결과에 블로그 계열 블록(인기글·스마트블록·통검블로그)이 뜨는지와 각 블록 안 글들의 발행일을 시트에 정리한다.
- **범위 (포함)**: 블로그 계열 3종 — 인기글 / 스마트블록(스블) / 통검 블로그 낱개. 각 블록 글별 발행일.
- **범위 (제외)**: 파워링크·가격비교·플러스스토어·네이버클립·인플루언서·지식iN·지식백과·뉴스·웹사이트·나무위키 등 비블로그 블록. 카페 낱개 글(카페 묶음 스블은 스블로 기록).
- **맥락**: 마케팅 키워드 리서치용. 순위체커(`순위체커/`)와 별개 — 순위체커는 "내 URL 순위", 이건 "SERP 블록 구조 분석". 드라이버 생성 등 검증 코드 일부 재사용.

## 2. 시스템 정의

### 2.1 AI — 블록 분류 (실캡처 5건 검증, 오탐 0)

main_pack 전체에서 섹션 박스를 넓게 수집(조상 중복 제거) → 유닛별로 헤더·`_fe_view_root` 클래스·블로그/카페 링크 수·날짜 토큰 수로 판정:

- `n_posts` = 유닛 안 날짜 토큰 수 ≈ 묶인 글 수. `n_posts >= 2` 이면 묶음 블록.
- `_fe_view_root` + 헤더 "…인기글" + 묶음 → **인기글** (분야명 포함 "건강·의학 인기글"도 인기글)
- 그 외 묶음 → **스마트블록** (헤더=주제명, "[키워드] 인기글/추천/순위/인기 카페글" 등)
- 낱개(글 1개) + 블로그 링크 → **통검 블로그**
- `_fe_view_root` + 블로그글 0 ("관련 브랜드 콘텐츠") → 제외(브랜드 광고)
- 제외 헤더 목록(가격비교/스토어/클립/AI/인플루언서/지식백과/지식iN/나무위키/도메인 등) → 제외

핵심: **링크 수가 아니라 날짜 토큰 수(=글 수)** 로 묶음/낱개를 가른다 (낱개 글도 프로필·댓글 링크로 링크가 10개+ 잡혀 오탐 유발).

### 2.2 AI — 발행일 추출/환산

- 유닛 텍스트에서 정규식으로 날짜 토큰 추출: `N초/분/시간/일/주/개월 전`, `어제/그제`, `YYYY.MM.DD`.
- 상대 날짜는 실행일(KST) 기준 절대일로 환산. 개월은 30일 근사.
- 표기: `5/3(25일 전)` — 올해는 M/D, 다른 해는 YYYY/M/D + 실행일 기준 경과일 괄호 병기. 셀엔 `N건: 최신순...`.

### 2.3 시트 — 블록 체커 탭

- 데이터 소스: 사용자가 A열에 키워드 수동 입력, B열 체크박스로 트리거.
- 시트: 공통 운영시트 `1jflc…`의 `블록 체커` 탭 **단독 사용** (기존 24개 탭 불가침, `append_rows` 금지, `ws.update` 명시 범위만).
- 컬럼: A 키워드 / B 실행 / C 인기글 / D 인기글 날짜 / E 스블 / F 스블 주제·날짜(줄바꿈) / G 통검블로그 / H 통검 날짜 / I 상태
- 출력 셀: 가로·세로 가운데 정렬 + 줄바꿈. 상태 = `완료 MM/DD HH:MM` / `오류: …`.

## 5. 핵심 데이터·숫자

- **검증 키워드**: 5건 (인기글 2·스블 2·통검 1), 분류 정답 일치율 100%
- **블록당 글 수(실측)**: 인기글 ~7, 스블 블록당 ~3, 통검 낱개 1
- **테스트**: 24개 통과 (순수 함수 단위테스트)
